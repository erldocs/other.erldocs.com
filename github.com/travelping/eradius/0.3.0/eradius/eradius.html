modules_ready(Pid, Modules) -> term()

Announce request handler module availability. Applications need to call this function (usually from their application master) in order to make their modules (which should implement the eradius_server behaviour) available for processing. The modules will be revoked when the given Pid goes down.

trace_on(ServerIP, ServerPort, NasIP) -> term()

Start tracing requests from the given NAS coming in on the given server. Do not do do this on a production system, it generates lots of output.

trace_off(ServerIP, ServerPort, NasIP) -> term()

Stop tracing requests from the given NAS.
