tp_json_rpc_service

This module implements the generic part of a JSON-RPC service.

This module implements the generic part of a JSON-RPC service. It also manages the service registry and serves as the backbone of parameter validation.

Object-to-Record Conversion

One particularly useful facility is the automated conversion of JSON objects to and from Erlang records. Because record definitions exist only at compile time, the conversion routines are defined as macros in the tp_json_rpc.hrl include file. They are documented below.


?record_to_json_obj(RecordName::atom(), Record::tuple()) -> tpjrpc_json:jsonobj() | none()

This macro converts a record to a JSON object. Some things to be aware of:

undefined is converted to null

The values contained in the record should adhere to the tpjrpc_json:json() type specification. Among other things, this means that all strings must be encoded as binaries and the only kind of tuple allowed is {obj, proplist()}.
If any value cannot be encoded, the conversion will exit with reason json_incompatible.

The value passed in as Record must match the record definition of RecordName. If it doesn't, the conversion will exit with reason badarg.


?json_obj_to_record(RecordName::atom(), Obj::tpjrpc_json:jsonobj()) -> {ok, tuple()} | {error, not_object}

This macro converts a JSON object into an Erlang record. The conversion will ignore any keys in the object that that do not have a corresponding field in the record. For missing keys the default value specified in the record definition will be used. Furthermore,

null is converted to undefined.

{error, not_object} is returned if Obj is not a JSON object.


?json_obj_into_record(RecordName::atom(), Defaults::tuple(), Obj::tpjrpc_json:jsonobj()) -> {ok, tuple()} | {error, not_object} | {error, bad_defaults}

This macro performs the same function as ?json_obj_to_record/2, except that in case of missing keys the value used is taken from the Defaults record. You might find this macro useful if you want to merge an object into an existing instance of the record type in question.

{error, bad_defaults} is returned if Defaults does not match the definition of RecordName.
