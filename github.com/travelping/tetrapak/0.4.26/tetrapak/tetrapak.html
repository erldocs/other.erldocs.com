get(Key::string()) -> term()

Get a tasks return value. This function requires the task that is expected to contain the given return value key and then returns it's value.

Example use: 

     tetrapak:get("config:vcs:branch")//

get(Key::string(), Default::term()) -> term()

Get a tasks return value. Works like get/1, except that Default is. returned if Key refers to an unknown task or return value key.
Use this function with care because it somewhat obscures dependency problems.

fail(Fmt::string(), Args::[term()]) -> no_return()

Exit the current task. This function displays a message, expanding tilde escape sequences like io:format/2, then exits the current task. The failure will propagate to all other tasks which are currently running. If tetrapak was executed from the command line, the Erlang VM will be halted with a non-zero status code.

config_path(Key::string()) -> filename() (see module file)

Read an absolute filename from the configuration. If the filename is given relative in the config file it will be resolved against the current task's working directory. This function fails the current task if an unknown configuration parameter is specified.

Read an absolute filename from the configuration. Behaves like config_path/1, except Default is returned if the Key is unknown.
