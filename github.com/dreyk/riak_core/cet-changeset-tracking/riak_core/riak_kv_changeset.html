async_object_changed(BKey, Val, Pid) -> term()

Inform the changeset that the object with {Bucket, Key} has changed; this will cause a bit to get set in the bloom for that BKey for the current timebox interval. This function does not block the caller.

object_changed(BKey, Val, Pid) -> term()

Same as async_object_changed/3, except that it does block the caller. This ensures that all previous updates are logged; and thus creates backpressure on the caller to avoid a huge mailbox.
