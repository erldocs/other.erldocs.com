leader_cron

The leader_cron module provides a distrubuted task scheduler for executing tasks periodically.

The leader_cron module provides a distrubuted task scheduler for executing tasks periodically. The connected nodes elect a leader to manage task scheduling and execution. Should the current leader become unavailable a new leader node is elected who resumes task execution responsibilities.

There are several different ways to specify the schedule for a task. See leader_cron_task for details.

Each node that is part of the scheduling cluster must be working with the same list of nodes as given to start_link/1. If the node list needs to change leader_cron must be stopped on all nodes. Once stopped everywhere restart leader_cron with the new node list. Rolling updates currently are not supported.

start_link(Nodes) -> {ok, pid()} | {error, Reason}

  • Nodes = [node()]

Creates a linked process to manage scheduled tasks in coordination with the given nodes. The current node must be part of the node list. Each leader_cron node must be working with the same list of nodes to coordinate correctly.

  • Schedule = schedule() (see module leader_cron_task)

Schedules a named task. There cannot be more than one task with a given name at any one time. See leader_cron_task for scheduling details.

The name 'undefined' is reserved for all unnamed tasks and cannot be used.
