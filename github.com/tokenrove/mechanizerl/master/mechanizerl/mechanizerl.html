mechanizerl

A tool for testing web applications.

A tool for testing web applications. Follows the WWW::Mechanize API where convenient. Most methods return undefined if no request/response is available.

new() -> handle()

Creates a new mechanizerl instance. Use the return value as a prefix to call other functions, as in: 

  1> M = mechanizerl:new().
  {mechanizerl, <0.64.0>}
  2> M:get("http://localhost/").
  ok

find_link(N::integer() | {text, iodata()} | {url, iodata()}, Handle::handle()) -> undefined | #link{type=undefined | a | frame | link, id=undefined | string(), url=undefined | url(), text=undefined | string()}

Finds a link on the current page. If an integer is passed, finds the nth link, indexed from 1; if {text,iodata()} is passed, the link whose text content matches the regexp is found, per the re module; a tuple of the form {url,iodata()} chooses the link whose URL matches the supplied regexp.

See also: re.
