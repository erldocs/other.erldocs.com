ar_serial

Arduino serial port low level communication.

Arduino serial port low level communication.
Process communicate with Arduino via serial port that name placed in app.config as {tty, "/path/to/dev/file"}.
Module write data to serial port and read data from port per line.

Functions

send(Cmd::string()) -> ok

Send command to serial port

start_trace(F::string()) -> ok | {error, atom()}

Start controller commands and replies tracing to file F.

stop_trace() -> ok

Stop controller commands and replies tracing

reset_device() -> ok

Controller hardware reset. Function reset DTR/RTS.

start(F::Params) -> {ok, Pid::pid()} | {stop, Error::any()}

  • Params = {Module, Function, LineEndCharacters} | {Fun, LineEndCharacters} | Fun
  • Module = atom()
  • Function = atom()
  • Fun = function()
  • LineEndCharacters = string()

Start communication server, with given callback specified as {Module, Function, LineEndCharacters}
, {Fun, LineEndCharacters} or Fun. When is only Fun given, LineEndCharacters is "\r\n".

start() -> {ok, Pid::pid()} | {stop, Error::any()}

Start communication server, with default callback specification - module ar_grbl, function reply and LineEnd - "\r\n".

read(FD::pid(), Parent::pid(), LineEnd::string()) -> ok

Start serial port reader. Read data from serial port, that identified as low-level process pid - FD, and send data when line ends to parent process. Function should't be used manualy!
