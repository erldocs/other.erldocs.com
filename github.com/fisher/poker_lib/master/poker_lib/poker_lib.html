fair_shuffle(List::list()) -> ShList::list()

Associate each element in the list with a random number. The list is then sorted based on the generated number. Repeat this process log(n) times to ensure a fair shuffle.

get the winning probability for a given starting hand, exactly two cards. Nota bene: this can't be taken into account if we know more than our hand. If we know all players hands or the remaining deck we should use more complicated measurement.

analyse_hand(Hand::[integer()]) -> Answer::term()

Returns recognized type of combo. The length of Hand can be 5 or 7 cards. Most common combinations is High Card and One Pair (they form a cluster of more than 92% probability in sum). Thus, the fun is optimized to recognize those two first. Yes, I know, it looks ugly, but runs relatively fast.
