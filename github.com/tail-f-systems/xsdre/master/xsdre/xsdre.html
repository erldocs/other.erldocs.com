xsdre

XML Schema Definition Regular Expression Library.

XML Schema Definition Regular Expression Library.

This library implements W3C XML Schema regular expression functionality. It is implemented with NIF:s that use the regular expression functions in libxml2.

For a good summary of the difference between different regular expression implementations see, http://www.regular-expressions.info/refflavors.html.

Functions

compile(Pattern::binstring()) -> {ok, xre()} | {error, string()}

Compile a regular expression pattern. The Pattern is compiled and, if successful, a resource that can be used in subsequent calls to match/2 or run/2 is returned.

run(String::binstring(), RegExp::xre() | binstring()) -> {match, Captured::term()} | nomatch | {error, string()}

Run RegExp on String.

Returning Captured is not yet implemented, currently it is always [].

Note that if you are going to use the same pattern many times it is more efficient to compile it once and keep the compiled expression.

match(String::binstring(), RegExp::xre() | binstring()) -> boolean() | {error, string()}

Run RegExp on String and return true if it matches.

This function is equivalent to run/2 except it has a boolean return value.

Note that if you are going to use the same pattern many times it is more efficient to compile it once and keep the compiled expression.

is_deterministic(CompiledRegex::xre()) -> boolean()

Returns true if the compiled regular expression is deterministic.

is_xre(CompiledRegex::term()) -> boolean()

Returns true if term is a compiled regular expression (as returned by compile/1)

string(CompiledRegex::xre()) -> string()

Return the pattern (as a string) which this regexp corresponds to. This is the original pattern given in compile/1.
