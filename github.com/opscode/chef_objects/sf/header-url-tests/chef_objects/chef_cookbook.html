parse_version(Version::binary()) -> {Major::non_neg_integer(), Minor::non_neg_integer(), Patch::non_neg_integer()}

Given a binary parse it to a valid cookbook version tuple {Major, Minor, Patch} or raise a badarg error. Each of Major, Minor, and Patch must be non-negative integer values less than 2147483647 (max size of value in pg int column). It is acceptable to provide a value with zero, one, or two dots (1 is the same as 1.0.0). More than two dots is an error.
