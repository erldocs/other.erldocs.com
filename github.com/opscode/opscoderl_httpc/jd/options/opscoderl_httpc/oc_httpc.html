multi_request(PoolName::atom(), Fun::function(), Timeout::non_neg_integer()) -> any()

Sets up a multi_request in named pool. The fun passed in a single arity fun that will be passed an arity four fun. The arity four fun is RequestFun(Path, Headers, Method, Body). This RequestFun can be used to issue multiple requests on the same connection. The result of the RequestFun will be returned on each invocation.

add_pool(PoolName::atom(), Config::pool_config()) -> any()

Creates a pool with pool_name, which is an atom. The config must have the following values in it as a proplist; {root_url, string()}, {init_count, non_neg_integer()}, {max_count, non_neg_integer}. Ibrowse specific configuration can be passed as {ibrowse_options, [options]}. Additional pooler configuration can be passed as documented in the pooler application https://github.com/seth/pooler
