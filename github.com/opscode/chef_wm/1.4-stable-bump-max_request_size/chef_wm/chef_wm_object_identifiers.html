chef_wm_object_identifiers

The internal object identifier resource.

The internal object identifier resource. This resource maps a Chef object type and name to the internal identifiers for the object. It is not routed at present in OSC, but is used in OPC so that federated services can track Chef objects using the internal ids.

A request to obtain the identifiers for node web1 looks like GET /nodes/web1/_identifiers. If the object is found, the JSON response will have the following format: 

  {
    "id": ID,
    "authz_id": AUTHZID,
    "org_id": ORGID
  }

NOTE: only implemented for node objects
