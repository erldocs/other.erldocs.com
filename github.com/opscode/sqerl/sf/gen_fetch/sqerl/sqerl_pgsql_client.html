Functions

execute(StatementOrQuery::sqerl_query(), Parameters::[any()], State::#state{cn=any(), statements=dict(), ctrans=dict() | undefined}) -> {sqerl_results(), #state{cn=any(), statements=dict(), ctrans=dict() | undefined}}

Execute query or prepared statement. If a binary is provided, it is interpreted as an SQL query. If an atom is provided, it is interpreted as a prepared statement name.

Returns:{Result, State}

Result: - {ok, Rows} - {ok, Count} - {ok, {Count, Rows}} - {error, ErrorInfo}

Row: proplist e.g. [{<<"id">>, 1}, {<<"name">>, <<"Toto">>}]

prepare(Name::atom(), SQL::binary(), State::#state{cn=any(), statements=dict(), ctrans=dict() | undefined}) -> {ok, #state{cn=any(), statements=dict(), ctrans=dict() | undefined}}

Prepare a new statement.

unprepare(Name::atom(), X2::[], State::#state{cn=any(), statements=dict(), ctrans=dict() | undefined}) -> {ok, #state{cn=any(), statements=dict(), ctrans=dict() | undefined}}

Unprepare a previously prepared statement Protocol between sqerl_client and db-specific modules uses 3 parameters (QueryOrName, Args, State) for all calls for simplicity. For an unprepare call, there are no arguments, so the second parameter of the function is unused.
