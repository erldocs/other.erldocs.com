nksip_subscription

User Subscriptions Management Module.

User Subscriptions Management Module. This module implements several utility functions related to subscriptions.

Types

field() = subscription_id
                | event
                | parsed_event
                | class
                | answered
                | expires

id() = binary()

spec() = id() | nksip_request:id() | nksip_response:id()

status() = init
                 | active
                 | pending
                 | {terminated, terminated_reason()}
                 | binary()

terminated_reason() = {deactivated,
                               undefined | integer() >= 0}
                            | probation
                            | rejected
                            | timeout
                            | {giveup, undefined | integer() >= 0}
                            | noresource
                            | invariant
                            | forced
                            | undefined
                            | {code, nksip:response_code()}
                            | binary()

Functions

field(AppId::app_id() (see module nksip), SubscriptionSpec::spec(), Field::field()) -> term() | error

Gets specific information from the subscription indicated by SubscriptionSpec. The available fields are:

FieldTypeDescription
app_id {link nksip:app_id()} SipApp that created the subscription
status status() Status
event binary() Full Event header
parsed_event nksip:token() Parsed Event header
class uac|uas Class of the event, as a UAC or a UAS
answered nksip_lib:timestamp()|undefined Time first NOTIFY was received
expires integer() Seconds reamaining to subscription expiration

field(Subscription::subscription() (see module nksip), Field::field()) -> any()

Extracts a specific field from a #subscription structure

fields(AppId::app_id() (see module nksip), SubscriptionSpec::spec(), Fields::[field()]) -> [{atom(), term()}] | error

Gets a number of fields from the Subscription as described in field/3.

Get the subscripion's id from a request or response.

get_subscription(AppId::app_id() (see module nksip), SubscriptionSpec::spec()) -> dialog() (see module nksip) | error
