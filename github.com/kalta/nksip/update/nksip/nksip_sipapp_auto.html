nksip_sipapp_auto

Automatic pings and registrations support for SipApps.

Automatic pings and registrations support for SipApps. This module allows a SipApp to program automatic periodic sending of OPTION or REGISTER requests and related functions.

Functions

start_ping(AppId::app_id() (see module nksip), PingId::term(), Uri::user_uri() (see module nksip), Time::pos_integer(), Opts::proplist() (see module nksip_lib)) -> {ok, boolean()} | {error, invalid_uri}

Programs the SipApp to start a series of pings (OPTION requests) to the SIP element at Uri, at Time (in seconds) intervals. PingId indentifies this request to stop it later. Use get_pings/1 to know about ping status, or the callback function nksip_sipapp:register_update/3.

stop_ping(AppId::app_id() (see module nksip), PingId::term()) -> ok | not_found

Stops a previously started ping request.

get_pings(AppId::app_id() (see module nksip)) -> [{PingId::term(), OK::boolean(), Time::non_neg_integer()}]

Get current ping status, including if last ping was successful and time remaining to next one.

start_register(AppId::app_id() (see module nksip), RegId::term(), Uri::user_uri() (see module nksip), Time::pos_integer(), Opts::proplist() (see module nksip_lib)) -> {ok, boolean()} | {error, invalid_uri}

Programs the SipApp to start a series of registrations to the registrar at Uri, at Time (in seconds) intervals. RegId indentifies this request to stop it later. Use get_regs/1 to know about registration status, or the callback function nksip_sipapp:ping_update/3.

stop_register(AppId::app_id() (see module nksip), RegId::term()) -> ok | not_found

Stops a previously started registration series.

get_registers(AppId::app_id() (see module nksip)) -> [{RegId::term(), OK::boolean(), Time::non_neg_integer()}]

Get current registration status, including if last registration was successful and time remaining to next one.
