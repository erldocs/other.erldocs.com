new(Amount::binary(), Currency::currency_code()) -> laundry_money()

Assumes Amount uses comma for decimal separator, has no whitespace, no thousands separator. That means the only valid characters are 0123456789,-

It also assumes the amount is well-formed, ie ,13 is not 0,13, but money_laundry will not necessarily do anything good with it.

new(Amount::binary(), Currency::currency_code(), X3::format() (see module money_format)) -> laundry_money()

Will create a new money_laundry object assuming specified input format and converting to an accurate representation

format(Format::format() (see module money_format), Money_laundry::laundry_money()) -> binary()

Returns a string representation of the amount, defined by the given format. Oere returns an integer for SEK amounts. decimal returns a decimal form of any amount.

proper:check_specs/1 quickly finds examples that don't work, e.g. [oere,{money_laundry,sek,{rational,1,3}}] which is infinitely repeating.
