display

This is essentially a window that the framework will use for drawing and checking for controls.

This is essentially a window that the framework will use for drawing and checking for controls.

You should create only one of these and then use it with the mainloop.

Functions

new(Width::integer(), Height::integer()) -> Display::display()

This is exactly the same as the other new function, however this will use the default title.

new(Width::integer(), Height::integer(), Title::string()) -> Display::display()

Creates a new Display of the given size and with the given title. Note that the width and height are the size of the graphics area inside of the window, not the size of the window itself.

By default ErlWorld will shutdown the erlang runtime when this display is closed.

new(Width::integer(), Height::integer(), Title::string(), OnCloseFun::() -> term()) -> Display::display()

This new creates a new Display of the given Width and Height with the given Title. It will also run the given OnCloseFun when the display is closed. The result of this fun is ignored and it should have no parameters.

paint_actor(Display::display(), Actor::actor()) -> ok

This paints the given Actor to this Display. It will block until painting has finished.

Typically you don't ever need to call this. It is called for you by the mainloop.

get_size(Display::display()) -> {Width::integer(), Height::integer()}

Return a tuple in the format {Width, Height} depicting the size of the canvas.

get_width(Display::display()) -> Width::integer()

Returns the width of the graphics area of the Display.

get_height(Display::display()) -> Height::integer()

Returns the height of the graphics area of the Display.

get_controls(Display::display()) -> Controls::controls()

Returns a snapshot of the current state of the controls input.
