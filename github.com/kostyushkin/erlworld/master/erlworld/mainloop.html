mainloop

This is the mainloop structure in a game.

This is the mainloop structure in a game. It can be thought of as the binding between the applications display and the gameplay objects (the Actors). The Mainloop holds an Actor (which is typically a World) as the root of the game. The rest of the games contents should be placed within this.

The Mainloop also requires a Display to be placed into it for when it's run for it to draw the Actor it holds to, and to retrieve the controls from.

A Mainloop can be run or stopped with the corresponding functions. When stopped the Actor within it is removed.

Functions

new(Display::display()) -> MainLoop::mainloop()

Creates and returns a new mainloop object which wraps the given display.

new(Display, FrameRate) -> term()

run(MainLoop::mainloop(), Actor::actor()) -> ok

Tells the mainloop to run with the given Actor. This can also be used for replacing the Actor the MainLoop is using.

stop(MainLoop::mainloop()) -> ok

Stops the mainloop running. It will subsequently no longer hold an actor.

set_actor(MainLoop::mainloop(), Actor::actor()) -> ok

Sets the actor to use as the root actor in this mainloop on the next cycle of acts and paints

get_actor(MainLoop::mainloop()) -> Actor::actor()

Gets and returns the actor running within this mainloop.

get_controls(MainLoop::mainloop()) -> Controls::controls()

Returns a snapshot to the current state of the controls.
