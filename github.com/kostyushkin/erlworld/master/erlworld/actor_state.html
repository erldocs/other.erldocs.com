actor_state

The actor_state is essentially a map that stores properties to values.

The actor_state is essentially a map that stores properties to values. What makes this different to other maps is that it also includes some common properties that all actors require. These are a name, location and size.

Extra properties can be added by the user through the set function, and accessed via the get function.

By default this will terminate itself when removed from a World. To alter this behaviour you can redefine it's on remove fun with the 'set_on_remove' function.
