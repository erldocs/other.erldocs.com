gen_tcpd

The gen_tcpd behaviour is a generic way of accepting TCP connections.

The gen_tcpd behaviour is a generic way of accepting TCP connections.

As with all OTP behaviours it is using a set of callbacks for the specific parts of the code. 
   gen_tcpd module            Callback module
   ---------------            ---------------
   gen_tcpd:start_link -----> Module:init/1
   -                   -----> Module:handle_connection/2
   -                   -----> Module:handle_info/2
   -                   -----> Module:terminate/2
Callbacks 
   Module:init(Args) -> Result
       Types Args = term()
             Result = {ok, State} | {stop, Reason}

After start_link/5 has been called this function is called by the new to initialise a state. If the initialisation is successful the function should return {ok, State} where State is the state which will be passed to the client in in the next callback. Args is the Args passed to start_link/5.

   Module:handle_connection(Socket, State) -> void()
       Types Socket = socket()
             State = term()

When a TCP / SSL connection is accepted, Module:handle_connection(Socket, State) will be called from a new process. This process should handle the TCP connection and do whatever it wants to. When this function returns, the process exits.

The process which this is called from is linked to the gen_tcpd process. It is allowed to trap exits in the gen_tcpd process if this is wanted. It's also possible to pass the gen_tcpd process as part of the State argument to unlink from it.

It might seem strange that the process is not under some individual supervisor, but it has been shown that starting children under supervisors in a vary rapid pace can overload a supervisor and become a bottleneck in accepting connections.

   Module:handle_info(Info, State) -> noreply
       Types Info = term()
             State = term()

This function is called if the gen_tcpd process receives any messasge it dosen't recognise. E.g. {'EXIT', Pid, Reason} messages if the process is trapping exits.

   Module:terminate(Reason, State) -> void()
       Types Reason = term()
             State = term()
This function will be called if any of the other callbacks return {stop, Reason}.
