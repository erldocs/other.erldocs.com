Module Index


bit-torrent-erlang-

download
A module to handle the downloading the blocks and subblocks

erbit_app
the main application module to start the software module has application behavior

erbit_core
The purpose of the module is to spawn new processes per each torrent this module is responsible for updating the gui as well module has gen_server behavior

erbit_fileManager
The purpose of the module is to handle reading and writing in file Contributers: Farrokh tavakoli, readFile/1 -> read file normally Contributers: Amir Almasi, implement OTP behaviour, and estimate/4 -> to update the aplication bitfield to the file which already downloaded in this case we do not download the file that we already downloaded, application just starts seeding that one

erbit_guiConnector
The purpose of the module is to update jinterface mail box in java part

erbit_handChecker
The purpose of the module is to check the info hash of other peers peer that got application ip and port from tracker and they want to download

erbit_hashMaker
a module to make the hash based on the information in torrent file -info part-

erbit_makeId
The purpose of this module is to make application peer id and transaction id for udp trackers

erbit_messageProvider
a module to provide all messages and keep the bitfield

erbit_parser
The purpose of the module is to read and parse .torrent file containing information To see more detail about the module and its functions, take a look at bittorrent specification website.

erbit_peerConnector
A module to keep track of the information related to peers and their connection

erbit_pieceChecker
A module for keeping pieces hash and compare them with the downloaded data

erbit_pieceHash
The purpose of the module is to get all concatenated picese's hashes and separate 20bytes for each piece

erbit_portListener
module for listening to the port Contributers: Amir Almasi, implement OTP behaviour for the module

erbit_socketHandler
A module for implementing ets and take care of all sockets if there would be any new it will be added to ets Contributers: Amir Almasi, implement OTP behaviour, getting query from ets, comparing new sockets

erbit_sup
This module is the main supervisor that it's responsibility is to spawn the main modules of the system

erbit_torrent_sup
This module is a supervisor that it's responsibility is to spawn the torrent process per each torrent

erbit_torrentInfoManager
The purpose of the module is to get the file in binary and parse all data, also communicate with erbit_parser.erl, erbit_hashMaker.erl, and erbit_pieceHash.erl module has gen_server behavior

erbit_torrentManager
This module will be per each torrent and it's main responsibility is to monitoring the torrent

erbit_torrentManager_sup
This module is a supervisor that it's responsibility is to spawn the processes realted to each torrent for example parsing , connecting to tracker

erbit_trackerConnector
This module is for connecting to the http trackers Implement OTP behaviour: Amir Almasi - normal massage passing was changed to otp behaviuor

erbit_udpHandler
This module is for connecting to udp trackers

peerHandeler
This module is for connecting to peers

sender
This module is for send the binary messages

upload
This module is for uploading blocks
