ponos_task_runner_callbacks

Functions

call(CbMod, LoadGenName, Task, State) -> term()

Executed by the load generator when a task should be triggered, according to the load generator's load_spec.

concurrency_limit(CbMod, LoadGenName, State) -> term()

Executed by the load generator when a task was prevented from being triggered by the max_concurrent option.

This only applies to load generators where the max_concurrent option has been set to anything other than the default value of 0.

This will only execute once every time the concurrency limit is reached.

init(CbMod, LoadGenName, Args) -> term()

Executed by the load generator during the init phase (i.e. in the gen_server callback).

The return value is used as State for the task runner.

pause(CbMod, LoadGenName, State) -> term()

Executed by the load generator when beeing paused.

start(CbMod, LoadGenName, State) -> term()

Executed by the load generator when starting (through ponos:init_load_generators() or by setting the option {auto_init, true}).

Apart from init/3, this is the only function where altering the state of the task runner is allowed.

terminate(CbMod, LoadGenName, State) -> term()

Executed by the load generator when terminating.
