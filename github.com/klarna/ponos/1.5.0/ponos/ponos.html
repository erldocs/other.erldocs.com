Functions

add_load_generators(Args::[args()]) -> [ok | {error, {duplicated, name()}}]

Add load generators to ponos. If a load generator with the same name is added twice, the original one *won't* be overwritten.

get_load_generators() -> list()

Return a readable representation of all registered load generators.

get_max_concurrent(Name::name()) -> non_neg_integer()

Return the max_concurrent option of the load generator Name.

0 is equal to unlimited number of ongoing tasks.

init_load_generators() -> [ok | {error, {non_existing, name()}}]

Same as init_load_generators/1, but for all registered load generators.

init_load_generators(Names::[name()]) -> [ok | {error, {non_existing, name()}} | {error, already_started}]

Initialize all generators present in Names. Already initialized generators will be ignored.

is_running(Name::name()) -> boolean()

Return true if the load generator is generating load, false otherwise.

pause_load_generators() -> [ok | {error, {non_existing, name()}}]

Same as pause_load_generators/1, but for all registered load generators.

pause_load_generators(Names::[name()]) -> [ok | {error, {non_existing, name()}}]

Stop generating load for all load generators present in Names. Already paused load generators will be ignored.

remove_load_generators() -> [ok | {error, {non_existing, name()}}]

Same as remove_load_generators/1 but for all registered load generators.

remove_load_generators(Names::[name()]) -> [ok | {error, {non_existing, name()}}]

Stop generating load for load generators and remove them from ponos.

set_max_concurrent(Name::name(), MaxConcurrent::non_neg_integer()) -> ok

Set the maximum number of ongoing tasks a load generator may spawn.

This can be done as a protective measure (i.e. to not be overrun by processes), or as a means to simulate a fixed number of workers.

top() -> list()

Return a fairly readable list of properties regarding the load ponos is currently generating.
