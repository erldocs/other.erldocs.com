jesse_database

Schema definitions cache handling.

Schema definitions cache handling.

All the schema definitions are stored in an ETS table for quick access during validation. This module provides an interface for: 1) updating of schema definitions in runtime; 2) getting of a schema definition by a key. When an update is ordered, the update function checks a schema definition file timestamp and compares it to a timestamp for the same schema in the cache, so, it will never update a schema in the database if the definition file was not updated.

Functions

add(Schema::json_term() (see module jesse), ValidationFun::(any()) -> boolean(), MakeKeyFun::(json_term() (see module jesse)) -> any()) -> update_result()

Adds a schema definition Schema to in-memory storage associated with a key Key. It will overwrite an existing schema with the same key if there is any.

delete(Key::any()) -> ok

Deletes a schema definition from in-memory storage associated with the key Key.

update(Path::string(), ParseFun::(binary()) -> json_term() (see module jesse), ValidationFun::(any()) -> boolean(), MakeKeyFun::(json_term() (see module jesse)) -> any()) -> update_result()

Loads schema definitions from filesystem to in-memory storage. The function loads all the files from directory Path, then each schema entry will be checked for a validity by function ValidationFun, and will be stored in in-memory storage with a key returned by MakeKeyFun function.

In addition to a schema definition, a timestamp of the schema file will be stored, so, during the next update timestamps will be compared to avoid unnecessary updates.

Schema definitions are stored in the format which json parsing function ParseFun returns.

NOTE: it's impossible to automatically update schema definitions added by add_schema/2, the only way to update them is to use add_schema/2 again with the new definition.

read(Key::any()) -> json_term() (see module jesse) | no_return()

Reads a schema definition with the same key as Key from the internal storage. If there is no such key in the storage, an exception will be thrown.
