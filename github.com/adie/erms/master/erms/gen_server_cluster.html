gen_server_cluster

Functions

start(Name, TargetModule, TargetArgs, Options) -> term()

Starts the global or local server under the given name where the target gen_server is given by the target module. If some global server is already running, the server is started as a local server, otherwise as the new global server.

start_local_server(Name) -> term()

Starts a local server if a global server is already running, otherwise noGlobalServerRunning is returned.

get_all_server_pids(Name) -> term()

Returns the global server pid and the list of all local server pids: {pid(), [pid()]}

get_all_server_nodes(Name) -> term()

Returns the global server node and the list of all local server nodes: {node(), [node()]}

is_running(Name) -> term()

Returns true if there is some global server running, otherwise false.

stop(Name, Node) -> term()

Stops the global server.

get_target_state(Name) -> term()

Returns the target state.

link(Name, Pid) -> term()

Links the pid with the global server. This link is transferred to a new global server if the current one dies.

unlink(Name, Pid) -> term()

Removes the link between pid and the global server that was established before.

terminate(Reason, State) -> void()

This function is called by a gen_server when it is about to terminate. It should be the opposite of Module:init/1 and do any necessary cleaning up. When it returns, the gen_server terminates with Reason. The return value is ignored.
