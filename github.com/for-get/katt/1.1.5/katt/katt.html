Klarna API Testing Tool.

Klarna API Testing Tool

Use for shooting http requests in a sequential order and verifying the response.

Types

callbacks() = [{atom(), function()}]

details() = any()

http_header() = {string(), string()}

param() = string() | atom() | binary() | integer()

params() = [{param(), param()}]

reason() = atom()

run_error() = {error, reason(), details()}

run_result() = run_error() | scenario_result()

scenario_filename() = nonempty_string()

scenario_result() =
            {pass | fail,
             scenario_filename(),
             params(),
             params(),
             [transaction_result()]}

transaction_result() =
            {string(),
             #katt_request{method = string(),
                           url = string(),
                           headers = [http_header()],
                           body = binary() | null},
             params(),
             validation_result()}

validation_result() = pass | {fail, reason(), details()}

Run test scenario. Argument is the full path to the scenario file. The scenario file should be a KATT Blueprint file.

run(Scenario::scenario_filename(), Params::params(), Callbacks::callbacks()) -> run_result()

Run test scenario. First argument is the full path to the scenario file. Second argument is a key-value list of parameters, such as hostname, port. You can also pass custom variable names (atoms) and values (strings). Third argument is a key-value list of custom callbacks such as a custom parser to use instead of the built-in default parser (maybe_parse_body).
