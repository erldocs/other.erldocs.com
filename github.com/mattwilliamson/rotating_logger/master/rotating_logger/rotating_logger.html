rotating_logger

This is a plaintext logger which has the ability to rotate between log files when they are considered full.

This is a plaintext logger which has the ability to rotate between log files when they are considered full. When it reaches the last file, it will open the first file.

init(Args::PropList) -> {ok, State}

  • PropList = [Property]
  • Property = {Name, Value}

Whenever a new event handler is added to an event manager, this function is called to initialize the event handler.

Properties

Name Type Description Default
name string() Name of the log file, e.g. Name.0, Name.1, Name.N "log"
dir string() Directory where the files are stored. "."
max_files integer() Maximum number of log files before they are recycled. 5
max_bytes integer() Maximum number of bytes written before the logger moves to the next file. 10 * 1024 * 1024 (10MB)

handle_event(Event, State) -> {ok, State} | {swap_handler, Args1, State1, Mod2, Args2} | remove_handler

Whenever an event manager receives an event sent using gen_event:notify/2 or gen_event:sync_notify/2, this function is called for each installed event handler to handle the event.

Preferrably called by gen_event:notify/2 e.g. gen_event:notify(event_logger, {Class, Msg}) Class could be any term, but the following are recommended: error | warning | info

handle_call(Request, State) -> {ok, Reply, State} | {swap_handler, Reply, Args1, State1, Mod2, Args2} | {remove_handler, Reply}

Whenever an event manager receives a request sent using gen_event:call/3,4, this function is called for the specified event handler to handle the request.

handle_info(Info, State) -> {ok, State} | {swap_handler, Args1, State1, Mod2, Args2} | remove_handler

This function is called for each installed event handler when an event manager receives any other message than an event or a synchronous request (or a system message).
