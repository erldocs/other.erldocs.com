httpcluster

Handles cluster information.

Handles cluster information. This module runs the main primary-node functions such as triggering TTD timers on nodes, refreshing them if the nodes check in, and manipulating node info when requested.

The cluster information being run by the primary node then becomes the source of truth for all secondary nodes.

Nodes check in (health check) using raw_from_node/3. If they fail to do that within their TTD window, a node-down event is automatically triggered within the module and the cluster info is updated.

raw_from_node(Name, Raw, DoSync) -> {ok, {Evt, Nodes}}

  • Name = name() (see module hc_node)
  • Raw = undefined | evt_raw() (see module hc_evt)
  • DoSync = boolean()
  • Evt = undefined | evt() (see module hc_evt) | evt_err() (see module hc_evt)
  • Nodes = undefined | cnodes() (see module hc_node)

Process an evt_raw, if present, coming from a single source (Name). Return the result, along with the most recent list of nodes if DoSync is true. The nodes returned in this way is the list just before the evt_raw is applied.

Calling this function also serves as a health check for Name regardless if there was an evt_raw or not.

new_cluster_info(Nodes, Evts) -> ok

  • Nodes = cnodes() (see module hc_node)
  • Evts = [evt() (see module hc_evt) | evt_err() (see module hc_evt)]

Reset the server state with a new set of Nodes and Evts.
