read_certs(Path::filename_all() (see module file)) -> [certificate() (see module x509)]

Given a file or directory path, returns a list of records of type x509:certificate() representing all of the identifiable, unencrypted X.509 certificates at or below the specified Path.

The returned certificates HAVE NOT been verified in any way beyond successfully parsing them. See read_cert_file/1 for details about how each located file is evaluated and parsed.

Duplicates are removed from the result set, but otherwise NO ordering is assured.

If the operation cannot be completed successfully, an error is raised, most likely containing one of the file:posix() reasons and, if you're lucky, possibly the path where the error occurred.

read_cert_file(Path::filename_all() (see module file)) -> [certificate() (see module x509)]

Read a single file and return its certificates as a list of records of type x509:certificate().

Filename extensions are considered as follows, regardless of case:

Files with extension ".der" are assumed to be binary DER-encoded certificates, and if they cannot be parsed as such they are ignored.

Files with extension ".pem" are assumed to be PEM-encoded, and any entities identified as certificates are decoded.

Files with extension ".key" are assumed to be private keys, in any format, and are ignored.

All other files are first parsed as binary DER-encoded certificates, and if that fails they are re-tried as PEM-encoded, as above.

If the operation cannot be completed successfully, an error is raised, most likely containing one of the file:posix() reasons and the path where the error occurred.

read_key_file(Path::filename_all() (see module file)) -> privatekey() (see module x509)

Read a single file and return its certificates as a list of records of type x509:certificate().

Filename extensions are considered as follows, regardless of case:

Files with extension ".der" are assumed to be binary DER-encoded certificates, and if they cannot be parsed as such they are ignored.

Files with extension ".pem" are assumed to be PEM-encoded, and any entities identified as certificates are decoded.

Files with extension ".key" are assumed to be private keys, in any format, and are ignored.

All other files are first parsed as binary DER-encoded certificates, and if that fails they are re-tried as PEM-encoded, as above.

If the operation cannot be completed successfully, an error is raised, most likely containing either the atom 'notkey' or one of the file:posix() reasons, and the path where the error occurred.
