parse_dt3(TS::nonempty_string(), Pats::[nonempty_string()]) -> {any(), any(), any()}

Parse the specified timestamp into three fields using one of a specified list of patterns.

Patterns are as specified for io:fread/2. The list of patterns is tried starting from the head, and the first one that matches is used to produce the result.

The result is assumed to be either a calendar:date() or calendar:time(), though no validation is performed.

If the timestamp does not match any of the specified patterns exactly (with no unparsed characters) error:{unrecognized, TS} is raised.

parse_dt6(TS::nonempty_string(), Pats::[nonempty_string()]) -> gsecs_ts() (see module x509)

Parse the specified timestamp into gregorian seconds using one of a specified list of patterns.

Patterns are as specified for io:fread/2. The list of patterns is tried starting from the head, and the first one that matches is used to produce the result.

The result is assumed to be a list of six integers, representing year, month, day, hour, minute, and second, respectively.

The hour is assumed to be in 24-hour format.

The year will be normalized as if by normalize_year/1.

The time zone is assumed to be UTC.

The resulting values must equate to a well-formed calendar:datetime(), which is then parsed as if by calendar:datetime_to_gregorian_seconds/1.

If the timestamp does not match any of the specified patterns exactly (with no unparsed characters) error:{unrecognized, TS} is raised.

See also: normalize_year/1, calendar:datetime_to_gregorian_seconds/1.

parse_dt7(TS::nonempty_string(), Pats::[nonempty_string()]) -> gsecs_ts() (see module x509)

Parse the specified timestamp into gregorian seconds using one of a specified list of patterns.

Patterns are as specified for io:fread/2. The list of patterns is tried starting from the head, and the first one that matches is used to produce the result.

The result is assumed to be a list of six integers and a trailing string, representing year, month, day, hour, minute, second, and timezone, respectively.

The hour is assumed to be in 24-hour format.

The year will be normalized as if by normalize_year/1.

The timezone is parsed as if by parse_tz_offset/1.

The values prior to the timezone must equate to a well-formed calendar:datetime(), which is then parsed as if by calendar:datetime_to_gregorian_seconds/1 and adjusted by the time zone offset from UTC.

If the timestamp does not match any of the specified patterns exactly (with no unparsed characters) error:{unrecognized, TS} is raised.

See also: normalize_year/1, parse_tz_offset/1, calendar:datetime_to_gregorian_seconds/1.

parse_tz_offset(TZ::nonempty_string()) -> integer()

Parses the specified string and returns the represented offset from UTC in seconds.

Common timezone offset numeric representations, such as +/-0, +/-hmm, and +/-hhmm are recognized, as are the strings Z, UTC, and GMT representing UTC, but other alphabetical timezone names/acronyms are not.

If the string does not match any recognizable pattern error:{unrecognized, TZ} is raised.

parse_time(S::nonempty_string()) -> non_neg_integer()

Parses the specified 3 or 4 character string and returns the represented number of minutes or seconds.

The input string is in the form HiLo, where Hi is the number of high-order elements and Lo is the number of low-order elements, with each Hi equal to 60 Los. Hi may be represented by one or two digits, while Lo must always be represented by two digits.
As such, the string may represent hours and minutes, or minutes and seconds, and the result is (Hi * 60) + Lo. Neither Hi nor Lo are constrained to being within any range other than 0..99.

If the string is not comprised of 3 or 4 decimal digits error:{unrecognized, S} is raised.

normalize_year(Year::integer()) -> integer()

Normalize a year value to include the century.

If the specified year is greater than 99, it is returned unchanged.
If the specified year is less than zero, the result is the current year adjusted by the specified value.
If the specified year is in the range 0..99, it is added to the current or previous century, based on the value of ?X509_TWO_DIGIT_YEAR_PIVOT, defined in "x509.hrl".

This operation is functionally equivalent to normalize_year(Year, ?X509_TWO_DIGIT_YEAR_PIVOT).

normalize_year(Year::integer(), Pivot::0..99) -> integer()

Normalize a year value to include the century.

If the specified year is greater than 99, it is returned unchanged.
If the specified year is less than zero, the result is the current year adjusted by the specified value.
If the specified year is in the range 0..99, it is added to the current or previous century, based on the value of Pivot.

If the Pivot value is outside the range 0..99 error:{badarg, Pivot} is raised.
