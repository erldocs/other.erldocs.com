Compute baseball stats using Riak Map/Reduce.

Compute baseball stats using Riak Map/Reduce.

This module works with data from the Retrosheet archive (http://www.retrosheet.org/) to compute baseball statistics (currently only batting average).

Types

bat_count() = {Hits::integer(), AtBats::integer()}

chunk() = {chunk_type(), Offset::integer(), Data::binary()}

chunk_type() = suffix | prefix

map_result() = bat_count() | chunk()

Functions

load_events(Directory::string()) -> ok

Loads all of the event files (*.EV*) into a luwak file named for the directory, that is, it concatenates all events files into one big Luwak file.

batting_average(Filename::binary(), PlayerID::binary()) -> integer()

Find the batting average for the specified player in the specified Luwak file. The basic idea is to evaluate a map on the file's blocks that picks out every at-bat for the player id, and then reduce the results.

ba_map(LuwakBlock::luwak_block(), Offset::term(), PlayerId::binary()) -> [map_result()]

Find the at-bat results in this block for the given player. Result is a list of at least one element, a tuple of the form {Hits, AtBats}. The result may also include one suffix and/or one prefix record.

ba_reduce(CountsAndChunks::[map_result()], PlayerId::binary()) -> [map_result()]

Sum the intermediate results produced by the map phase. Result is a list of of at least one element, a tuple of the form {Hits, AtBats}. The result may also include suffix and prefix records that could not be matched to other prefix and suffix records.

count_at_bats(Data::binary(), PlayerId::binary()) -> bat_count()

Find the at-bat results in this block for the given player. Result is {Hits, AtBats}

find_uncounted_chunks(Data::binary(), PlayerId::binary(), Offset::integer()) -> [chunk()]

Finds chunks that are not complete records at the starts and ends of blocks. Return is an empty list if there are no incomplete records, a list of two elements if there is an incomplete record at the start and at the end, or a list of one element if there is only a incomplete record at either the start or the end.

Incomplete records at the start of a block are marked 'suffix', and their offset is that of the first byte of their data. Incomplete records at the end of a block are marked 'prefix', and their offset is that of the byte immediately following their last byte. The intention is that a prefix should be matched with its suffix by matching the offset of the two.
