ustring

Expose unicode string manipulation from ICU.

Expose unicode string manipulation from ICU.

The ustring module implements some unicode string manipuation functionality as NIF calls to the ICU library.

The library expects strings to be represented as Erlang binaries, encoded as UTF16 in this machine's native endianness.

Types

encoding() = latin1 | unicode | utf8 | utf16 | utf32 | {utf16, little} | {utf16, big} | {utf32, little} | {utf32, big}

length_type() = codeunits | graphemes

ustring() = binary()

Functions

encoding() -> {utf16, little | big}

Get the native encoding format - UTF16, either 'big' or 'little' endian.

This is useful for transcoding ustring output to other formats: 

       unicode:characters_to_binary(UstringOutput,
                                    ustring:encoding(),
                                    utf8)

new(String::binary(), Encoding::encoding()) -> ustring() | {error, term()}

Create a new ustring-compatible string. The input String will be re-encoded as the native UTF16 format, and its characters will be normalized (NFC).

The input Encoding may be any atom understood by the 'unicode' Erlang module.

cmp(String1::ustring(), String2::ustring()) -> integer() | {error, term()}

Compare the two strings for bit-wise equality. Returns 0 if the strings are equal, less than 0 if String1 sorts before String2, or greater than 0 if String2 sorts before String1.

casecmp(String::ustring(), String2::ustring()) -> integer() | {error, term()}

Compare the two strings for case-insensitive equality. Returns 0 if the strings are equal, less than 0 if String1 sorts before String2, or greater than 0 if String2 sorts before String1.

toupper(String::ustring()) -> ustring() | {error, term()}

Convert all characters in the input string to their upper-case equivalents.

tolower(String::ustring()) -> ustring() | {error, term()}

Convert all characters in the input string to their lower-case equivalents.

length(String::ustring(), LengthType::length_type()) -> integer() | {error, term()}

Find the length of the given string. If LengthType is the atom 'codeunits', then the length returned is the number of 16-bit elements in the string. If LengthType is the atom 'graphemes', then the length returned is the number of user-visible characters.
