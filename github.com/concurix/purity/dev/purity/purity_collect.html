purity_collect

Functions

module(Core::c_module() (see module cerl)) -> dict()

Analyse a module and return a lookup table of functions and dependencies, indexed by {Module, Function, Arity}.

Analysis starts from parsed core erlang terms.

See also: files/2.

files(Filenames::[filename() (see module file)]) -> dict()

Analyse a list of Erlang modules.

Each module is analysed separately, and the lookup table is incrementally updated. The modules can be Erlang source code or already compiled BEAM files with debug information.

See also: module/2.

file(Filename::filename() (see module file)) -> dict()

Analyse a single file. In case of error a message is printed and an empty lookup table is returned.

pfiles(Filenames::[filename() (see module file)]) -> dict()

Analyse a list of modules in parallel.

The number of parallel operations is limited to the number of cpu processors/cores in order to minimise memory use.

See also: files/1, module/2.
