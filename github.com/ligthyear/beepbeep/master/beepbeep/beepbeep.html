BeepBeep Dispatcher

This is called from the MochiWeb server to dispatch requests. Requests are mapped to modules (controllers) and functions (actions) based on path components in the Url. For example, the request:

'/feed/show'

would get mapped to the module (controller) 'feed' and the function (action) 'show'. By default the root request:

'/'

is automatically mapped to the module 'home' and the function 'index' Maps Urls to controllers and their views

Functions

dispatch(Env::record()) -> tuple()

Dispatches the incoming request to the proper module and function and returns the tuple() from the controller.

render_template(FullPathToFile, ViewFile, Data) -> term()

Render the template with the given data. This is called from YOUR_APP_web.erl automatically.
