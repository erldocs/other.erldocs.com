spi

This is the implementation of the SPI interface module.

This is the implementation of the SPI interface module. There is one process for each spi device. Each process is linked to the supervisor of the spi application.

Types

bits() = integer()

channel() = atom()

data() = tuple()

delay() = integer()

devname() = string()

len() = integer()

mode() = integer()

speed() = integer()

Functions

start_link(X1::{channel(), devname()}) -> {ok, pid()} | {error, reason}

Starts the process with the channel name and Initialize the devname device. You can identify the device by a channel name. Each cannel drive a devname device.

stop(Channel) -> term()

Stop the process channel and release it.

config(Channel::channel(), Mode::mode(), Bits::bits(), Speed::speed(), Delay::delay()) -> ok | {error, reason}

Configure the SPI device.

transfer(Channel::channel(), Data::data(), Len::len()) -> {data()} | {error, reason}

Transfer data trough the SPI bus.
