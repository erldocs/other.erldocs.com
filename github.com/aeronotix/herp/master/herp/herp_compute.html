Functions

create_server(ClientRef::ref(), ServerProp::proplist()) -> ok | {error, {Field::atom(), Reason}}

create_server/2 will provision a new server instance in the HPCloud. you must provide: 

     [{<<"flavorRef">>, ?FLAVOR_REF},
      {<<"imageRef">>, ?IMAGE_REF},
      {<<"name">>, string()}]

delete_server(ClientRef::ref(), ServerID::binary()) -> ok | {error, Reason}

delete_server/2 will take a ServerID and schedule that server for termination.

list_flavours(ClientRef::ref()) -> proplist()

list_flavours/1 returns a proplist with all the available flavours in the HPCloud. You shouldn't really need to use this as the list never really changes and we provide flavour/1 which holds all you need.

list_images(ClientRef::ref()) -> proplist()

list_images/1 returns a proplist with all the available images in the HPCloud. The images are fully listed and can be used to provision new servers.

error(Status::ref(), Body::proplist()) -> Exit | {error, Reason}

The compute API has a very clear outline for the different kinds of errors which it handles. They are all in the form: 

  {
       $REASON: {
           "message": $MESSAGE,
           "code"   : $CODE,
           "detail" : $DETAIL
       }
  }

This function will retreive the specific body for each message and return the correct message for it.

flavour(X1::atom()) -> binary()

flavour/1 will return the appropriate term for the flavorRef.
