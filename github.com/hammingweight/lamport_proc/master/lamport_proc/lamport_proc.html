lamport_proc

Lamport modelled processes as a sequence of events where an event can be the sending of a message, the receiving of a message or some local computation.

Lamport modelled processes as a sequence of events where an event can be the sending of a message, the receiving of a message or some local computation. An implicit assumption in Lamport's model is that a receiving process is informed of the time at which a message was sent as well as the process that sent the message. In particular, Lamport introduced the idea of 'logical time' also known as a Lamport timestamp.

Erlang's message passing does not intrinsically carry information about the originating process and the time. This module supplies Lamport timestamps to processes that implement the specified behaviour.

The lamport_proc behaviour is similar to the gen_server behaviour. The differences between the the two behaviours are:

A gen_server process is passed the sending process's identifier, PID, in synchronous (handle_call) calls while a lamport_process is passed a {T, PID} tuple where T is a Lamport timestamp.

A gen_server process is passed only a message and state in asynchronous (handle_cast) calls. A lamport_process is passed a third parameter, {T, PID} pair, where PID is the calling process identifier and T is a Lamport timestamp.
