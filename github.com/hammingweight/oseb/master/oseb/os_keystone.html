os_keystone

Functions for interacting with the OpenStack Keystone service.

Functions for interacting with the OpenStack Keystone service.

This module is used to authenticate a user against the Keystone service, to access authentication tokens and to locate service endpoints.

Functions

clone(Pid::pid()) -> {ok, pid()}

Spawns a clone of a Keystone process with the same state as the original process. Most services need access to Keystone, if one service crashes a Keystone process, unrelated services should not crash as a result. The service processes (e.g. Nova, Glance, etc.) clone the Keystone process and interact with the clone.

close(Pid::pid()) -> ok

Closes the process.

get_auth_token(Pid::pid()) -> {ok, string()}

Gets the authentication token to be sent in requests to OpenStack services.

get_endpoint(Pid::pid()) -> {ok, string()}

Gets the Keystone service endpoint.

get_extension_details(Pid::pid(), Alias::string()) -> {ok, jsondoc()}

Gets the details about an extension to the Keystone API.

get_public_url(Pid::pid(), ServiceType::string()) -> {ok, string()}

Gets the first public URL in the service catalog for a service of a specified type.

get_service_catalog(Pid::pid()) -> {ok, jsondoc()}

Gets the service catalog.

get_tenant_id(Pid::pid()) -> {ok, string()}

Gets the tenant ID.

list_extensions(Pid::pid()) -> {ok, jsondoc()}

Lists supported extensions to the Keystone service.

list_public_urls(Pid::pid(), ServiceType::string()) -> {ok, [string()]}

Lists public URLs for a specified service type.

list_tenants(Pid::pid()) -> {ok, jsondoc()}

Lists all tenants.

start(Endpoint::url(), Credentials::term()) -> {ok, pid()}

Spawns a process for interacting with the Keystone service if the supplied credentials can be validated.

start(Endpoint::url(), UserName::string(), Password::string(), TenantName::string()) -> {ok, pid()}

Spawns a process for interacting with the Keystone service if the supplied password can be verified for the associated user and tenant.
