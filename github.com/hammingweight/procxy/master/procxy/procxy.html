The proxcy behaviour defines callback functions to simplify writing client libraries.

The proxcy behaviour defines callback functions to simplify writing client libraries. Client libraries need to maintain state information about the state of a connection to a server (e.g. an authentication token). Assumptions about a typically client library are:

Client processes are typically short-lived compared to server processes. Since client processes are not long-lived, they do not need to be started by a supervisor. Instead client processes are instantiated explicitly by some client code and will interact only with the process that started the client process. When the process that instantiated the client process terminates, the process maintaining client state should also terminate. This in contrast to gen_server processes that process messages from more than one process and should not terminate if a process sending messages to the server process exits.

Differences: how they're started, short-lived and supervision, number of processes that interact with the process, when they terminate.
