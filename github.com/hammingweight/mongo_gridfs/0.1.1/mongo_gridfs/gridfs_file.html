gridfs_file

MongoDB GridFS File API.

MongoDB GridFS File API. This module provides functions for reading from a GridFS file and getting file information about a file.

Types

bucket() = atom()

file() = pid()

Functions

close(Pid::file()) -> ok

Closes a file.

file_name(Pid::file()) -> binary()

Returns the name of the file; i.e. the value of the file_name attribute.

file_size(Pid::file()) -> integer()

Returns the size of the file in bytes.

md5(Pid::file()) -> binary()

Returns the MD5 checksum of the file.

new(Gridfs_connection::#gridfs_connection{}, Bucket::bucket(), Id::value() (see module bson), ParentProcess::pid()) -> file()

Opens a file using a specified connection to a database. The file is identified by its ID.

pread(Pid::file(), Offset::integer(), Length::integer()) -> binary()

Reads bytes from an offset up to a maximum length.

read_file(Pid::file()) -> binary()

Reads the whole contents of a file.

set_timeout(Pid::file(), Timeout::integer()) -> ok

Sets the timeout for a file. If the file isn't accessed within the specified time, the file is closed.

init(X1) -> term()

Initializes the server with connection parameters, a bucket and an ID.

handle_call(X1, From, State) -> term()

Responds synchronously to server calls.

handle_cast(Msg, State) -> term()

Responds to asynchronous server calls. This process doesn't expect any asynchronous calls.

handle_info(Info, State) -> term()

Responds to out-of-band messages. The server ignores any such messages.

terminate(Reason, State) -> term()

Handles the shutdown of the server.

code_change(OldVsn, State, Extra) -> term()

Responds to code changes. Any code changes are ignored (the server's state is unchanged).
