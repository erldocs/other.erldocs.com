gridfs_cursor

MongoDB GridFS Cursor API.

MongoDB GridFS Cursor API. This module provides functions for retrieving GridFS files from a cursor.

Types

bucket() = atom()

cursor() = pid()

file() = pid()

Functions

close(Pid::cursor()) -> ok

Closes a cursor.

new(Gridfs_connection::#gridfs_connection{}, Bucket::bucket(), MongoCursor::cursor() (see module mongo), ParentProcess::pid()) -> cursor()

Creates a cursor using a specified connection to a database collection of files.

next(Cursor::cursor()) -> file() | {}

Returns the next GridFS file from a cursor or an empty tuple if there are no further files.

rest(Cursor::cursor()) -> [file()]

Returns all GridFS files from a cursor.

set_timeout(Cursor::cursor(), Timeout::integer()) -> ok

Sets a timeout for a cursor. If the cursor is not read within the specified time, the cursor is closed.

take(Limit::integer(), Cursor::cursor()) -> [file()]

Retrieves GridFS files from a cursor up to the specified maximum number.

init(X1) -> term()

Initializes the server with connection parameters, a bucket and a mongo cursor.

handle_call(X1, From, State) -> term()

Responds to synchronous messages. Synchronous messages are sent to get the next file, to get remaining files, to get the mongo:cursor(), to close the cursor and to set the timeout of the cursor.

handle_cast(Msg, State) -> term()

Responds asynchronously to messages. The module does not expect to receive asynchronous messages.

handle_info(Info, State) -> term()

Responds to non-OTP messages. The messages that are handled are a timeout and the the termination of the parent process.

terminate(Reason, State) -> term()

Shuts down the server.

code_change(OldVsn, State, Extra) -> term()

Handles code changes (changes are ignored).
