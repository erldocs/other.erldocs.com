sk_data

This module encapsulates all functions relating to data messages that are sent around the system.

This module encapsulates all functions relating to data messages that are sent around the system.

Data messages are an instance of an applicative functor, allowing various operations to happen in a very logical way.

Allows a given function to be applied to a value in the form of a Skel data message. Preserves message identifiers.

pure(D::any()) -> data_message()

Formats a piece of data under D of any data type as a data message.

apply(X1::data_message()) -> data_fun()

Allows a given function, formatted as a data message, to be applied to a value in the form of a Skel data message. Preserves message identifiers.

value(X1::data_message()) -> any()

Extracts and returns a data message's value.

push(Identifier::dm_identifier(), X2::data_message()) -> data_message()

Adds an identifier to a given data message.

pop(X1::data_message()) -> {dm_identifier(), data_message()}

Retrieves and removes the topmost identifier of a given message. Returns a tuple containing retrieved identifier, and the now-updated message.

peek(X1::data_message()) -> {ok, dm_identifier()} | empty

Retrieves, but does not remove, the topmost identifier of a given message. Returns only the identifier, or empty when no identifiers exist.
