sk_farm_collector

This module contains the collector logic of a Farm skeleton.

This module contains the collector logic of a Farm skeleton.

A task farm has the most basic kind of stream parallelism - inputs are sent to one of n replicas of the inner skeleton for processing.

The collector takes inputs off the inner-skeletons' output streams, sending them out on the farm skeleton's output stream. It does not preserve ordering.

Functions

start(NWorkers::pos_integer(), NextPid::pid()) -> eos

Initialises the collector; forwards any and all output from the inner- workflow to the sink process at NextPid.
