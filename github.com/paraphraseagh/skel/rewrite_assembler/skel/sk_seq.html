sk_seq

This module contains the most logic of the most basic kind of skeleton - seq.

This module contains the most logic of the most basic kind of skeleton - seq.

A 'seq' instance is a wrapper for a sequential function, taking an input from its input stream, applying the function to it and sending the result on its output stream.

Example 
        skel:run([{seq, fun ?MODULE:p/1}, {seq, fun ?MODULE:f/1}], Input).

In this example, Skel is run using two sequential functions. On one process it runs the developer-defined p/1 on the input Input, sending all returned results to a second process. On this second process, the similarly developer-defined f/1 is run on the passed results. This will only start once p/1 has finished processing all inputs and the system message eos sent. Results from f/1 are sent to the sink once they are available.
