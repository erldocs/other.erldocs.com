fetch(NodeId::integer(), Ix::integer(), Si::integer(), TransferMode::block | segment, Destination::{app, Pid::pid(), Mod::atom()} | {value, Type::integer()} | data) -> ok | {ok, Value::term()} | {ok, Data::binary()} | {error, Reason::atom()}

Description: Fetch object specified by Index, Subind from remote CANOpen node identified by NodeId.
TransferMode controls whether block or segment transfer is used between the CANOpen nodes.
Destination can be:

{app, Pid, Module} - data is sent to the application specified by Pid and Module.

{value, Type} - data is decoded and returned to caller. NOT IMPLEMENTED YET

data - data is returned to caller. NOT IMPLEMENTED YET

store(NodeId::integer(), Ix::integer(), Si::integer(), TransferMode::block | segment, Source::{app, Pid::pid(), Mod::atom()} | {value, Value::term(), Type::integer()} | {data, Bin::binary}) -> ok | {error, Reason::atom()}

Description: Stores object specified by Index, Subind at remote CANOpen node identified by NodeId.
TransferMode controls whether block or segment transfer is used between the CANOpen nodes.
Destination can be:

{app, Pid, Module} - data is fetched from the application specified by Pid and Module.

{value, Value, Type} - value is supplied by caller and encoded before the transfer. NOT IMPLEMENTED YET

{data, Data} - data is supplied by caller. NOT IMPLEMENTED YET
