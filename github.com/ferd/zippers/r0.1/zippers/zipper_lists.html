zipper_lists

Zipper lists allows to 'browse' a list in two directions, much like an imperative doubly-linked list, or a list with iterators that can go in both directions.

Zipper lists allows to 'browse' a list in two directions, much like an imperative doubly-linked list, or a list with iterators that can go in both directions. A zipper list can be used to represent items such as a browser's history. Access to the next and previous elements are both amortized constant time, and so are the costs of insertion and deletion in the current position. Looking up from the beginning or the end is in linear time. Note that zippers are not search data structures.

prev(X1::zipper_list()) -> zipper_list() | undefined

Accesses the previous element of the zipper list. If there is no previous element, the function returns undefined.

next(X1::zipper_list()) -> zipper_list() | undefined

Accesses the next element of the zipper list. If there is no next element, the function returns undefined.

current(X1::zipper_list()) -> {ok, term()} | undefined

Allows to read the value of the current list element. If the zipper is empty, the atom undefined is returned.

replace(Val::term(), X2::zipper_list()) -> zipper_list()

Changes the value of the current zipper list item.

insert(Val::term(), X2::zipper_list()) -> zipper_list()

Inserts a new element in the zipper. The element is added at the current zipper position. Successively adding the numbers 1..4 in a zipper without changing positions will result in a zipper equivalent to [4,3,2,1]. To avoid this and obtain a list that is in order, use the function next/1 after each insertion.

delete(X1::zipper_list()) -> zipper_list()

Deletes the element at the current position in the zipper.
