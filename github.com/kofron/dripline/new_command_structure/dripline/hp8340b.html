locator_to_ch_spec(Other::locator()) -> channel_spec() | {error, bad_locator}

locator_to_ch_spec is part of a generic instrument interface. essentially it allows us to take a string that is human readable and turn it into something that the instrument understands as referring to a channel. This is nice because in the database, the amount of information that the user needs to know about the instrument itself is reduced.

read(InstrumentID::atom(), Locator::locator()) -> binary() | {error, term()}

read/2 maps a request for data onto the correct instrument and synchronously returns either data or a descriptive error. Error codes that are returned are from the instrument itself. Because these are externally facing functions, we use locators instead of channel specs.

write(InstrumentID::atom(), Locator::locator(), NewValue::binary()) -> ok | {error, term()}

write/3 maps a write request onto the correct instrument and synchronously returns either an ok message or a descriptive error tuple. Error codes come from the instrument.
