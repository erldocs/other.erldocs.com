client_lowe

This is the client module for Needham-Schroeder-Lowe Protocol.

This is the client module for Needham-Schroeder-Lowe Protocol. Client module is the one which simulates Alice and Bob as clients in a communication. It will receive start signal from the driver process and start a negotiation with the Target specified by the driver. This is the version that client will send itself in the 6th message so that it can prevent man-in-the-middle attack.

Functions

start() -> pid()

Spawn a client_lowe process and return the pid.

gen_nonce() -> integer()

Generate a random nonce.

decrypt_server(Server::pid(), Msg::binary()) -> tuple()

Decrypt the message from server. It will get the keys from the key_table and decrypt the binary message from the server. It will parse the decrypted message as a tuple and return.

encrypt(Key::RSA_PublicKey, Msg::tuple()) -> binary()

Encrypt the message using specified key. Usually it will use other clients' public key to encrypt the message and send to other clients. It will accept a tuple and encrypt it into binary.

decrypt(Key::RSA_PrivateKey, Msg::binary()) -> tuple()

Decrypt the message from other clients. It will use the private key and decrypt the message from other clients. It will parse the decrypted message as a tuple and return.

loop() -> none()

Loop to keep receiving messages. 

 		Suppose Alice initiates a session with Bob.
 		{start, Target, Server}->
 			Start message from driver.
 		{replyPk, Server, Msg}->
 			Alice receive the Kpb requested, and try to initiate a session with Bob.
 		{comm_req, Msg}->
 			Receive the encrypted session request and get the Public Key of Alice.
 		{replyPk_2, Server, Msg}->
 			Only the one which is Bob will receive this, it will get the Kpa from server.
 			Also send the nonce as a part of the message. To prevent man-in-the-middle attack,
 			it will add itself into the encrypted message.
 		{nonces, Msg}->
 			Alice receives Na, Nb and B, then reply Bob with a Nb encrypted using Kpb.
 		{verified, Msg}->
 			Bob receives the Nb from Alice, and checks whether it's the correct one.
 			If yes then session is created. Then send a message to driver to annonce the end.
