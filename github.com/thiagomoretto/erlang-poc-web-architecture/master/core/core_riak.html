core_riak

Functions

connect(X1::connection_info()) -> pid()

Create a connection to the specified Riak cluster and return the Pid associated with the new connection.

create(Bucket::binary, Key::binary, JsonData::json) -> riakc_obj()

Create a new instance of a riak object using the parameters given. The riak object can then be persisted to a Riak node/cluster. This overload assumes that the data passed in is JSON and sets the MIME type to "application/json" for you.

create(Bucket::binary, Key::binary, Item::term(), MimeType::string) -> riakc_obj()

Create a new instance of a riak object using the parameters given. The riak object can then be persisted to a Riak node/cluster. This overload takes arbitrary data and requires the user to specify the mime type of the data that is being stored.

fetch(RiakPid::pid(), Bucket::binary, Key::binary) -> riakc_obj()

Fetches a riakc object from a Riak node/cluster using the connection given.

update(RiakObj::riakc_obj(), NewValue::term()) -> riakc_obj()

Updates the stored value for a riakc object with the new one specified.

get_value(RiakObj::riakc_obj()) -> term()

Retrieves the stored value from within the riakc object.

save(RiakPid::pid(), RiakObj::riakc_obj()) -> {ok, riakc_obj()} | {error | Reason}

Saves the given riak object to the specified Riak node/cluster.

delete(RiakPid::pid(), Bucket::binary, Key::binary) -> ok | {error | Reason}

Deletes riak object based on given key to the specified Riak node/cluster.

new_key() -> key()

Generate an close-to-unique key that can be used to identify an object in riak. This implementation is blatantly borrowed (purloined) from the wriaki source (thanks basho!)

new_key(List::list()) -> key()

Generate an close-to-unique key that can be used to identify an object in riak using the given list parameter as the stuff to hash.
