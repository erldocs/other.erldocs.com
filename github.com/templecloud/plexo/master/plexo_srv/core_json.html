core_json

The core_json module provides centralised Erlang Term / JSON conversion routines.

The core_json module provides centralised Erlang Term / JSON conversion routines.

Notes The current implementation uses the JSX and JSXN and libraries to perform the marshalling operations.

Functions

to_json(Term::#{} | list()) -> binary()

Encode the specified 'Erlang term' into a JSON encoded binary. All 'string' entities in the specified term must be binary encoded.

Example Input (Map Term) 

    #{<<"app_nfo">> => #{
      <<"description">> => <<"ERTS  CXC 138 10">>,
      <<"name">> => <<"kernel">>,
      <<"version">> => <<"3.0.1">>
      }
    }

Example Output (JSON Binary) 

    <<"{
      \"app_nfo\":{
        \"description\":\"ERTS  CXC 138 10\",
        \"name\":\"kernel\",
        \"version\":\"3.0.1\"
      }
    }">>

from_json(Json::binary()) -> #{} | [#{}]

Decode the specified JSON encoded binary into an 'Erlang term'. All 'string' entities in the specified term will remain binary encoded.

NB: All keys must be of a string type and will be converted to atoms upon conversion.

Example Input (JSON Binary) 

    <<"{
      \"app_nfo\":{
        \"description\":\"ERTS  CXC 138 10\",
        \"name\":\"kernel\",
        \"version\":\"3.0.1\"
      }
    }">>

Example Output (Map Term) 

    #{app_nfo => #{
      description => <<"ERTS  CXC 138 10">>,
      name => <<"kernel">>,
      version => <<"3.0.1">>
      }
    }
