Functions

append_global_option(X1::ao_option()) -> ok | efail

Append a key-value pair to the internal list of global options obeyed by libao itself.

open_live(DriverId::integer(), SampleFormat::ao_sample_format(), Options::[ao_option()]) -> ao_device() | ao_error()

Open a live playback audio device for output.

open_file(DriverId::integer(), Filename::string(), Overwrite::boolean(), SampleFormat::ao_sample_format(), Options::[ao_option()]) -> ao_device() | ao_error()

Open a file for audio output. The file format is determined by the audio driver used.

play(Device::ao_device(), OutputSamples::binary()) -> ok | efail

Play a block of audio data to an open device. Samples are interleaved by channels (Time 1, Channel 1; Time 1, Channel 2; Time 2, Channel 1; etc.) in the memory buffer.

close(Device::ao_device()) -> ok | efail

Closes the audio device and frees the memory allocated by the device structure.

driver_id(ShortName::string()) -> integer() | efail

Looks up the ID number for a driver based upon its short name. The ID number is need to open the driver or get info on it.

default_driver_id() -> integer() | efail

Returns the ID number of the default live output driver. If the configuration files specify a default driver, its ID is returned, otherwise the library tries to pick a live output driver that will work on the host platform.

driver_info(DriverId::integer()) -> ao_info() | efail

Get information about a particular driver.

driver_info_list() -> [ao_info()]

Get a list of ao_info() records for every available driver.

file_extension(DriverId::integer()) -> string() | efail

Returns the normal file extension associated with a particular driver (like "wav" or "au"). This is just an information function to allow library users to guess appropriate file names. You can safely ignore the recommended extension.

Warning, this is broken at the moment and will always return an error.

is_big_endian() -> boolean()

Test if this computer uses big-endian byte ordering.

start(X1::normal, X2::any()) -> ok

Initialize the port and the libao library. The system and user configuration files are read at this time. You must call start/0 before using any other function from this module. Do not call it more than once though.

stop(X1::any()) -> ok

Unloads the libao library and closes the port. You must close all previously opened devices before calling stop/0.
