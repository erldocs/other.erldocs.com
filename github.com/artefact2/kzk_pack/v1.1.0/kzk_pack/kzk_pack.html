kzk_pack

This module provides the primitives needed to create and maintain KZK pack files.

This module provides the primitives needed to create and maintain KZK pack files. The full format is specified in the README.format file.

Functions

open(Filename::string(), Readonly::boolean()) -> {ok, pack()}

Open an already existing pack from the given Filename. The append_file/3 and append_file_raw/3 will only work as expected if the Readonly parameter is false.

dispose(Pack::pack()) -> ok

Close a pack opened with open/2 or created with create/1. This function does not automatically call commit/1, be sure to always commit dirty packs or you will end up with an inconsistent file!

create(Filename::string()) -> {ok, pack()}

Create a new, empty pack in file Filename. You must have write access to the file, and the file must not exist. Calling this method will create the file immediately.

commit(Pack::pack()) -> {ok, pack()}

Rewrite the new header and table of contents of the pack. You must absolutely use it after appending files!

append_file(Pack::pack(), Filename::string(), Data::binary()) -> {ok, pack()} | {error, name_clash}

Append a file in the pack from existing data. If a file with the same filename is already in the pack, an error will be returned if the contents differ.

append_file_raw(Pack::pack(), ArchiveFilename::string(), RealFilename::string()) -> {ok, pack()} | {error, name_clash}

Append a file in the pack from a file on the disk. This is obviously more efficient than append_file/3 for large files.

has_file(Pack::pack(), Filename::string()) -> boolean()

Checks whether a given filename is stored in the pack.

get_file(Pack::pack(), Filename::string()) -> {ok, binary()} | {error, file_not_found} | {error, contents_not_found}

Get the contents of a file stored in the pack. Use get_file/4 instead when dealing with large files.

get_file(Pack::pack(), Filename::string(), Offset::integer(), Length::integer()) -> {ok, binary()} | {ok, eof} | {error, file_not_found} | {error, contents_not_found}

Get some contents of a file stored in the pack.

list(Pack::pack()) -> {ok, [{binary(), integer(), string()}]}

Get a list of all the files stored in the pack, with their size and SHA-1 checksum.

integrity_check(Pack::pack()) -> ok | {error, corrupted_toc} | {error, {corrupted_files, [string()]}}

Check the integrity of the pack, by checking the SHA-1 hashes of every stored file and of the table of contents. This is, as expected, slow for large packs but the process will always use a constant amount of memory.
