constrictor

Mini server that ensures that no more than N processes concurrently access a resource.

Mini server that ensures that no more than N processes concurrently access a resource.

Functions

start_link(Name::atom(), Slots::pos_integer()) -> {ok, pid()}

Start the constrictor server. It takes one parameter which is the number of available slots.

stop(Pid) -> term()

Termiantes the server

execute(Name::pid(), Job::() -> any()) -> any()

Takes a function to execute, and executes it when sufficient resources are available. Jobs are executed in first come first serve basis.
