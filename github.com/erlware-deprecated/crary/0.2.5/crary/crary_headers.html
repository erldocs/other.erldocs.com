crary_headers

This module acts an an associative data-structure to hold headers.

This module acts an an associative data-structure to hold headers. This can be used for both request as well as response headers.

This module is fairly flexible with inputs. Stringish inputs can always be given in the form of iolists() or atoms(). Headerish inputs can always be given as headers(), tuple_list(), dict(), gb_trees(), or as a socket to read/parse them from. However, outputs will always be header() or string(). In the case of names, since HTTP header names are case insensitive they will always be returned lower cased.
