Functions

add_pool(PoolName, PoolArgs, MysqlArgs) -> term()

Adds a pool to the started mysql_poolboy application.

checkin(PoolName, Connection) -> term()

Returns a mysql connection to the given pool.

checkout(PoolName) -> term()

Checks out a mysql connection from a given pool.

child_spec(PoolName, PoolArgs, MysqlArgs) -> term()

Creates a supvervisor:child_spec. When the need to supervise the pools in another way.

execute(PoolName, StatementRef, Params) -> term()

Execute a mysql prepared statement with given params.

execute(PoolName, StatementRef, Params, Timeout) -> term()

Execute a mysql prepared statement with given params and timeout

query(PoolName, Query) -> term()

Executes a query to a mysql connection in a given pool.

query(PoolName, Query, ParamsOrTimeout) -> term()

Executes a query to a mysql connection in a given pool with either list of query parameters or a timeout value.

query(PoolName, Query, Params, Timeout) -> term()

Executes a query to a mysql connection in a given pool with both a list of query parameters and a timeout value.

with(PoolName, Fun) -> term()

Wrapper to poolboy:transaction/2. Since it is not a mysql transaction. Example instead of: Conn = mysql_poolboy:checkout(mypool), try mysql:query(Conn, "SELECT...") after mysql_poolboy:checkin(mypool, Conn) end.

mysql_poolboy:with(mypool, fun (Conn) -> mysql:query(Conn, "SELECT...") end).

transaction(PoolName, TransactionFun) -> term()

Executes a mysql transaction fun. The fun needs to take one argument which is the mysql connection.

transaction(PoolName, TransactionFun, Args) -> term()

Executes a transaction fun. Args list needs be the same length as TransactionFun arity - 1.

transaction(PoolName, TransactionFun, Args, Retries) -> term()

Same as transaction/3 but with the number of retries the mysql transaction should try to execute.
