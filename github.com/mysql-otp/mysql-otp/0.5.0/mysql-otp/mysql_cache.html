A minimalistic time triggered dict based cache data structure.

A minimalistic time triggered dict based cache data structure.

The cache keeps track of when each key was last used. Elements are evicted using manual calls to evict_older_than/2. Most of the functions return a new updated cache object which should be used in subsequent calls.

A cache can be initialized to 'empty' which represents the empty cache.

Types

cache(K, V) = {cache, timestamp() (see module erlang), dict(K, {V, non_neg_integer()}) (see module dict)} | empty

Functions

evict_older_than(Cache::cache(K, V), MaxAge::non_neg_integer()) -> {Evicted::[{K, V}], NewCache::cache(K, V)}

Deletes the entries that have not been used for MaxAge milliseconds and returns them along with the new state.

lookup(Key::K, Cache::cache(K, V)) -> {found, Value::V, UpdatedCache::cache(K, V)} | not_found

Looks up a key in a cache. If found, returns the value and a new cache with the 'last used' timestamp updated for the key.

new() -> cache(K::term(), V::term())

Returns the atom empty which represents an empty cache.

size(X1::cache(K::term(), V::term())) -> non_neg_integer()

Returns the number of elements in the cache.

store(Key::K, Value::V, Cache::cache(K, V)) -> cache(K, V)

  • K = term()
  • V = term()

Stores a key-value pair in the cache. If the key already exists, the associated value is replaced by Value.
