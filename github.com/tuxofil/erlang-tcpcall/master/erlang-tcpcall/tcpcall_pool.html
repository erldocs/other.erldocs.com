tcpcall_pool

Client connection pool.

Client connection pool. Provides a load balancer feature to the tcpcall library.

Functions

start_link(PoolName::client_pool_name() (see module tcpcall), Options::client_pool_options() (see module tcpcall)) -> {ok, Pid::pid()} | ignore | {error, Reason::any()}

Start connection pool process. The process can be started as part of the supervision tree.

call(PoolName::client_pool_name() (see module tcpcall), Request::data() (see module tcpcall), Timeout::pos_integer()) -> {ok, Reply::any()} | {error, Reason::any()}

Make a Request-Reply interaction with a remote server through the started connection pool. See tcpcall:connect_pool/2 function description for more details.

stop(PoolName::client_pool_name() (see module tcpcall)) -> ok

Tell the pool process to stop.

workers(PoolName::client_pool_name() (see module tcpcall)) -> [pid()]

Return process ID list of all active workers.
