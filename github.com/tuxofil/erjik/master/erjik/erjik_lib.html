challenge(Fun::(A::any()) -> B::any(), List::[A::any()]) -> {ok, B::any()} | undefined

Like lists:map/2, but:

spawns each fun in separate process; returns only result of first successfully finished fun (rest of processes spawned will be killed).

If supplied list is empty or no functions finished normally, it will return 'undefined' atom. Example:

challenge(fun(T) -> timer:sleep(T), T end, [2000, 1000, 3000]) will return {ok, 1000};

challenge(fun(_) -> exit(oops) end, [1, 2, 3, 4]) will return 'undefined'.

pmap(Fun::(A::any()) -> B::any(), List1::[A::any()]) -> List2::[B::any()]
