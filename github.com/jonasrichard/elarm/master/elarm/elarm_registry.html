subscribe(Subscriber::pid()) -> {ok, [{atom(), pid()}]}

Subscribe to listening elarm server events, e.g. starting new elarm server, stop elarm server, elarm server crashed. The subscriber process - identified by self() - will get messages when an elarm event happens.

{elarm_started, Name}

when an elarm server is started
{elarm_down, Name}

when an elarm server is stopped or crashed

unsubscribe(Subscriber::pid()) -> ok

Unsubscribe from elarm events

which_servers() -> [{atom(), pid()}]

Get all registered elarm server processes

server_started(Name::atom()) -> term()

An elarm server has to call this when it is started in order that registry listeners knows about that elarm server.

server_stopped(Name::atom()) -> term()

Elarm server may call this when it terminates. Registry will send a message to its subscribers about that event. Elarm server is monitored by the registry, so calling this function is optional.
