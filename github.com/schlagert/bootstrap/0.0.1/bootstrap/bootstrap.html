bootstrap

Main module of the bootstrap application.

Main module of the bootstrap application.

This module contains the API for bootstrap handler registration as well as the bootstrap behaviour definition that must be implemented by handlers.

Furthermore this module implements the application callback as well as the top level supervisor of the bootstrap application.

Types

arg() = term()

Functions

add_handler(Module::module(), Arg::arg()) -> ok | {error, term()}

Adds a new bootstrap handler module. The second argument is the initial handler state passed into the behaviour callbacks. A specific module can only be added once per time. If the same module was already added before it must be deleted in advance. There's no limit on the number of different added handler modules. Callback processing is sequential and in order of addition. To remove a bootstrap handler delete_handler/1 must be called.

add_sup_handler(Module::module(), Arg::arg()) -> ok | {error, term()}

This function works much like add_handler/2 but additionally the calling process will be associated with the bootstrap handler. This means that the calling process will be monitored and the corresponding handler will be removed as soon as the associated process exits. Alternatively the handler can be removed manually using delete_handler/1.

delete_handler(Module::module()) -> ok

Deletes a bootstrap handler module. Calling this function with a module that was not added before is not considered an error.

handlers() -> [module()]

Return a list of all registered bootstrap handlers.
