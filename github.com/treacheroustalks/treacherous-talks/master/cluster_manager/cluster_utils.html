parallel_startup_order(ClustConf::clustconf()) -> [start_order()]

Generate a parallel startup order from a given clustconf() which simply puts any riak release before any backend, and any backend before anything else. Releases that can be executed in parallel will be in a list

For now, two or more of the same release types will be returned in the order they occurred in the clustconf() but this should not be relied upon.

Contacts all Backends and calls backends:change_state/2 so that the watch-ring stays intact.
