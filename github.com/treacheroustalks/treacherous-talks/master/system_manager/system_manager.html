system_manager

The system manager application handles configuration of the Treacherous Talks system.

The system manager application handles configuration of the Treacherous Talks system. It is intended to be the single point of entry for configuring and controlling the whole system on a server.

Types

hostconf() = {host, hostname(), sys_mgr_name(), [relconf()]}

hostname() = string()

node_prefix() = atom()

relconf() = {release, relname(), node_prefix(), [term()]}

relname() = atom()

sys_mgr_name() = string()

Functions

update_config(HostConfig::hostconf()) -> ok | {error, term()}

Update all the config files needed to fulfill the given host configuration tuple.

start_release(Relname::relname()) -> ok | {error, term()}

Starts a given release and returns as soon as it is possible to ping the VM. Please notice that this does not ensure that any applications has started correctly.

stop_release(Relname::relname()) -> ok | {error, term()}

Stops a given release and returns only when the release has been truly stopped (VM is shutdown).

ping_release(Relname::relname()) -> up | down | {error, term()}

Pings a given release and returns the result.

join_riak(Node::hostname()) -> ok | {error, term()}

Adds the local Riak node to a cluster by sending a join request to the given Node.
