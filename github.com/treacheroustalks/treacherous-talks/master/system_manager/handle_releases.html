handle_releases

This module handles Erlang releases, that is starting, stopping and pinging them through low-level means.

This module handles Erlang releases, that is starting, stopping and pinging them through low-level means. It basically executes the control scripts directly. It is assumed that the release_path environment variable is set.

Types

hostname() = string()

relname() = atom()

Functions

start_release(Path::string(), Relname::relname()) -> {ok, binary()} | {error, term()}

Starts a release given the name of it and the path to the release directory. It will also return the stdout of the command as a binary.

stop_release(Path::string(), Relname::relname()) -> {ok, binary()} | {error, term()}

Stops a release given the name of it and the path to the release directory. It will also return the stdout of the command as a binary.

ping_release(Path::string(), Relname::relname()) -> up | down | {error, term()}

Pings a release given the name of it and the path to the release directory. It returns if it is up or down.

ping_release_and_wait(Path::string(), Relname::relname(), NbrOfTimes::integer()) -> up | down | {error, term()}

Pings a given release and checks for an "up" reply. If the reply is "down", it will sleep one second and ping again. This is repeated either until an "up" reply is received or when it reaches the NbrOfTimes limit. The Path argument is the path to the release directory.

run_riak_search_command(Args::list()) -> {ok, binary()} | {error, term()}

Runs the Riak search-cmd file with arguments according to Args.

run_riak_join_command(Node::hostname()) -> {ok, binary()} | {error, term()}

Joins a Riak node to a Riak cluster by sending a join request to Node.
