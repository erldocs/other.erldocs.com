diplomacy_rules

The rules are implemented here, the rule engine merely executes them.

The rules are implemented here, the rule engine merely executes them.

Possible Orders: diplomacy_rules can process a certain syntax of rules. They are documented as type specs in this file, go to the "Data Types" section below.

rules:process/4 will return replies that are implemented in this module. depending on the game phase, we have different possibilities. Most important are:

order_phase {dislodge, unit(), province ()} the unit needs to be moved in retreat phase or it will be disbanded
count_phase {has_builds, nation(), integer()} the nation is allowed to build / has to disband that many units
count_phase game_over} the game is finished

look for return values with {reply, Something} if you wonder why stuff is in rules:process/4's return list.

Functions

create(GameType, GamePhase) -> [tuple()]

  • GameType = standard_game
  • GamePhase = order_phase | retreat_phase | count_phase | build_phase

creates the rules for a game type. The first parameter decides the game type, the second one the game phase currently, the only supported game type is the atom 

standard_game

through clauses that differ in the game phase parameter, you can use a different set of rules for each game phase.
