backends

provides some convenience functions related to the set of backends we have in the system.

provides some convenience functions related to the set of backends we have in the system

Functions

watch_neighbour() -> undefined | node()

necromancer:watches the neighbour this node is responsible for. assumes that necromancer is running

get_responding_backend() -> node() | undefined

Go through the backend_nodes-list in the app file and return the one that is answers to a ping the quickest.

get_remote_responding_backend() -> node() | undefined

like get_responding_backend/0 but guarantees that _this_ node is not returned. Obviously it doesn't make a difference if you are using this from a frontend ;-)

change_state(OtherKey::atom(), NewList::any()) -> ok

can be called when the backend_nodes value changed. if the left neighbour is not the same after the change, the watch-connection will be exchanged to the new one

get_all_backends() -> [node()] | []

returns all known backends in the system
