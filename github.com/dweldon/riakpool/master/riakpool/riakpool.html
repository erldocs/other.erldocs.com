riakpool

riakpool implements a pool of riak protocol buffer clients.

riakpool implements a pool of riak protocol buffer clients. In order to use a connection, a call to execute/1 must be made. This will check out a connection from the pool, use it, and check it back in. This ensures that a given connection can only be in use by one external process at a time. If no existing connections are found, a new one will be established. Note this means the pool will always be the size of the last peak need. The number of connections can be checked with count/0.

Prior to any calls to execute/1, the pool must be started. This can be accomplished in one of two ways:

1. Before the server is started, set the riakpool application environment variables riakpool_host and riakpool_port.

2. After the server is started, call start_pool/0 or start_pool/2

Types

host() = string() | atom()

Functions

count() -> integer()

Returns the number of connections as seen by the supervisor.

execute(Fun::(pid()) -> any()) -> {ok, Value::any()} | {error, any()}

Finds the next available connection pid from the pool and calls Fun(Pid). Returns {ok, Value} if the call was successful, and {error, any()} otherwise. If no connection could be found, a new connection will be established. 

  > riakpool:execute(fun(C) -> riakc_pb_socket:ping(C) end).
  {ok,pong}

start_link() -> {ok, pid()} | {error, any()}

Starts the server.

start_pool() -> ok | {error, any()}

Starts a connection pool to a server listening on {"127.0.0.1", 8087}. Note that a pool can only be started once.

start_pool(Host::host(), Port::integer()) -> ok | {error, any()}

Starts a connection pool to a server listening on {Host, Port}. Note that a pool can only be started once.

stop() -> ok

Stops the server.
