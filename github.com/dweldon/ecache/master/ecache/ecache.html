ecache

ecache is a simple, fast, in-memory hash table for storing arbitrary erlang terms on a local node.

ecache is a simple, fast, in-memory hash table for storing arbitrary erlang terms on a local node. It has a memcached-like interface, complete with expiring keys.

ecache stores its data in a private ets table. All reads and writes are serialized by the server. All data are stored as {Key, Value, ExpirationTime}. During a read, the server will guarantee that if a key has expired, it will delete the key and return {error,not_found}. During a store, if a timeout is set, an asynchronous message is sent to the purge server to track (and later purge) the given key.

Functions

start_link() -> term()

stop() -> term()

count() -> integer()

Returns the number of keys in the cache.

decrement(Key::any()) -> {ok, NewValue::integer()} | {error, not_found} | {error, not_an_integer}

Decrement the value associated with Key by 1. Returns {ok, NewValue} if the decrement succeeded. Returns {error, not_found} if the Key could not be found in the cache. Returns {error, not_an_integer} if the original value was not an integer.

delete(Key::any()) -> ok

Deletes Key from the cache.

flush() -> ok

Deletes all keys from the cache.

increment(Key::any()) -> {ok, NewValue::integer()} | {error, not_found} | {error, not_an_integer}

Increment the value associated with Key by 1. Returns {ok, NewValue} if the increment succeeded. Returns {error, not_found} if the Key could not be found in the cache. Returns {error, not_an_integer} if the original value was not an integer.

load(Key::any()) -> {ok, Value::any()} | {error, not_found}

Returns {ok, Value} if the value associated with Key could be found, and {error, not_found} otherwise.

load_all() -> {ok, [{Key::any(), Value::any()}]}

Returns a list of all {Key, Value} pairs in the cache.

store(Key::any(), Value::any()) -> ok

Stores a Key - Value pair in the cache. If Key already exists in the cache, the associated value is replaced by Value.

store(Key::any(), Value::any(), Timeout::integer()) -> ok

Stores a Key - Value pair in the cache for Timeout seconds. If Key already exists in the cache, the associated value is replaced by Value. After Timeout seconds have elapsed, the key will be removed from the cache. Subsequent stores to the same key will always use the most recent timeout value (if one was given). Storing with a timeout of 0 is equivalent to storing without a timeout.
