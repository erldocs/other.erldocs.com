gmt_event_h

The GMT event handler module.

The GMT event handler module.

This module is responsible for taking care of general event notifications, including info & error logging.

The attempts to convert SASL and other system messages to PSS-style module names, severity level, etc. are not perfect. They may need to be better, but they probably do not need to be perfect.

Functions

add_report_handler(Type::pss | lss | atom() | proplist()) -> term()

Add a gmt_event_h-style event handler to the kernel's error_logger gen_event event handler.

This function is usually called by an application callback or a supervisor callback. In the rare case of a bug in this module's event or call handling, a {gen_event_EXIT, X, Y} message will be sent to that module. Unfortunately, application & supervisor processes will simply discard the message. Therefore, if an application wishes to do something to react to a bug in this module, it should use the add_terminate_fun/1 function or add_exception_fun/1 after calling add_report_handler/1.

For backward compatibility, the argument may be a single atom, which will control only which "central.conf" config item name is used for the application log file:

pss -< pss_app_log_path

lss -< lss_app_log_path

any other atom -< application_app_log_path

any other atom -< application_app_log_log_fmt

any other atom -< application_app_log_date_fmt

any other atom -< application_app_log_field_sep

any other atom -< application_app_log_custom_token

If a proplist-style properties list is specified, the following properties are honored:

{log_type, X} where X may the atom pss, lss, or other (see backward compatibility above).

{log_fmt, X} where X may be the atom default | cstm1.

default: OS pid, date, module, severity, code7, event string

cstm1: date, OS pid, module, severity, code7, event string

{date_fmt, X} where X may be the atom default | cstm1.

default: YYYYMMDDHHMMSS

cstm1: YYYY/MM/DD HH:MM:SS:000

 (000 is a fixed constant)

{field_sep, N} where N is the ASCII character used to separate fields of an application log entry.
