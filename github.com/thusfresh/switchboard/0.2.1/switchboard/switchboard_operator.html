switchboard_operator

An operator process is paired with a single IMAP idler for each mailbox to be monitored.

An operator process is paired with a single IMAP idler for each mailbox to be monitored. The idler sends its low-level IMAP idle messages to the operator, which checks whether a new email has arrived, and, if so, fetches and publishes a notification to the new channel.

Functions

start_link(Account::account() (see module imap), Mailbox::mailbox() (see module imap)) -> startlink_ret() (see module supervisor)

Start the operator as part of the supervision tree.

stop(Oper::pid()) -> ok

Stop the operator.

update_uid(Oper::pid()) -> ok

Commands for the operator to update its state using the latest email uid in the operator's mailbox.

get_last_uid(Oper::pid()) -> integer()

Returns the operator's last known UID.
