etorrent_peer_send

Handle outgoing messages to a peer.

Handle outgoing messages to a peer

This module handles all outgoing messaging for a peer. It supports various API calls to facilitate this

Note that this module has two modes, fast and slow. The fast mode outsources the packet encoding to the C-layer, whereas the slow mode doesn't. The price to pay is that we have a worse granularity on the rate calculations, so we only shift into the fast gear when we have a certain amount of traffic going on.

The shift fast to slow, or slow to fast, is synchronized with the process etorrent_peer_recv, so if altering the code, beware of that

check_choke(Pid::pid()) -> ok

Ask the process to check if a rechoke is necessary.

This call is used whenever we want to check the choke state of the peer. If it is true, we perform a rechoke request. It is probably the wrong place to issue the rechoke request. Rather, it would be better if a control process does this.

go_fast(Pid::pid()) -> ok

Request that we enable fast messaging mode.

In this mode, message encoding is done by the underlying Erlang VM and not by us.

Note: The only intended caller of this function is etorrent_peer_recv, when it wants synchronization on the gear cange.

go_slow(Pid::pid()) -> ok

Request that we enable slow messaging mode.

In this mode, message encoding is done by the code in this module, not by the VM kernel

Note: The only intended caller of this function is etorrent_peer_recv, when it wants synchronization on the gear cange.
