Handle communication with trackers

This module handles all communication with a tracker. It will periodically announce to the tracker for an update. Eventual errors and new Peers are fed back into the Peer Manager process so they can be processed to completion.

For UDP tracking, we delegate the work to the UDP tracker system.

Events If specified, must be one of started, completed, stopped, (or empty which is the same as not being specified). If not specified, then this request is one performed at regular intervals.

- started: The first request to the tracker must include the event key with this value. - stopped: Must be sent to the tracker if the client is shutting down gracefully. - completed: Must be sent to the tracker when the download completes. However, must not be sent if the download was already 100% complete when the client started. Presumably, this is to allow the tracker to increment the "completed downloads" metric based solely on this event.

- paused: From BEP-21

In order to tell the tracker that a peer is a partial seed, it MUST send an event=paused parameter in every announce while it is a partial seed.

Implementing event=paused is not save: https://forum.transmissionbt.com/viewtopic.php?f=4&t=7117
