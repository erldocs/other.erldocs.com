cb_config_couch_sync

This server syncs part of (replicated) ns_config to local couch config.

This server syncs part of (replicated) ns_config to local couch config. Actual sync is performed by worker process that's direct son of supervisor. ns_config changes are noted by gen_server process that's son of worker process. And it's using ns_pubsub to tap into ns_config changes.

Such seemingly unusual design (with worker being parent of main process) is to ensure that supervisor shutdown synchronizes with worker death (which mutates world state). Main process does not mutate world's state, so worker can exit without waiting for main process death (it'll die soon but we don't care when).
