local_client(ClientId::binary() | undefined) -> {ok, Client::riak_client()}

When you want a client for use on a running Riak node. ClientId should be a 32-bit binary. If it is not, a 32-bit binary will be created from ClientId by phash2/1. If ClientId is the atom 'undefined', a random ClientId will be chosen.

client_connect(Node::node()) -> {ok, Client::riak_client()} | {error, timeout}

Equivalent to client_connect(Node, undefined).

client_connect(Node::node(), ClientId::binary() | undefined) -> {ok, Client::riak_client} | {error, timeout}

The usual way to get a client. Timeout often means either a bad cookie or a poorly-connected distributed erlang network. ClientId should be a 32-bit binary. If it is not, a 32-bit binary will be created from ClientId by phash2/1. If ClientId is the atom 'undefined', a random ClientId will be chosen.
