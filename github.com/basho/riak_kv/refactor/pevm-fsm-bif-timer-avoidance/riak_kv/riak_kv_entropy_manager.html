Acquire the necessary locks for an entropy exchange with the specified remote vnode. The request is sent to the remote entropy manager which will try to acquire a concurrency lock. If successsful, the request is then forwarded to the relevant index_hashtree to acquire a tree lock. If both locks are acquired, the pid of the remote index_hashtree is returned.

enabled() -> boolean()

set_mode(Mode::automatic | manual) -> ok

Set AAE to either automatic or manual mode. In automatic mode, the entropy manager triggers all necessary hashtree exchanges. In manual mode, exchanges must be triggered using manual_exchange/1. Regardless of exchange mode, the entropy manager will always ensure local hashtrees are built and rebuilt as necessary.

manual_exchange(Exchange::index() | {index(), index_n()} | {index(), index(), index_n()}) -> ok

Manually trigger hashtree exchanges. -- If an index is provided, trigger exchanges between the index and all sibling indices for all index_n. -- If both an index and index_n are provided, trigger exchanges between the index and all sibling indices associated with the specified index_n. -- If an index, remote index, and index_n are provided, trigger an exchange between the index and remote index for the specified index_n.
