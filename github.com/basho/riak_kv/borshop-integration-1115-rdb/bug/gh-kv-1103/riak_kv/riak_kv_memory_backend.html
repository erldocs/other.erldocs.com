riak_kv_memory_backend

riak_kv_memory_backend is a Riak storage backend that uses ets tables to store all data in memory.

riak_kv_memory_backend is a Riak storage backend that uses ets tables to store all data in memory.

Configuration Options

The following configuration options are available for the memory backend. The options should be specified in the memory_backend section of your app.config file.

ttl - The time in seconds that an object should live before being expired.

max_memory - The amount of memory in megabytes to limit the backend to.

test - When true, allow public access to ETS tables so they can be cleared efficiently.
