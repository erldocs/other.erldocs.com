riak_kv_wm_apiep

Webmachine resource providing a location service to external clients for optimal access to hosts with partitions (all, or a subset containing certain bucket/key).

Webmachine resource providing a location service to external clients for optimal access to hosts with partitions (all, or a subset containing certain bucket/key).

   GET /ring/coverage/bucket/BUCKET/key/KEY?proto=PROTO&force_update=F
   GET /ring/coverage?proto=PROTO&force_update=F

Return a list of dicts with keys (address, port, last_modified), describing http or pbc (as specified in parameter PROTO, defaulting to pbc) API entry points at riak_kv nodes containing the data (first form) or all entry points (second form), as JSON object like this:

[{"addr":Addr, "port":Port, "last_checked":Unixtime}]

If parameter force_update is specified and has a value of "1", this information is collected via (expensive) rpc calls to all riak_kv nodes; else, values cached from a previous call are returned.

By default, returned value is cached until the next multiple of 15 seconds (configurable via ring_coverage_cache_expiry_time init parameter).
