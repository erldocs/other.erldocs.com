cowboy_metrics

Application Entry Point.

Application Entry Point. Allow cowboy web servers to start gathering SNMP metrics. This module provide mechanism to either generate appropriate cowboy:onrequest_fun/1 and cowboy:onrequest_fun/4.

Functions

start_http(Ref::ref() (see module ranch), Acceptors::non_neg_integer(), TransOpts::opts() (see module ranch_tcp), PorotoOpts::opts() (see module cowboy_protocol)) -> {ok, pid()} | {error, any()}

Start a cowboy http listener. This is an identical function to cowboy:start_http/4. It is also is an equivalent to start_http(1, Ref, Acceptors, TransOpts, ProtoOpts). It starts the metrics with a service of index equals to 1.

See also: //cowboy/cowboy:start_http/4, //cowboy_metrics/cowboy_metrics:start_http/5.

A helper function that will create a cowboy server with some SNMP service information. It will also set the appropriate onrequest and onresponse cowboy hooks. This function will detect existing cowboy hooks and will compose them with the SNMP hooks.

create_server(Cm_svc::#cm_svc{index=non_neg_integer(), description=string(), contact=string(), name=string(), port=undefined | pos_integer(), type=other | server | client | proxy | caching_proxy}) -> {ok, onrequest_fun() (see module cowboy), onresponse_fun() (see module cowboy)} | {error, term()}

Create a service entry in the SNMP service information table. This function returns the appropriate cowboy:onrequest_fun/1 and cowboy:onresponse_fun/4.
