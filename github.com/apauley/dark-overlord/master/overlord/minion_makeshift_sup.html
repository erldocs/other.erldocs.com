Why write a makeshift supervisor instead of using a real one? One reason is to learn more about supervisors by writing something that resembles one. Another reason might be that I had trouble spawning a real supervisor on a remote node when the remote node only had access to the code I loaded remotely onto it. I can probably get past those issues with a bit of effort, but for now a makeshift supervisor will do nicely.
