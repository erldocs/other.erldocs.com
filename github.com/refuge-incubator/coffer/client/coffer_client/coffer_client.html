coffer_client

Functions

open(Url) -> term()

open a connection to a storage, the url given is the Url of a storage. A client is dedicated to a storage and has its own pool.

open(Url, Opts) -> term()

same as above but with connections options. See the hackney options for it. A special pool_opts property has been added to create the client pool and pass needed options to it.

close(Client) -> term()

close a connection to a storage and stop the pool do nothing for now

is_exists(Client_ctx, BlobRef) -> term()

test if a blob is already available on the storage

fetch(Client_ctx, BlobRef) -> term()

fetch a blob from the storage, return a {ReaderFun, State} tuple where the function return {ok, Chunk, NewState} when there is still some data to fetch, eob at the end of the binary or {error, Error} when an error happen

upload(Client_ctx) -> term()

upload a blob to the storage, refurn a writer fun

enumerate(Ctx) -> term()

enumerate blobs on the remote. return a {ReaderFun, State} tuple

enumerate(Client_ctx, Params) -> term()

reader_fun(Client) -> term()

upload_fun(Client, Bin) -> term()

enumerate_fun(Q) -> term()

start() -> term()

Start the coffer_client application. Useful when testing using the shell.

stop() -> term()

Stop the coffer_client application. Useful when testing using the shell.
