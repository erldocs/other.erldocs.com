e_file

Module responsible for manipulating the uploaded files.

Module responsible for manipulating the uploaded files.

Functions

save(Name::string(), Content::binary() | string()) -> string()

Saves the Content into the specified path inside the upload dir. Returns the path to the saved file (prefixed with upload directory).

See also: e_conf:upload_dir/0.

save(Name::string(), Content::binary() | string(), Path::string()) -> string()

Saves the Content into the specified file (prefixed with Path).

See also: save/2.

copy(Source::string(), Destination::string()) -> DestinationPath::string()

Copies the file from Source to the Destination in upload directory. Returns the path to the copied file.

See also: e_conf:upload_dir/0.

get_relative_name(Filename::string()) -> RelativeFilename::string()

Returns the relative from docroot part of the filename. It is useful when the the database holds the absolute paths to the files (webservers expects the relative from the docroot site).
E.g. when the absolute filename is "/home/erlangweb/docroot/upload/image.png" it will return "/upload/image.png".

get_size(Filename::string()) -> Size::integer()

Returnes the size of the file in bytes. In case of error the function shall return -1.
