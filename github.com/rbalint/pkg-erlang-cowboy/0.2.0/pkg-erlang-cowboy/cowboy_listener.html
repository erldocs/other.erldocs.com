add_connection(ServerPid::pid(), Pool::atom(), ConnPid::pid()) -> {ok, non_neg_integer()}

Add a connection to the given pool in the listener.

Pools of connections are used to restrict the maximum number of connections depending on their type. By default, Cowboy add all connections to the pool default. It also checks for the maximum number of connections in that pool before accepting again.

When a process managing a connection dies, the process is removed from the pool. If the socket has been sent to another process, it is up to the protocol code to inform the listener of the new ConnPid by removing the previous and adding the new one.

wait(ServerPid::pid(), Pool::atom(), MaxConns::non_neg_integer()) -> ok

Wait until the number of connections in the given pool gets below the given threshold.

This function will not return until the number of connections in the pool gets below MaxConns. It makes use of gen_server:reply/2 to make the process wait for a reply indefinitely.
