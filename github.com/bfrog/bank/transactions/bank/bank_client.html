Behaviour for Bank SQL Client.

State must be returned and kept up to date by the caller. Doing otherwise is a programming error that leads to undefined results.

If a call returns {fatal, error_result_type(), string(), state()} it means the client has fatally errored out and closed the socket. Any further action should simply return {error, connection, string()}.

connect/1 connects to the SQL server.

close/1 closes the connection.

ping/3 determines if the connection to the SQL server is ok.

prepare/4 creates a prepare sql statement.

unprepare/2 removes a prepared sql statement.

execute/4 executes a prepared sql statement.

query/4 executes a sql query with or without parameters safely.

begin_transaction/1 start a transaction.

commit_transaction/1 commit a transaction.

rollback_transaction/1 rollback a transaction.
