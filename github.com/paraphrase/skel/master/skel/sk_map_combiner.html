sk_map_combiner

This module contains the Map skeleton combiner logic.

This module contains the Map skeleton combiner logic.

The Map skeleton is a parallel map. The skeleton applies a given function to the elements within one or more lists.

The combiner receives each partite element of each input following the original element's application to the given workflow. This module collects each resulting partite element of all inputs and restores the structure of the original inputs.

Similar to sk_map_partitioner, this module supports both the automatic creation of workers, and the ability to define the number used.

Functions

start(NextPid::pid()) -> eos

Initialises the recomposition process for when the number of workers is not set by the developer.

Recomposition consists of rebuilding a list from its elements, in the correct order. For each set of elements, a data_message() is produced and sent to the process given by NextPid.
