use(ProviderName::atom()) -> {ok, Pid::pid()}

Obtains configuration provider process with default settings. Reads configuration terms from "conf/ProviderName.conf". Subscribes caller to configuration reload notification.

use(ProviderName::atom(), Opts::opts()) -> {ok, Pid::pid()}

Obtains configuration provider process

See also: opts().

reload(ProviderName::atom()) -> ok

Reloads given provider's configuration from disk. Notifies subscribers with {config_reloaded, Conf} message on success. This function is not intented to be called directly. Confetti uses it via management console commands.

fetch(ProviderName::atom()) -> Conf::term()

Fetch configuration terms from given provider process. User is responsible for implementing detailed config value getters.

start_link(ProviderName, Opts) -> term()

Starts the server, and pg2 group if needed. This function is called by confetti_sup module and there is probably no need to call it directly.

init(Subject) -> term()

Tries to load configuration for given provider. If disk configuration loading fails (i.e. config terms are broken, or configuration file does not exists), init/1 will try to fetch the last working configuration from DETS.
