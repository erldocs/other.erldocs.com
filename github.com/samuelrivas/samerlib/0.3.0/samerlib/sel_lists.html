Functions to handle lists

Compared to the official lists module, this one aspires to have a cleaner output type definition, avoiding tagged values by using exceptions to handle errors

Functions

keysearch(Key::any(), L::[tuple()]) -> any()

Return the first tuple in the list with Key as first element

cut_and_zip(List1, List2) -> List3

  • List1 = [A]
  • List2 = [B]
  • List3 = [{A, B}]
  • A = term()
  • B = term()

Like lists:zip/2, but works with lists of different lengths

When used with lists of different lengths, the last elements of the longest list are discarded to make both lists of the same length
