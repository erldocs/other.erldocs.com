test_in_dir(Fun::(string()) -> any()) -> any()

Create clean directories to run tests that require actual files.

This function creates an empty directory in /tmp, runs Fun with that directory as argument, and deletes the directory and all its contents

Returns the value returned by Fun, letting any exception raised through (but still deleting the directory)

test_in_dir(Cleanup::boolean(), Fun::(string()) -> any()) -> any()

This function is deprecated: use it only for dirty local debugging, for committed code use test_in_dir/1

Same as test_in_dir/1, but cleaning up only if Cleanup is true.

This function is intended only for debugging. Ideally, code using it should never be committed to a public repository.

props_to_eunit(Module::module()) -> [any()]

Equivalent to props_to_eunit(Module, 10).

props_to_eunit(Module::module(), Timeout::non_neg_integer()) -> [any()]

Return an Eunit test that checks all properties in a module.

To get all properties tested as part of the eunit suite (e.g. when running <module>:test()) you must include something like this in your test module: 

  all_props_test_() -> sel_test:props_to_eunit(?MODULE).

Currently, only proper properties are supported.
