moka_call_handler

A server to simulate the calls to Moked functions.

A server to simulate the calls to Moked functions

Moked functions will be redirected to a call handler that will hold the specifications created with moka "replace" calls.

Exceptions occurring when calling the provided behaviour (reply_fun) are wrapped safely (so that no aliasing is possible) and sent to the client (the calling process) that will raise them again.

get_response(CallHandler::call_handler_ref(), Args::[term()]) -> term()

Gets the response for a call

Moked modules call this function instead of the original destination function.

If the call_handler fun throws any exception, the call_handler server will catch it, wrap it safely (with no possible aliasing) and send it as a result to the caller process (i.e. the one evaluating this function). This function will then raise the exception in the caller process.
