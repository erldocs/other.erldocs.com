moka

The moka main interface.

The moka main interface

This module implements the operations to create new mokas and use them to mock calls from one module to other.

Types

moka()

Functions

start(Mod::module()) -> moka()

Starts a new moka

Mod will be know as the moked module in this documentation. The created Moka will be used to modify calls from this module.

NOTE: You should never create two Mokas for the same module.

stop(Moka::moka()) -> ok

Stops an existing moka

The original module code is restored after stopping the moka.

replace(Moka::moka(), Module::module(), Function::atom(), NewBehaviour::function()) -> ok

Substitutes all calls from the moked module to Mod:Fun/N

The arity of NewBehaviour determines the arity of the substituted function.

export(Moka::moka(), Function::atom(), Arity::non_neg_integer()) -> ok

Adds Fun/Arity to the list of exported functions of the moked module

Note that this doesn't check if exporting Fun/Arity is possible, so it will succeed for any atom-integer pair. load/1 will fail if exporting Fun/Arity is not possible (e.g. if that function is undefined).

load(Moka::moka()) -> ok

Makes all substitutions effective

The moked module code will be changed after this function is called.

You can easily know when a module is moked calling Module:module_info(attributes). For moked modules, the returned list contains a moka_orig_module tuple.
