trigger

Generic trigger for (parameterized) modules.

Generic trigger for (parameterized) modules.

Can be used by a module Module in order to get a configurable message (by default {trigger}) in intervals defined by a given trigger. The basic pattern for the use of this module is as follows:

TriggerState = trigger:init(Trigger, ?MODULE), TriggerState2 = trigger:now(TriggerState)

Then on each received {trigger} message, the trigger needs to be told to issue another {trigger} message:

NewTriggerState1 = trigger:next(TriggerState), NewTriggerState2 = trigger:next(TriggerState, base_interval),

Note: When parameterized modules are used, trigger:init(Trigger, THIS) does not work. Use code like the following instead:

TriggerState = trigger:init(Trigger, fun get_base_interval/0)

Types

interval() = max_interval | base_interval | min_interval

interval_fun() = fun(() -> integer() >= 1)

state()

Functions

init(Trigger::module(), Module_or_BaseIntervalFun::module() | interval_fun()) -> state()

Initializes the given trigger with the given base interval function (also used for min and max interval). If a Module is given instead, the trigger will use the module's get_base_interval(), get_min_interval() and get_max_interval() functions for the according intervals.

init(Trigger::module(), BaseIntervalFun::interval_fun(), MsgTag::message_tag() (see module comm)) -> state()

Initializes the given trigger with the given base interval function (also used for min and max interval) and the given message tag used for the trigger message.

init(Trigger::module(), BaseIntervalFun::interval_fun(), MinIntervalFun::interval_fun(), MaxIntervalFun::interval_fun()) -> state()

Initializes the trigger with the given interval functions.

init(Trigger::module(), BaseIntervalFun::interval_fun(), MinIntervalFun::interval_fun(), MaxIntervalFun::interval_fun(), MsgTag::message_tag() (see module comm)) -> state()

Initializes the trigger with the given interval functions and the given message tag.

now(X1::state()) -> state()

Sets the trigger to send its message immediately, for example after its initialization. Any previous trigger will be canceled!

next(State::state()) -> state()

Sets the trigger to send its message after BaseIntervalFun() milliseconds. Any previous trigger will be canceled!

next(X1::state(), IntervalTag::interval()) -> state()

Sets the trigger to send its message after the given interval's number of milliseconds. Any previous trigger will be canceled!

stop(X1::state()) -> state()

Stops the trigger until next or now are called again.
