julius_cmd

Interface to Linux shell commands that communicate through stdin and stdout.

Interface to Linux shell commands that communicate through stdin and stdout.

Functions

start(Cmd::string(), Args::list(), ID::atom()) -> {ok, pid()}

Start a command process. It will terminate automatically when the external command terminates. Output will be sent by gen_server:cast to the calling process.

send(P, Data) -> term()

Send a line of text to the command subprocess.

send(P, Format, Data) -> term()

init(X1) -> term()

Initialize the command interface process, This spawns the specified subcommand and sets up a port through which we can communicate with it.

handle_call(Data, From, State) -> term()

Many simple command line programs behave synchronously, so this is how those commands are issued.

handle_cast(Data, State) -> term()

Send data to stdin of the subprocess and do not wait. The caller should be prepared for cast output at any time.

handle_info(X1, S) -> term()

terminate(Reason, State) -> term()

Process is terminating. Close the subcommand port if it is still open.
