sbroker_time

This module provides a behaviour for reading the time and utility functions to get the current time.

This module provides a behaviour for reading the time and utility functions to get the current time.

The sbroker_time behaviour has two callbacks: 

  -callback monotonic_time() :: Time :: integer().
  -callback monotonic_time(TimeUnit :: unit()) -> Time :: integer().

monotonic_time/1 should return the time in the given time unit as an integer. monotonic_time/0 should be equivalent to monotonic_time(native).

The TimeUnit is a named time unit: native, nano_seconds, micro_seconds, milli_seconds or seconds, or a pos_integer(), which is the integer increment for a second. For example 1000 is equivalent to milli_seconds and 1000000 is equivalent to micro_seconds.

Prior to 18.0, the native time units will always be micro_seconds. For releases greater than and including 18.0 see the "Time and Time Correction" chapter in the OTP documentation for more information. sbroker is multi time warp safe and designed to be used with multi time warps.

Types

unit() = native
               | nano_seconds
               | micro_seconds
               | milli_seconds
               | seconds
               | integer() >= 1

Functions

monotonic_time() -> Time

  • Time = integer()

Get the time, Time, as an integer() in the native time units.

Uses erlang:monotonic_time/0 if it is exported, otherwise falls back to sbroker_legacy:monotonic_time/0.

See also: erlang:monotonic_time/0, sbroker_legacy:monotonic_time/0.

monotonic_time(TimeUnit) -> Time

  • TimeUnit = unit()
  • Time = integer()

Get the time, Time, as an integer() in the TimeUnit time units.

Use erlang:monotonic_time/1 if it is exported, otherwise falls back to sbroker_legacy:monotonic_time/1.

See also: erlang:monotonic_time/1, sbroker_legacy:monotonic_time/1.
