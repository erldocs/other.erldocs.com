mcb_db_mnesia

Database layer for using Mnesia as distributed storage We only use mnesia for operations which require transactions and locking, that's thread counters.

Database layer for using Mnesia as distributed storage We only use mnesia for operations which require transactions and locking, that's thread counters. Other data is stored in RIAK (see macaba_db_riak)

start() -> term()

Prepare database for use

read(Type::mcb_mnesia_object(), Key::binary()) -> {ok, mcb_mnesia_record()} | {error, not_found}

write(Type::mcb_mnesia_object(), Value::mcb_mnesia_record()) -> {atomic, any()} | {error, any()}

update(Tab::atom(), Key::binary(), Fun::function()) -> {atomic, any()} | {error, any()}

Start transaction, read, do Fun(Object), write, return new value. If object was not found, Fun is called with Fun({error, not_found)) if it didn't crash, return value is written to database

delete(Tab, Key) -> term()
