sup_beagle_management

Functions

start_link() -> term()

Starts management client process.

trigger_session(Reason) -> term()

Triggers a session with management server for a given reason.

suspend_sessions() -> term()

Suspends session execution. While session execution is suspended, all incoming trigger_session requests are waiting in the process queue except for periodic_notify requests, which are dropped to prevent them from stacking.

resume_sessions() -> term()

Resumes session execution.

loop() -> term()

Management client main loop. Message {perform_session, Reason} makes the management client process initiate the session. Example reasons include connection request, periodic notify, scheduled notify etc. These are triggered externally.
