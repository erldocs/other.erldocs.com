Support library for solving problems from project-euler.com in Erlang.
This library contains conveniance functions that can be reused in-between problems. The praxis is that when a function is used by more than one problem it is placed in this file. This library is also a victim of me trying out edoc for the first time. Just in case you wondered why the level of documentation is far better than in all any other source file that I leave behind ;)

Functions

abs(Num::integer()) -> integer()

Returns the absolute value of a number.

parse_digits(Number::integer()) -> [integer()]

Parses a base-10 number into digits. The digits are returned in a 'big-endian' list.

See also: numlist_to_integer.

numlist_to_integer(List::[integer()]) -> integer()

Interprets a list as a base-10 integer number. List is interpreted in a big-endian sense, and each position in the list is interpreted as a power of 10 lower than the previous one.

See also: parse_digits.

count_digits(Number::integer()) -> integer()

Counts the number of digits in a base-10 number.%

fac(Number::integer()) -> integer()

Calculates the faculty of a number.

fib_iter(S) -> term()

Iterate thorugh all fibonacci numbers, c-style. Returns an updated state with the newest number as next.

proper_divisors(Number::integer()) -> [integer()]

Returns all the proper divisors of a number.

is_prime(Number::integer()) -> boolean()

Checks is a number is a prime or not. Number that is tested must be a true integer, e.g 123 (123.0 is not ok).

parse_int_problem_file(Filename::atom() | string()) -> [[any()]]

Parses a plain text file containing integer problem data. The data in the file must have Unix file endings '\n' and the individual tokens in the file must be space separated. The file is returned as a list of lists, one for each row in the file.

mapreduce(Mapper::(any()) -> any(), Reducer::(Result::any(), Acc::any()) -> Acc2::any(), Data::[any()], Acc0::any()) -> AccN::any()

A simple map-reduce implementation.
Mapper => Function that does the mapping. Each mapper will be spawned as a separate process. The Mapper takes a value of any type and returns a result that will eventually be passed to the reducer.
Reducer => Function that does the reduction. Will be spawned as one process. Takes a result from the mapper along with an accumulator, returns the new accumulator.
Data => The data that each mapper should work on. One mapper will be spawned for each term in this list.
Acc0 => Initial accumulator that will be passed to the reducer, along with the response from the first mapper.
