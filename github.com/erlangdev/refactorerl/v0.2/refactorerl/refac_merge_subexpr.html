Functions

merge_subexpression(File::string(), Line1::integer(), Col1::integer(), Line2::integer(), Col2::integer(), NewName::string()) -> none()

Performs the precondition checks and the and the collects all the necessary information for the refactoring. It throws various terms when there is an error, with this syntax: {atom(), term()}. If the atom is ok, the refactoring has been preformed without a problem.

Parameter description:

  File : The path of the module.
  Line1 : The beginning line number pointed in the editor.
  Col1 : The beginning column number pointed in the editor.
  Line2 : The ending line number pointed in the editor.
  Col2 : The ending column number pointed in the editor.
  NewName : The name to bind the expression to.
