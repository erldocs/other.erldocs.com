check(X1::any(), X2::atom() | {atom(), atom()}) -> boolean()

Return whether a value matches an Erlang type.

This function takes two arguments: the value, and the type it will be matched against. Currently the type must be given either as an atom Type (for a local type of arity 0) or as a {Module, Type} tuple (for an external type of arity 0). Built-in types or types of other arities aren't supported as an argument at this point. To work around this limitation you need to define your own base type that will be a super-type of other built-in or user types.

This function is actually a placeholder and will never be called, as Sheriff will replace all the sheriff:check/2 calls by a call to the generated validation function for the given type.
