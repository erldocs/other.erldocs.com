spawn_n(W, Lim) -> term()

Spawn N ordered processes responsible for evaluating an expression. Erlang may allocate a Pid which is less than the next pid spawned. Theoretically we need a model which follows the evaluation tree, but for now this is the most efficient way of handling the problem. The processes must be ordered for the cache to work. -------------------------------------------------------------------------------------
