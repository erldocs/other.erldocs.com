start(URI::string(), Options::[{atom(), any()}]) -> {error, any()} | {ok, pid()}

Start the websocket client.

This function will open a connection with the specified remote endpoint.

If the option ssl is given it will be overwritten by the scheme type of the URI.

start(Host::string(), Port::port_number() (see module inet), Resource::string(), Options::[{atom(), any()}]) -> {ok, pid()} | {error, {shutdown, any()}} | {error, any()}

Start the websocket client

This function will open a connection with the specified remote endpoint. The options to this method can be the following:

ssl

Connect to the server using ssl or not. Values: true or false
register

Register the client in the Erlang runtime. Values: true to register using the default name wsecli, false to not register and any other atom to register it with other name

If no options are given the process will not be registered and it will not try to connect using SSL.

stop() -> ok

Stop the websocket client

This is the same as calling stop/1 with the atom wsecli as a parameter.

stop(Client::client()) -> ok

Stop the websocket client

Calling this function doesn't mean that the connection will be inmediatelly closed. A closing handshake is tryed and only after that the connection is closed and the client stopped. Use the callback on_close/2 to know when the connection has ben closed and the client has stopped.

send(Data::binary() | string()) -> ok

Send data to a remote endpoint

This is the same as calling send/2 with the atom wsecli (see start/4) as first parameter

send(Client::client(), Data::binary() | string()) -> ok

Send data to a remote endpoint

This is the same as calling send/3 whit the third parameter being the type of the Data sent. The type will determined by is_binary and is_list NIFs. So if you want to send a text message which is stored in a binary value you will have to use send/3 and specify the type as text. Otherwise it will be sent as binary type

send(Client::client(), Data::binary() | string(), Type::encoding()) -> ok

Send data to a remote endpoint

Sends data using the specified client, encoding the data as the give type.

on_open(Client::client(), Callback::on_open_callback()) -> any()

Add a callback to be called when a connection is opened

This callback will be called when the connection is opened (after a successful websocket handshake) or inmediatelly if added while in the open state. Adding it in any other state will raise an error.
