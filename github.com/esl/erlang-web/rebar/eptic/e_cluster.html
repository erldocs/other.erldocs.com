e_cluster

Module responsible for managing the backend nodes of the Erlang Web cluster.

Module responsible for managing the backend nodes of the Erlang Web cluster. All the functions should be called from the backend server.

Functions

dispatcher_reload() -> ok

Updates the dispatcher entries on the all frontend nodes. In order to keep the dispatcher rules consistent all over the Erlang Web cluster it is strongly recommended to change it only on the backend (even if it is not used there) and trigger the update by calling the dispatcher_reload/0 function.

invalidate(List0::Entries) -> ok

  • Entries = [Url]
  • Url = string()

Removes the selected urls from the cache on all frontend servers. Url is a regular expression describing the URL that should be removed from cache (as they are no longer valid, e.g. the model has changed).
The regular expression must be in the PCRE format (described in the Erlang manual - http://www.erlang.org/doc/man/re.html).

invalidate_groups(Groups) -> ok

  • Groups = [Group]
  • Group = string()

Removes all the cache contents that belongs at least to one of the Group. Group must be a string which labels and categorizes the cache entry.

synchronize_docroot(Filename::string()) -> WorkerPid::pid()

Sends the file with the Filename to the all frontend servers. Since uploaded files are stored on the backend node this function must be called after successful saving to synchronize the state on all nodes.
The path to the file should be relative to the server root.

See also: e_conf:server_root/0.
