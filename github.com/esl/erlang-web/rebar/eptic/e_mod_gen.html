handle_request(URL::string()) -> HTML | list({status, Code::integer()}, HTML) | {ret_view, Ret::controller_response(), View::string()} | enoent

  • HTML = {html, Html::string()}

Dispatches the request and calls the right module/expands the template. The returning value is interpreted by the server callback module.

template(Path::string(), X2::[], Transform::atom()) -> list({status, ErrorCode}, {html, HTML::string()}) | {html, HTML::string()}

Expands the template from the given Path. The requested template is read from disk cache and properly expanded using Transform:process_xml function (by default //wpart/wpart_xs:process_xml/1).

See also: //wpart/wpart_xs:process_xml/1, e_cache:read_file/1.

error_page(ErrorCode::integer(), URL::string()) -> list({status, ErrorCode}, {html, HTML::string()})

Generates the error site for given error code. If error code is 404, then calls the error_page/3 with reason not_found. In other cases, the reason is empty.

See also: error_page/3, e_dispatcher:error_page/1.

error_page(ErrorCode::integer(), URL::string(), Reason::term()) -> list({status, ErrorCode}, {html, HTML::string()})

Generates the error site with the reason Reason for given error code. If the debug_mode flag is set to true, then the error page will always contain the detailed information about the error. Otherwise, the proper error page will be returned.

See also: e_conf:debug_mode/0, e_dispatcher:error_page/1.

bind_session(Cookie::term()) -> no_session | Cookie

Binds the session to the given Cookie. If the session kept in request dictionary is empty, then it deletes it from internal state and returns no_session. Otherwise returns the given cookie.

restore_session(Cookie::term()) -> true

Restores the session with the given Cookie. If the Cookie is empty, it sets the request dictionary session variable to empty list. Otherwise, it loads the previous state of session from the internal data storage.

template_file(Path::string()) -> FullPath::string()

Returns the full path to the requested template. The returned path is sanitized and prefixed with the template root directory.

See also: e_conf:template_root/0.
