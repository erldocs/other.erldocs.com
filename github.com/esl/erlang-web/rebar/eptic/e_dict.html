Module responsible for managing the request dictionary.

Functions

fset(Key::string(), Value::term()) -> true

Inserts the Value under the specified Key in the request dictionary.

fset(List::string(), Key::string(), Value::term()) -> true

Appends the Value into the List under the Key in the request dictionary. List is the top-level key to the proplist in the dictionary. If there is no list, the new one is created - the call is equivalent to: 

     eptic:fset(List, {Key, Value})

If there is a list and it contains the element {Key, Value} then nothing happens. Otherwise, the {Key, Value} is appended to the list and the list is then stored in the request dictionary.

finsert(List::term(), Key::term(), Value::term()) -> true

Inserts the Value into the List under the Key in the request dictionary. List is the top-level key to the container in the dictionary. There must be a proper container inside the request dictionary.
If there was a tuple with the first element equal to Key, then the tuple is replaced with {Key, Value}. Otherwise, the {Key, Value} is appended to the List

finsert(Key::string(), Value::term()) -> true

fget(Key::string()) -> Value::term() | undefined

Fetches the element kept in the request dictionary under the Key. If there is no element under Key inside the request dictionary, undefined is returned.

fget(List::string(), Key::string()) -> term() | undefined

Fetches the element kept in the List in the dictionary under the Key. List is top-level key to the container stored in the request dictionary. The values kept in the container under the Key are returned. If there is no values under Key or there is no List in the request dictionary, the undefined is returned.

fget0(Rest::[string()], Dict::tuple()) -> term()

fdelete(Key::term()) -> true

Removes the value stored in the request dictionary under the Key The value that is stored under the Key will be lost pernamently.

fdelete(List::term(), Key::term()) -> true

Removes the value stored under the Key inside the List in the request dictionary If the List after the removal process is empty, it is also removed.
