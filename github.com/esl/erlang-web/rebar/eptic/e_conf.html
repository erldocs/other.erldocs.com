reinstall() -> ok

Loads the configuration from the default path.

load_conf() -> ok

Loads the configuration from the default path. The default path is e_conf:server_root()/config/project.conf.

See also: load_conf/1.

load_conf(Filename::string()) -> ok

Loads the configuration from the given file. The previous configuration is overwritten. File given as a parameter must be parsable by the file:consult call.
The content of the configuration is stored inside the e_conf ets table.

upload_dir() -> UploadDir::string()

Returns the path to the upload directory. The project.conf tuple which sets it should look like: 

     {upload_dir, Path}

where Path is a relative path from the server root directory.
The default value is e_conf:server_root()/docroot/upload

default_language() -> DefaultLanguage::atom()

Returns the default language of the project. During the translation process, if no language is set inside the session variable, this one will be selected. The project.conf tuple which sets it should look like: 

     {default_language, Lan}

where Lan is the atom representing the default language.
The default value is en.

cache_dir() -> CacheDirPath::string()

Returns the directory for disk cache. All cached files will be kept there. The specified directory must be writeable for the server. The project.conf tuple which sets it should look like: 

     {cache_dir, Dir}

where Dir is the path to the desired cache directory. Path is relative to the server root.
The default value is templates/cache.
This option is used only if the disk cache engine is selected.

See also: e_cache, e_cache_disk.

host() -> Hostname::string()

Returns the host the server is running on. It could be useful for redirections to the other ports on the same server. The project.conf tuple which sets it should look like: 

     {host, Host}

where Host is the string name of the host.
The default value is localhost.

debug_mode() -> IsDebugMode::boolean()

Checks if the server is running in the debug mode. If so, all the errors will be displayed in the browser instead of the nice error 404/501 pages. The project.conf tuple which sets it should look like: 

     {debug_mode, Bool}

where Bool is either true or false atom.
The default value is false.

primitive_types() -> List

  • List = [atom()]

Returns the list of all available primitive types. Each provided type should add at least two modules: wpart_Name and wtype_Name. The project.conf tuple which sets it should look like: 

     {primitive_types, List}

where List is a list of the user provided primitive types
The default value is [].

See also: //wpart/wpart, //wpart/wtype.

http_port() -> Port::string()

Returns the http port for the server. The specified port will be used for server to bind to. The project.conf tuple which sets it should look like: 

     {http_port, Port}

where Port is a http port number.
The default value is 80.

https_port() -> Port::string()

Returns the https port for the server. The specified port will be used for server to bind to. The project.conf tuple which sets it should look like: 

     {https_port, Port}

where Port is a https port number.
The default value is 443.

project_name() -> Name::string()

Returns the name of the project we are working on. This name is used in naming databases with objects and ids in e_db_couchdb module. The project.conf tuple which sets it should look like: 

     {project_name, Name}

where Name is the name of the project.
The default value is erlangweb.

See also: e_db_couchdb.

couchdb_address() -> URL::string()

Returns the URL of the CouchDB server. The address is used during the connection to the CouchDB server. The project.conf tuple which sets it should look like: 

     {couchdb_address, URL}

where URL is the CouchDB address.
The default value is http://localhost:5984/.

See also: e_db_couchdb.

dbms() -> DatabaseEngineCallbackModule::atom()

Returns the name of the callback module for the database engine. The currently supported engines are for Mnesia and CouchDB (experimental) databases. The project.conf tuple which sets it should look like: 

     {dbms, DBMS}

where DBMS is atom:

mnesia - for Mnesia support

couchdb - for CouchDB support

The default value is mnesia. The callback modules could be either e_db_mnesia or e_db_couchdb.

See also: e_db, e_db_couchdb, e_db_mnesia.

template_cache_mod() -> CacheModule::atom()

Returns the name of the callback module for the XML records cache. XML scanned files are stored either in the internal memory storage or on disk (in the directory specified by cache_dir option).
The possible values are ets and disk.

See also: e_cache, cache_dir/0.

template_expander() -> TemplateExpanderCallbackModule::atom()

Returns the name of the callback module for templates expanding. This option could be changed by providing the application environment variable template_expander. Currently now wpart_xs is supported.

See also: //wpart/wpart_xs.

template_root() -> TemplateRootDir::string()

Returns the path to the root of the template directory. The templates which should be expanded must be placed there. This option could be changed by providing the application environment variable template_root.
The default value is server_root()/templates.

server_root() -> ServerRootDir::string()

Returns the path to the server root directory. This option could be changed by providing the application environment variable server_root.
The default value is the parent directory of the eptic application.

ecomponents() -> Ecomponents::[{atom(), [tuple()]}] | []

Returns the list of declared ecomponents with the provided configuration. The ecomponents should be declared inside the project.conf file: 

     {ecomponents, [{name_of_ecomponent1, [ListOfConfiguration]},
                    {name_of_ecomponent2, [ListOfConfiguration]},
                    ...
                    {name_of_ecomponentN, [ListOfConfiguration]}]}.

All declared components will be started during the system start (in the order of listing in configuration file).
If no ecomponents are declared, the empty list is returned.

See also: //ecomponent/ecompontent.

get_conf(Key::term()) -> Configuration::term()

Returns the configuration associated with the given Key. If no configuration has been found, the undefined atom is returned.
