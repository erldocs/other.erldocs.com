e_db

A common interface to the all supported database engines.

A common interface to the all supported database engines. Its behaviour depends on the database engine framework uses. The engine could be set inside the project.conf file.

Functions

install() -> any()

Sets up the environment for the selected database.

start() -> ok | {error, Reason::term()}

Starts the DBMS.

read(Type::atom()) -> [Element] | {error, Reason}

  • Element = tuple()

Reads and returns all the entites stored in the database of the given Type.
In case of error, {error, Reason} is returned.

read(Type::atom(), Id::any()) -> Element::tuple() | not_found | {error, Reason}

Reads and returns the entity of the given Type and Id. The pair {Type, Id} should match at most one element stored in the database.
When the element is not found, the atom not_found is returned.
In case of error, the {error, Reason} is returned.

delete(Type::atom(), Key::any()) -> ok | {error, Reason}

Removes the given element of the Type with Key from the database. If removal process succeeds, ok is returned.
In case of error, the {error, Reason} is returned.

write(Type::atom(), Element::tuple()) -> ok | {error, Reason}

Writes the Element of the Type to the database. If writing process succeeds, ok is returned.
In case of error, the {error, Reason} is returned.

update(Type::atom(), Element::tuple()) -> ok | {error, Reason}

Updates the Element of the Type in the database. If update succeeds, ok us returned.
In case of error, the {error, Reason} is returned.

size(Type::atom()) -> integer() | {error, Reason}

Checks the number of elements of the Type stored in database.

get_next_id(Type::atom()) -> integer() | {error, Reason}

Generates and returns the unique number for the Type. The operation is transactional. The generated id could be used for creating more sophisticated id's (it could be for example combined with the Type - but there is no need to).
This function will never return the same value twice for the same Type
