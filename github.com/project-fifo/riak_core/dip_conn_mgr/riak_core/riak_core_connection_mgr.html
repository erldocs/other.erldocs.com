connect(Target::string(), ClientSpec::clientspec(), Strategy::client_scheduler_strategy()) -> {ok, reference()}

Establish a connection to the remote destination. be persistent about it, but not too annoying to the remote end. Connect by name of cluster or IP address. Use default strategy to find "best" peer for connection.

Targets are found by applying a registered locator for it. The identity locator is pre-installed, so if you want to connect to a list of IP and Port addresses, supply a Target like this: {identity, [{IP, Port},...]}, where IP::string() and Port::integer(). You can also pass {identity, {IP, Port}} and the locator will use just that one IP. With a list, it will rotate trying them all until a connection is established.

Other locator types must be registered with this connection manager before calling connect().

Supervision must be done by the calling process if desired. No supervision is done here.
