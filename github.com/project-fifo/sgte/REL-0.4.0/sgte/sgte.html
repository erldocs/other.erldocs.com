The SGTE module is a library implementing a Template Engine The template system is inspired on String Template

The use of the Engige is as simple as (from the command line): 

   > {ok, Compiled} = sgte:compile(TmplStr),
   > sgte:render(Compiled, Data).
   or:
   > {ok, Compiled} = sgte:compile_file(FileName),
   > sgte:render(Compiled, Data).

Where Template can be either a string or a tuple {file, FileName} Data can be a Dict or a list of tuple (e.g. [{attr1, Val1}, {attr2, Val2}]) Values can be a simple value or a function/1. In this case the function is called with Data as an argument.

Template Features:

attribute reference: e.g. 
$name$
 template reference: e.g. 
$include tmpl$
. application of an attribute to another: e.g. 
$apply myFun
   aVar$
. When the first attribute is callable you get the result of myFun(aVar). Otherwhise the result is the value of myFun. conditional evaluation: e.g. 
     $if title$
         <h1>$title$</h1>
     $else
         <h1>default title</h1>
     $end if$
template application to a list of elements: e.g. if names is a list [{username, name1}, {username, name2}] 
$map li names$
map li template to names. Each element in names is passed to the template with name attr. If li is the template: 
<li><b>$username$</b></li>
We get the result: 
            <li><b>name1</b></li>
            <li><b>name2</b></li>
Another ways to express the same template inline is: 
$map:{<li><b>$username$</b></li>} names$
 alternate several templates on a list of elements: 
$map row1 row2 row3 names$
Inline syntax: 
     $map:{<li class="row1">$username$</li>,
           <li class="row2">$username$</li>,
           <li class="row3">$username$</li>} names$
join of items using a separator: 
SELECT $join "," columns$ FROM $table$;

Functions

compile(T) -> {ok, Compiled} | {error, Reason}

Compiles the template string T and returns the compiled template or an error. 

  Expects:
   T - The string containing the template.
 
  Types:
   T = [char()]
   Compiled = [char()|tuple()]
   Reason = tuple()

compile_file(FileName) -> {ok, Compiled} | {error, Reason}

Compiles the template file FileName and returns the compiled template or an error. 

  Expects:
   FileName - The file containing the template.
 
  Types:
   FileName = [char()]
   Compiled = [char()|tuple()]
   Reason = tuple()

render(Compiled, Data) -> Rendered

Renders the compiled template and returns it 

  Expects:
   Compiled - The compiled template.
   Data - The data referred in the template
 
  Types:
   Compiled = [char()|tuple()]
   Data = [tuple()]|dict()
   Rendered = [char()]
