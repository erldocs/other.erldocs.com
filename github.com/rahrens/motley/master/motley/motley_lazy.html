is_empty(LazySeq::lazy_seq()) -> bool()

This function is deprecated:

Avoid using this as it pays the cost up-front of generating the next member of the sequence -- something that will happen twice in a loop context. The approved way to check for emptiness is to use next:

case next(LazySeq) of {stopped, Reason} -> ... {ok, Datum, LazySeq2} -> ... end.

Used for look-ahead to determine if the sequence is empty or not. This is handled by calculating the next member of the sequence.
