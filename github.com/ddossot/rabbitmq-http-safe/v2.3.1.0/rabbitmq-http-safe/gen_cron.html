gen_cron

Provides a gen_server-like interface to periodic execution.

Provides a gen_server-like interface to periodic execution. The handle_tick/2 callback is executed in a seperate process periodically or in response to a force_run/1 command. Only one instance of a spawned handle_tick/2 is allowed at any time.

Schedule an immediate execution. If the process is already executing then { underway, Pid } is returned.

handle_info(Msg, State) -> Result

Just like the gen_server version, except that messages related to spawned process monitor are intercepted and handled (and forwarded to the callback module in a { tick_monitor, Msg } tuple), and messages related to the periodic timer are handled (and not forwarded to the callback module).

terminate(Reason::Result, State) -> Result

Just like the gen_server version, except that if a process is running, we wait for it to terminate (prior to calling the module's terminate).

handle_tick(Reason::reason(), State) -> none()

This is called as a seperate process, either periodically or in response to a force. The State argument is the server state at the time of the spawn.
