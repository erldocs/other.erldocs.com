Nekso Process Registry.

Nekso Process Registry.

This module implements an ETS-based process registry. It allows a process to register any term() as a process identification, and store any metadata with it. When the process exists, these terms are deleted.

Functions

put(Name::term()) -> ok

Equivalent to put(Name, undefined, self()).

put(Name::term(), Value::term()) -> ok

Equivalent to put(Name, Value, self()).

put(Name::term(), Value::term(), Pid::pid()) -> ok

Registers Name (and Value) as associated with this process. Several processes can register the same Name with the same of different values. When the process ends, its registrations are deleted.

del(Name::term()) -> ok

Equivalent to del(Name, self()).

del(Name::term(), Pid::pid()) -> ok

Unregisters a previously registered Name, associated to this process.

del_all() -> ok

Equivalent to del_all(self()).

del_all(Pid::pid()) -> ok

Unregisters all registrations from this process.

values(Name::term()) -> [{Value::term(), Pid::pid()}]

Gets all registered Values for Name, along with registering pids.

reg(Name::term()) -> true | {false, pid()}

Equivalent to reg(Name, undefined, self()).

reg(Name::term(), Value::term()) -> true | {false, pid()}

Equivalent to reg(Name, Value, self()).

reg(Name::term(), Value::term(), Pid::pid()) -> true | {false, pid()}

Similar to put(Name, Value, Pid) but only allows for one single registration. Returns true if this registration has succeded, or false if another process has already registered this name.

reg(Name::term(), Value::term(), Pid::pid(), Timeout::integer() | infinity) -> true | timeout

Similar to reg(Name, Value, Pid), but, in case other process has already registered this name, waits for it to be unregistered and then we are automatically registered.

wait_put(Name::term()) -> {ok, pid()} | timeout

Equivalent to wait_put(Name, 5000).

wait_put(Name::term(), Timeout::integer() | infinity) -> {ok, {Value::term(), pid()}} | timeout

Waits for Name to be registered from any process before Timeout.

wait_del(Name::term()) -> ok | timeout

Equivalent to wait_del(Name, 5000)).

wait_del(Name::term(), Timeout::integer() | infinity) -> ok | timeout

Waits for Name to be unregistered from all pids before Timeout.

register_name(Name::term(), Pid::pid()) -> yes | no

Process registration compatible with global:register_name/2.

unregister_name(Name::term()) -> pid() | undefined

Unregistration compatible with global:unregister_name/1.

whereis_name(Name::term()) -> pid() | undefined

Finds a registered Name compatible with global:whereis_name/1.

send(Name::term(), Msg::term()) -> pid()

Sends a message to a registered process compatible with global:send/2.

fold_names(UserFun::function(), Acc0::term()) -> term()

Folds over all registered names in the same way as lists:foldl/3. UserFun will be called for each record as UserFun(Name, Values, Acc) when Values :: [{Value::term(), pid()}].

fold_pids(UserFun, Acc0) -> term()

Folds over all registered pids in the same way as lists:foldl/3. UserFun will be called for each record as UserFun(Pid, Items, Acc) when Names :: [{name|put|del|reg, term()}].

start(Type::server | fsm, Name::term(), Module::atom(), Args::[term()]) -> {ok, pid()} | ignore | {error, {already_started, pid()} | term()}

Starts a new gen_server or gen_fsm, allowing registration under any term.

start_link(Type::server | fsm, Name::term(), Module::atom(), Args::[term()]) -> {ok, pid()} | ignore | {error, {already_started, pid()} | term()}

Starts a new gen_server or gen_fsm, allowing registration under any term, and linking the new process.
