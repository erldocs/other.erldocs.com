Nekso In-Memory Database.

Nekso In-Memory Database.

This module implements an ETS-based database used for registrations and caches. It is capable of timed auto expiring of records, server-side update funs and calling an user fun on record expire.

Functions

Equivalent to get(Key, not_found).

get(Key::term(), Default::term()) -> term()

Gets a record from database using Default if not found

put(Key::term(), Value::term()) -> ok

Equivalent to put(Key, Value, []).

put(Key::term(), Value::term(), Opt::[Opt]) -> ok

  • Opt = {ttl, integer()} | {notify, function()}

Inserts a Value in database under Key. If ttl option is used (seconds), record will be deleted after this time. If notify options is used, function will be called on record's delete.

update(Key::term(), Fun::function()) -> {ok, FunResult::term()} | {error, Error::term()}

Equivalent to update(Key, Fun, []).

update(Key::term(), Fun::function(), Opts::optslist() (see module nklib)) -> {ok, FunResult::term()} | {error, Error::term()}

Updates a record in database, applying Fun to the old record to get the new value. If no record is found, old value would be []. If the new generated value is [] record will be deleted. See put/3 for options.

del(Key::term()) -> ok | not_found

Deletes a record from database.

fold(UserFun::function(), Acc0::term()) -> term()

Folds over all records in database like lists:foldl/3. UserFun will be called as UserFun(Key, Value, Acc) for each record.
