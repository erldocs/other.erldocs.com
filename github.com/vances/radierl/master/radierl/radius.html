Callback Functions init/2

Module:init(Address :: ip_address(), Port :: pos_integer()) -> ok | {error, Reason :: term()}

Whenever a radius_server is started using radius:start/3,4, or radius:start_link/3,4, this function is called by the new process to initialize.

Address is the ip_address() which the socket() is listening on.

Port is the port_number() which the socket() is listening on.

This function should return ok if the callback handler will be able service RADIUS requests on this radius_server or {error, Reason} indicating the problem.

request/3

Module:request(Address :: ip_address(), Port :: pos_integer(), RadiusRequest:: binary()) -> {error, Reason :: ignore | term()} | RadiusResponse :: binary()

When a new valid RADIUS packet is received a radius_fsm transaction handler is started which will call this function to process the request.

Address is the ip_address() of the client making the request.

Port is the port_number() of the client making the request.

This function should return a RadiusResponse, as returned by radius:codec/1, if the request was valid. If the RADIUS Authenticator or Attributes were badly formed {error, ignore} should be returned to silently discard the received packet and ignore retransmissions. In the event of an error {error, Reason} should be returned which will stop the radius_fsm transaction handler and report an error.

terminate/1

Module:terminate(Reason :: term()) -> any()

This function is called by a radius_server when it is about to terminate. It should be the opposite of Module:init/1 and do any necessary cleaning up. When it returns, the radius_server terminates with Reason. The return value is ignored.
