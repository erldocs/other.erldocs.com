Stream socket (listening socket) behaviour.

Stream socket (listening socket) behaviour.

Stream socket is a socket with an idea of connection and listen state.

Expected callbacks 
listen(Address) ->
    {ok, State :: term()}
  | ignore
  | {error, Reason :: term()}

State contains listening socket. 

accept(State) ->
    {ok, Connection, State}
  | {ok, State}
  | {stop, Reason, State}

Connection will be passed as an argument to connection handler module.

It's a good idea for this function to return after some specified time passed (e.g. after 100ms), so the process can receive and handle incoming messages and calls. 
controlling_process(Connection, Pid) ->
    ok
  | {error, Reason}

Make Pid to be the controlling process ("owner") of the Connection. 

close(State) ->
  any()

 Connection handler module

Connection handler can be any module. Especially well suited are modules implementing gen_server or gen_event behaviours.

Such module should only export a function that spawns new linked process (default name for such function is start_link).

The function will be called with connection() (returned by ListenModule:accept/1) as its only argument. The function should return {ok,pid()}, ignore or {error,Reason} (returned values from gen_server:start_link/3 are fine here). It is this function's responsibility to close passed connection() in case of ignore or {error,Reason}.

Small code example Starting (unsupervised) my_connection_listener spawning my_connection_handler (TCP listener not shown here for brevity): 
    gen_sock_stream:start_link(
      my_connection_listener,
      {my_connection_handler,start_link}, % or just `my_connection_handler'
      {"localhost", 12345}
    ).
TCP connection handler: 
    -module(my_connection_handler).
    -behaviour(gen_server).
  
    -export([start_link/1]).
    % ...
  
    start_link(Connection) ->
      case gen_server:start_link(?MODULE, Connection, []) of
        {ok, Pid} ->
          {ok, Pid};
        Failure ->
          gen_tcp:close(Connection),
          Failure
      end.
    % ...

start_link(ListenModule::listener_module(), ConnHandlerModule::connection_handler(), Address::listen_address()) -> {ok, pid()} | ignore | {error, Reason}

Start listener process outside of supervision tree.

ListenModule is expected to implement gen_sock_stream behaviour (see Expected callbacks).

ConnHandlerModule specifies module to handle single connection in a separate process. Such process is started by function part in ConnHandlerModule (or start_link/1 if it's not specified). See connection_handler() for details.

supervisor(ListenModule::listener_module(), ConnHandlerModule::connection_handler(), Address::term()) -> {ok, Supervisor::pid()} | ignore | {error, Reason}

Start supervision (sub)tree for listener process. This function should be used instead of start_link/3 if the listener is meant to be a part of supervision tree.

ListenModule is expected to implement gen_sock_stream behaviour (see Expected callbacks).
