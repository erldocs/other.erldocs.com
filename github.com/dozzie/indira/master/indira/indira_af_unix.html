indira_af_unix

Module for working with AF_UNIX connections.

Module for working with AF_UNIX connections. The module mimics gen_tcp behaviour.

Active socket sends:

{unix, Socket, Data :: binary()}

{unix_closed, Socket}

{unix_error, Socket, Reason}

Types

connection_socket() = socket()

option() = {active, true | false}

server_socket() = socket()

socket() = port()

Functions

listen(Address::string()) -> {ok, server_socket()} | {error, Reason}

Setup a socket listening on specified address.

accept(Socket::server_socket()) -> {ok, connection_socket()} | {error, Reason}

Accept a client connection. The function waits infinitely for a client.

accept(Socket::server_socket(), Timeout::non_neg_integer()) -> {ok, connection_socket()} | {error, timeout} | {error, Reason}

Accept a client connection.

connect(Address::string(), Opts::[option()]) -> {ok, connection_socket()} | {error, Reason}

Connect to specified socket. Resulting connection defaults to passive (if not specified in options).

This function allows to send to datagram sockets as well, but recv/2 and recv/3 won't work on such socket.

send(Socket::connection_socket(), Data::iolist() | binary()) -> ok | {error, Reason}

Send data to the socket.

recv(Socket::connection_socket(), Length::non_neg_integer()) -> {ok, Packet} | {error, Reason}

Read Length bytes from socket.

recv(Socket::connection_socket(), Length::non_neg_integer(), Timeout::non_neg_integer() | infinity) -> {ok, Packet} | {error, Reason}

Read Length bytes from socket (with timeout).

controlling_process(Socket::socket(), Pid::pid()) -> ok | {error, Reason}

Set the socket owner.

setopts(Socket::socket(), Options::[option()]) -> ok | {error, Reason}

Set socket options.

close(Socket::socket()) -> ok

Close server connection.
