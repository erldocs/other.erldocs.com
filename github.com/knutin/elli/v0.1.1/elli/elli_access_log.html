HTTP access and error long, sending to syslog over UDP.

HTTP access and error long, sending to syslog over UDP

Sends a simple log line for every request, even errors, to syslog. The line includes the following timings, all specified in wall-clock microseconds:

RequestLine/Headers/Body/User/Response/Total

* RequestLine: time between accept returning and complete receive of the request line, ie. "GET /foo HTTP/1.1". If keep-alive is used, this will be the time since the initial accept so it might be very high.

* Headers: Time to receive all headers

* Body: Time to receive the entire body into memory, not including any decoding

* User: Time spent in the callback. If middleware is used, the runtime of the middleware is included in this number

* Response: Time taken to send the response to the client

* Total: The time between the request line was received and the response was sent. This is as close we can get to the actual time of the request as seen by the server.
