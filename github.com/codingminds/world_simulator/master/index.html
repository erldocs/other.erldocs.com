Module Index


world_simulator

world
world is the application which combines the simulated environment, the socket server which handles communications and the supervisors.

world_ctl_sserv
world_ctl_sserv is a socket based server which provides the interfaces to control the simulated world.

world_ctl_sservsup
world_ctl_sservsup is the supervisor of the socket server which provides the control interface.

world_env
world_env holds the environment and the logic to interact with the modeled virtual world.

world_envsup
world_envsup is the supervisor of the environments.

world_helper
world_helper holds some functions which are used by more then one module of application world.

world_http
world_http uses the erlang inets http implementation to provide a read only interface to the environment.

world_logging
world_logging get log messages as casts from the different modules, sort them and write it down.

world_ofo_sup
The one for one supervisor world_ofo_sup which monitors all supervisors and workers that can restarted without big trouble or dependencies.

world_sserv
world_sserv is a socket based server which provides the user interface for the simulated world.

world_sservsup
world_sservsup is the supervisor of the socket server which provides the user interface.

world_sup
The global supervisor world_sup which builds the supervision tree of the application world.
