trigger_dynamic

Dynamic trigger for (parameterized) modules.

Dynamic trigger for (parameterized) modules.

Can be used by a module Module in order to get a configurable message (by default {trigger}) every BaseInterval, MinInterval or MaxInterval milliseconds depending on a user-provided interval tag specified with next/2.

Use this module through the interface provided by the trigger module, initializing it with trigger_dynamic!

Types

state() =
            {BaseInterval :: trigger:interval_time(),
             MinInterval :: trigger:interval_time(),
             MaxInterval :: trigger:interval_time(),
             MsgTag :: comm:msg_tag(),
             TimerRef :: ok | reference()}

Functions

init(BaseInterval::interval_time() (see module trigger), MinInterval::interval_time() (see module trigger), MaxInterval::interval_time() (see module trigger), MsgTag::msg_tag() (see module comm)) -> state()

Initializes the trigger with the given interval functions and the given message tag used for the trigger message.

now(X1::state(), ReplyTo::erl_local_pid() (see module comm)) -> state()

Sets the trigger to send its message immediately, for example after its initialization. Any previous trigger will be canceled!

next(X1::state(), IntervalTag::interval() (see module trigger), ReplyTo::erl_local_pid() (see module comm)) -> state()

Sets the trigger to send its message after some delay. The given IntervalTag will determine which of the three interval functions will be evaluated in order to get the number of milliseconds of this delay. Any previous trigger will be canceled!

stop(State::state()) -> state()

Stops the trigger until next or now are called again.
