umb_bus

Frontend module that represents bus between devices.

Frontend module that represents bus between devices.

There is basically two methods for user to use: connect/3 and request/4.

Functions

connect(NewBusId::atom(), TransportName::atom(), TransportArgs::list()) -> {ok, Id::atom()}

Connect to device bus.

NewBusId must be any unique bus id just for identifying bus with next requests. TransportName must be a module name, that provides transport level for bus. TransportArgs is a arbitary args, passed to transport module.

For example of transport module umb_transport_tcp.

request(BusId::atom(), From::device() (see module umb_device), To::device() (see module umb_device), Frame::frame() (see module umb_frame)) -> {ok, Result::term()} | {error, Reason::term()}

Send request to bus and return answer.

BusId is a NewBusId, passed to connect/3. From is a device, that sends request (umb_device). To is a device, that is a target of request. Frame is a constructed frame for a request. You should use umb_request for constructing such frames.
