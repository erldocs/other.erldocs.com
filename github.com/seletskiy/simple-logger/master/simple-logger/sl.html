open(LogFile::list()) -> ok | {error, term()}

Open specified log file for use and make it available for current application.

Currently only one log per application and one top level log is supported.

Top level log is a log that opened without application context.

Application name obtained via application:get_application/0 call.

open(LogFile::string(), Opts::[atom() | tuple()]) -> ok | {error, term()}

Same as open/1, but with additional opts. Opts is a proplist with keys:

format specifies fun/2 that should format incoming item before writing to log

fun/2 must have 2 arguments: Severity, ItemToLog.

fun/2 should return list() result that represents formatted item.

Caution: formatting function will be called in context of caller module, so any error in formatting function can crash module itself.

Default format is: [Day/Mon/Year Hour:Min:Sec.Micro] Pid Module (Severity) ...

install(EnvName) -> term()

Install log to caller application to use, if posible.

Shorthand for: 

       case application:get_env(EnvName) of
           undefined -> ok;
           {ok, FileName} -> open(FileName, Opts)
       end.

EnvName is stands for env variable name to take log file name from.

install(EnvName, Opts) -> term()

Same as install/1, but with custom Opts passed to open/2.

close() -> ok | {error, term()}

Close log and flush it to disk.

debug(Item::string()) -> ok | {error, term()}

Log item with debug severity. See log/2 for more info.

debug(Item::string(), Args::list()) -> ok | {error, term()}

Same as debug/1, but uses io_lib:format to format Item with Args. See log/2 for more info.

debug(Log::atom(), Item::string(), Args::list()) -> ok | {error, term()}

Same as debug/2, but allows to specify log id (e.g. application name to log to).
