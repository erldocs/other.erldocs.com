gen_tcpd

gen_tcpd application.

gen_tcpd application. Makes easy to create TCP daemons. Should be started with application:start(gen_tcpd).

bind(Port::integer(), MFA::mfa(), Options::list()) -> {ok, pid()}

Binds callback module to specified port. This method creates listen socket and distributes it on acceptors pool (default - 5 workers).

MFA will be called after succeeded connection on Port.

MFA must return {ok, State}.

MFA must have behaviour gen_tcpd_handler_behaviour.

See gen_tcpd_handler_example_echo to learn how to write simple callback module.

See gen_tcpd_conn to learn how to control socket connection.

Options is a prolist with keys:

pool describes acceptors pool options:

workers - workers count (default - 5);

max_restarts - maximum restarts in restarts_time (default - 5);

restarts_time - maximum interval in seconds for max_restarts (default - 60);

socket describes listen socket options:

all options from inet:setopts() except active.

See also: gen_tcpd_handler_behaviour, gen_tcpd_handler_example_echo.

Switch socket to active state. recv/2 from callback module will be callbed, when some data arrived at socket.

activate(Pid::pid(), BufferSize::pos_integer()) -> ok

Switches socket to active state in buffered mode. recv/2 from callback module will be called only after BufferSize bytes was arrived at socket.
