gen_tcpd_conn

Module implements 2 roles: recv and send.

Module implements 2 roles: recv and send.

it receives all messages, that are sended by connected client; it sends messages to connected client that are specified in send/2.

Socket opens with {active, false} option, so you need manually to set it to be able to accept incoming messages. It can be done with activate/1 and wait_data/2 functions.

See gen_tcpd_handler_example_echo for example usage.

Functions

start_link(Socket::socket() (see module gen_tcp), MFA::mfa()) -> {ok, pid()}

Start connection module and spawn user defined Module.
