create_user(Name, Pass) -> term()

Creates a new user with the given password. The password will be hashed using sha512 and then passwed to the the passwd module to be salted and hashed again. Passwords are hashed twice so that when a user wants to authenticate, the password can be hashed *before* being sent to the auth server. This way, if an auth request is sent to the auth server and causes the auth server to crash, when the Erlang shell prints the last message sent to the server, the password won't be in plain text.
