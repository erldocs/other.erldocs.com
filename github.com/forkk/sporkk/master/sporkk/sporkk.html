API Functions for managing and controlling Sporkk bots.

API Functions for managing and controlling Sporkk bots. ============================================================================

Functions

start() -> term()

Starts the main application process and its dependencies.

start(BotId) -> term()

Starts the bot with the given ID.

stop(Id) -> term()

Stops the bot with the given ID. Returns ok if the bot stops successfully or isn't running.

join(Id, Channels) -> term()

Tells the bot with the given ID to join the given channels and adds the channels to the bot's DB entry.

part(Id, Channels, Reason) -> term()

Tells the bot with the given ID to part from the given channels and removes the channels from the bot's DB entry.

part(Id, Channels) -> term()

load_mod(BotId, ModId) -> term()

Tells the given bot to load a module with the given ID.

unload_mod(BotId, ModId) -> term()

Tells the given bot to unload the given module.

send(Id, Dest, Message) -> term()

Tells the bot with the given ID to send the given message to the given destination.

notice(Id, Dest, Message) -> term()

Tells the bot with the given ID to send the given notice to the given destination.

log_info(Id, Message) -> term()

Sends the given message to the info logging system. This will send the message to all the channels listed in the bot's 'log_chans' extra.

core(Id) -> term()

Gets the core process name for the given bot ID.

connector(Id) -> term()

Gets the connector process name for the given bot ID.

sender(Id) -> term()

Gets the sender process name for the given bot ID.

receiver(Id) -> term()

Gets the receiver process name for the given bot ID.

modserv(Id) -> term()

Gets the module server process name for the given bot ID.

modsup(Id) -> term()

Gets the module supervisor process name for the given bot ID.

authserv(Id) -> term()

Gets the authentication server process name for the given bot ID.
