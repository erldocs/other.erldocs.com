eswf_abc

The eswf_abc module provides functions for manipulating ActionScript Byte Code segments (herein ABC segments).

The eswf_abc module provides functions for manipulating ActionScript Byte Code segments (herein ABC segments).

A full description of the ABC data format is outside of the scope of this documentation. For details, see chapter 4 of ActionScript Virtual Machine 2 Overview.

Types

Functions

stringmap(Fun, ABCSegment::abcsegment()) -> iodata()

  • Fun = (String::binary()) -> iodata() | skip

Returns an ABC segment where each string constant String in ABCSegment has been replaced by Fun(String). Fun(String) must return a valid UTF-8 sequence. Alternatively, Fun(String) may return skip, and String will remain unmodified in the resulting ABC segment. (Returning skip is semantically equivalent to returning String but slightly faster.)

Except as mentioned above, the returned ABC segment does not otherwise differ from ABCSegment. The result of applying stringmap to an invalid ABC segment is undefined.

stencilify(ABCSegment::abcsegment(), Fun, UserAcc::Acc) -> {template(), NewAcc}

  • Fun = (String::binary()) -> {Action, NewAcc}
  • Action = skip | {punch, term()}

Returns a template() based on ABCSegment. For each string constant String in ABCSegment, Fun(String, Acc) is evaluated, which should return a tuple with an action and a new accumulator. If the action is {punch, Term}, then the corresponding section of the template contains {hole, Term}, which should later be replaced by a valid string constant encoding (see encode_string/1). Otherwise, the action must be skip, and the string constant is left in tact.
