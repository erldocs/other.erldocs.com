proc_mobility

Main module exporting API to process migration.

Main module exporting API to process migration. To use this mechanism functions from this module should be used.

Types

destination() = node() | {tcp, binary(), integer()}

mproc_state() =
            #mproc_state{name = undefined | atom(),
                         module = undefined | atom(),
                         state = undefined | term(),
                         code = list()}

Functions

migrate(PState::mproc_state(), Target::destination()) -> ok | {error, term()}

sends process to given destination Destination can be a node in erlang cluster or node outside the cluster but with proc_mobility_tcp_server started

started(Listener) -> term()

send(Proc::atom(), Msg::any()) -> any()

Sends message to mobile process Calling process location is transparent, it can be on the same node, other node in cluster or outside the cluster if mobility mechanism is configured to work over plain TCP connections

whereis_name(Proc::atom()) -> pid() | unknown

Return pid of mobile process It can be distributed process pid or process proxy pid if migrated outside Erlang cluster

register_name(Name::atom(), Pid::pid()) -> boolean()

Registers given mobile process. Has to be called by registering process

unregister_name(Name::atom()) -> boolean()

Unregister previously registered mobile process name

get_tcp_server_port() -> integer()

Returns current node TCP port number on which listens for migration requests outside cluster
