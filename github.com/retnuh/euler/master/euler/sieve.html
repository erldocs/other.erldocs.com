A version of Euler's the Seive of Eratosthenes (see http://en.wikipedia.org/wiki/Sieve_of_Eratosthenes). By using actors we can have a "prime server" that computes primes on demand - kind of like a lazy-seq in clojure or something, but one central dude.

Technically it may not be the Sieve as defined, because I'm computing values on demand. Rather than crossing off numbers from an existing list, I build a list of functions that I then test the next candidate number (next odd number after current known prime) against.

The functions created are simple filter functions (i.e. x rem 2 == 0). I reckon that the list of primes will grow a lot more slowly than taking a big list of numbers, and doing the various sieve things to it.
