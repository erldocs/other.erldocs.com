Generic TCP Acceptor.

Generic TCP Acceptor

This is a behaviour that implements an asynchronous TCP acceptor server. The server implementation then only needs to implement start_link/0, init/1, and handle_accept/2.

See tcp_listener, gen_server(3), and gen_tcp(3) for more information.

Inspired by http://www.trapexit.org/Building_a_Non-blocking_TCP_server_using_OTP_principles http://github.com/erlware/gen_socket http://github.com/kaos/gen_listener_tcp
