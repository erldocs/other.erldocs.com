validator

Handles interaction with the validator component.

Handles interaction with the validator component.
The validator is responsible for verifying and scoring propositions from the workers. This is the main validator module, the rest of the application should use this to interface with the validator component.
The function start_link starts a single validator_sup which then starts a proposition queuing process from validator_queue and a port process (validator_port) via another supervisor validator_port_sup).

Functions

start_link(ExtProg) -> term()

Starts the whole validator infrastructure, including queue and external executable.

validate(WorkerID::atom(), WorkerInput::[string()], WorkerOutput::string()) -> ok

Submits a single proposition from a worker to the validator.

round_started() -> ok

Informs the validator that the workers are working on a new round.
The validator has to be ware of this state because when the workers are stopped and the queue runs dry, the validator will tell the dj that it's OK to end the round. This shouldn't happen while the workers are running.

all_workers_stopped() -> ok

Informs the validator that all workers are stopped.
The next time the queue runs dry, the validator assumes that all propositions have been processed and tells the dj to end the round.
