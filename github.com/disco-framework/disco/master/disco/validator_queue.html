validator_queue

Provides a process to queue incoming propositions.

Provides a process to queue incoming propositions.
The validator pulls the propositions from the queue via get_proposition/0 and returns the answer via put_score/3. The queue process keeps proposition until it gets the validation result so when the validator crashes, no proposition is lost.
When a round is over, there may still be unvalidated propositions in the queue. Therefore, the dj tells the queue that the round time is over (all_workers_stopped/0) and the queue sends a message back the next time it runs dry. When this has happened, the final results for the round can be computed and the round is officially over.
In this text, "validator" means a process spawned by the validator_port module.
