gman

The GMan is the govener between the user configurations and plugins, and the outside world.

The GMan is the govener between the user configurations and plugins, and the outside world. It handles all authentication as well as securely passing messages to and from a user's uman.

Functions

start_link() -> term()

Start the Gman server.

add_uman(User::EMPUSER(), Pid::pid()) -> ok

Register the user with the GMan so that it knows who can log in.

send_event(User::EMPUSER(), Event::EMPEVENT()) -> ok

Broadcasts the event to all Eprocs for a given User.

login(X1::{rsa, PUBKEY::binary()}) -> loginresult()

Register a client using a rsa public key.

login(UserName::binary(), PassHash::binary()) -> loginresult()

Register a client using a Username and password hash.

logout(UUID::binary()) -> ok | {error, Reason::binary()}

Unregister a client and remove the sessionid from the session list.

run_cmd(UUID::binary(), Target::msgtarget(), Command::binary(), ParamList::[binary()]) -> utilret()

Ask to run a command on the user's uman. If the target is not null, it will be run as a plugin command.

run_query(UUID::binary(), Query::binary()) -> utilret()

Similar to run_cmd, it will ask to run a command on the user's uman. All queries are system queries, so theres no possibility of the target not being there. However, queries can require permissions.
