Module Index


empd

eman
The Event Manager handles the broadcasting of event messages to all handlers.

emp_plugin
This module defines a host of functions that EMP plugins can use to access and set state, trigger events and generally talk to the core of EMP.

empd_app
The entry point for the EMP core server.

empd_sup
This is the EMP Service Monitor.

empinterface
Library to generalize the EMP Interface API so that Interfaces can focus on the protocol they are supporting rather than the API issues.

eproc
The Event Processor facilitates the triggering of commands based on event subscriptions.

gman
The GMan is the govener between the user configurations and plugins, and the outside world.

iman
The Interface manager is a supervisor for all the interface handlers.

plugrunner
Plugin runner is a server that communicates with a plugin on behalf of Emp.

pman_sup
The PMan (Plugin Manager) is a supervisor that keeps track of the plugins and will monitor for if they crash, it will restart them if this happens.

pr_msg_handler
This module is the message handling portion of Plug Runner.

seproc
When EMP starts up, the root eman creates N System Plugin Event Processors (seproc), where N is the number of users.

uman_cmd
This Module contains the functions responsible for handling UMan's System command handlers.

uman_serve
The User Management Server is the primary source of Message handling for a given user within EMP.

uman_sup
The supervisor portion of the UMAN logical block.

uwatcher
User watcher is a supervisor that generates and runs all user nodes as their own processes.
