yanger_plugin

Yanger Plugin API A Yanger Plugin is defined using the 'yanger_plugin' behavior.

Yanger Plugin API A Yanger Plugin is defined using the 'yanger_plugin' behavior. This behavior defines a set of callback functions that the plugin implements.

This module contains the API functions a plugin should use to interact with Yanger. The header file "yang.hrl" is also considered part of the API. Specifically, "yang.hrl" specifies a set of records and types used by Yanger and the plugins. -------------------------------------------------------------------

install_arg_types(Types::[{ArgTypeName::atom(), XsdRegexp::string() | undefined, ErlType::return_type() (see module yang_parser)}]) -> ok | error

Install type definitions for YANG extension statements's arguments. Must be called from the plugin's init() function.

XsdRegexp is used to make the parser validate the argument. 'undefined' means that any string is accepted.

The parser converts each argument of type ArgTypeName to the erlang type in specified in ErlType.
