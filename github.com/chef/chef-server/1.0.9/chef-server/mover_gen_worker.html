We verify that the object names we fetched from couch at the start of this migration match those we can pull back from MySQL. If so, then the migration was successful and we issue deletes for all the couch ids to solr. If the lists are not the same, then one or more objects failed to be migrated. No solr delete is issued and the worker crashes to indicate that the migration (partially) failed.
