CPG Groups Handling. Method of using cpg instead of pg2.

CPG Groups Handling. Method of using cpg instead of pg2. The resulting process group handling is more scalable and more efficient. However, usage is limited to string (list of integers) group names (unless the GROUP_STORAGE macro is changed and pattern matching is disabled). The groups state is obtained from the cpg process for a specific scope and is then used with the functions provided here, so that contention for the cpg process can be avoided.

Functions

get_groups() -> term()

Get the group storage. This provides the internal representation of process groups so that requests will not be blocked by the single process managing the scope of the process groups.

get_groups(Scope) -> term()

Get the group storage for a particular scope or after a period of time. This provides the internal representation of process groups so that requests will not be blocked by the single process managing the scope of the process groups.

get_groups(Scope, Time) -> term()

Get the group storage for a particular scope after a period of time. This provides the internal representation of process groups so that requests will not be blocked by the single process managing the scope of the process groups.

get_empty_groups() -> term()

Get empty group storage.
