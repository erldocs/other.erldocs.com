pqueue4

Static Priority Queue. This priority queue implementation depends on a static number of priorities (-128 (high) to 128 (low)) so that tuple access times can be exploited for quick in/out priority queue operations.

Static Priority Queue. This priority queue implementation depends on a static number of priorities (-128 (high) to 128 (low)) so that tuple access times can be exploited for quick in/out priority queue operations. This implementation was created to avoid the slowness within the priority queue used by both RabbitMQ and Riak (https://github.com/basho/riak_core/blob/master/src/priority_queue.erl).
