pqueue

pqueue
Static Priority Queue. This priority queue implementation depends on a static number of priorities (-20 (high) to 20 (low)) so that tuple access times can be exploited for quick in/out priority queue operations.

pqueue2
Skew Heap Priority Queue. Ulf Wiger suggested pursuing a skew heap as an optimal Erlang priority queue implementation.

pqueue3
A Large Priority Queue. This priority queue implementation depends on layered tuples, so that tuple access times can be exploited for quick in/out priority queue operations when using 64 or more total priorities.

pqueue4
Static Priority Queue. This priority queue implementation depends on a static number of priorities (-128 (high) to 128 (low)) so that tuple access times can be exploited for quick in/out priority queue operations.
