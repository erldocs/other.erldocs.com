gen_websocket

A websocket is, conceptually, like a tcp socket, therefore it should be treated as one. Use this as you would use gen_tcp, and you will know joy.

connect(Url::url(), Opts::connect_opts()) -> {ok, websocket()}

Same as connect/3 with an infinite timeout. @see connect/3

connect(Url::url(), Opts::connect_opts(), Timeout::timeout()) -> {ok, websocket()}

Connect to the given websocket server. The timeout takes into account the handshake as well. Options:

active

true, false, or once. When true, a message is sent to the owner process when a new frame is available. When once, when a frame is available a message is sent to the owner process, then the socket is set to {active, false}, also known as passive. When in passive mode, recv/1,2 are used to retreive frames.
owner_exit

close, shutdown, {shutdown, Reason}, or nothing. If the owner process exits, the websocket will take the given action. Close means the onnection to the server will close, but requests to the gen_websocket will return an {error, closed}. shutdown will exit the gen_websocket for the same reason as the owner, while {shutdown, Reason} will use the given reason. This means requests to the gen_websocket will raise an exception. Nothing means the websocket will not change behavior. However, if the websocket is in active or active_once mode, messages will be lost. Furthermore, since only the owner can transfer control, no new owner can be established.
headers

[binary() | {binary(),binary()}]. The list will be appended to the usual headers needed for a handshake.
ping

pong | deliver. What to do if gen_websocket gets a ping message from the server. If set to pong, gen_websocket will send a pong message back. When set to deliver, gen_websocket will deliver the ping like any other message, never sending a pong.

When in active mode, the message send is of form: {gen_websocket, websocket(), frame()}

recv(Socket::websocket()) -> {ok, frame()}

recv/2 with an infinte timeout. @see recv/2

recv(Socket::websocket(), Timeout::timeout()) -> {ok, frame()} | {error, timeout}

Get a single frame from the socket, giving up after the timeout.

controlling_process(Socket::websocket(), NewOwner::pid()) -> ok | {error, not_owner}

set the controlling process to a new owner. Can only succede if called from the current owner.

close(Socket::websocket()) -> ok | {error, term()}

Close the socket without exiting the process created on socket connect.

shutdown(Socket::websocket(), How::term()) -> ok

Close the socket and shut it down with the given reason.

setopts(Socket::websocket(), Opts::connect_opts()) -> ok

Set the given options on the given socket.

ping(Socket::websocket()) -> pong | pang | {error, closed}

Like ping/2 with an infinite timeout.

ping(Socket::websocket(), Timeout::timeout()) -> pong | pang | {error, closed}

Send a ping message to the server, blocking the calling process until either the timeout is reached or a reply is given.
