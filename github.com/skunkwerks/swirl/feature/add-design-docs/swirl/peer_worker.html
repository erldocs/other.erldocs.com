Wraps a gen_udp instance with associated swarm options.

Note that every port has a 1-to-1 association to a specific swarm, comprising options, IP, and port. The main reason for this is to be able to pass these swarm options through to the packet decoder when checking that the requested hash is available in this swarm or not, without requiring additional lookups.

start the server with a port number, and ppspp options which includes the root hash and chunk addressing specification. These are required during subsequent packet parsing, even if the peer supports multiple swarms. If start_link/2 is called with a root hash, the default ppspp options are assumed.
