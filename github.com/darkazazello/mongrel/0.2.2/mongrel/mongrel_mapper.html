mongrel_mapper

Mongrel mapping server.

Mongrel mapping server. This module provides the Record/Document Mapping API.

Functions

start_link(EtsTableId::integer()) -> {ok, pid()}

Spawns a registered process on the local node that stores the structure of records as in an ETS table.

add_mapping(RecordDescriptor::{atom(), FieldIds::[atom()]}) -> ok

Specfies the field identifiers associated with a record name.

get_mapping(RecordName::atom()) -> [atom()]

Gets the field identifiers associated with a record name.

is_mapped(RecordOrRecordName::atom() | record()) -> boolean()

Returns whether a record is mapped. The argument can be either an atom (a possible record name) or a tuple (a possible record).

has_id(RecordOrRecordName::atom() | record()) -> boolean()

Returns whether a record has an '_id' field. The argument can be either an atom (a possible record name) or a tuple (a possible record).

get_field(Record::record(), Field::atom()) -> any()

Gets the value of a field from a record.

set_field(Record::record(), FieldId::atom(), FieldValue::any(), MapReferenceFun::function()) -> record()

Sets the value of a field in a record. If the value is a reference to a document, a callback function is invoked to map the document to a record. An updated record is returned.

get_type(Record::record()) -> atom()

A convenience function that extracts the first element of a tuple. If the tuple is a record, the first element contains the record name.

map(Record::record()) -> {{atom(), document() (see module bson)}, [{atom(), document() (see module bson)}]}

Maps a record to a document. If the record contains child records, they are also mapped. A mapped document is a 2-tuple consisting of the collection name and the document contents.

unmap(RecordName::atom(), Document::document() (see module bson), MapReferenceFun::function()) -> record()

Unmaps a document to a record. If the document references nested documents, a callback function is called to unmap the nested documents.

map_selector(SelectorRecord::record()) -> {Colection::atom, document() (see module bson)}

Maps a selector specifying fields to match to a document. Nested records are "flattened" using the dot notation, e.g. #foo{bar = #baz{x = 3}} is mapped to the document {'bar.x', 3}.

map_projection(ProjectionRecord::record()) -> document() (see module bson)

Maps a projection specifying fields to select in a document. Nested records are "flattened" using the dot notation, e.g. #foo{bar = #baz{x = 1}} is mapped to the document {'bar.x', 1}.

map_modifier(Collection::atom(), Modifier::{Key::atom(), record()}) -> {Key::atom(), document() (see module bson), [document() (see module bson)]}

Maps a modifier specifying a field to modify to a BSON document. A modifier is an atom like '$set', '$inc', etc.

init(EtsTableId::[integer()]) -> {ok, #state{}}

Initializes the server with the ETS table used to persist the mappings needed for mapping records to documents.

handle_call(Message::tuple(), From::pid(), State::#state{}) -> {reply, Reply::any(), NewState::record()}

Responds synchronously to server calls. This function is invoked when a mapping is added or a mapping needs to be retrieved.

handle_cast(Message::any(), State::#state{}) -> {no_reply, State::#state{}}

Responds asynchronously to messages. Asynchronous messages are ignored.

handle_info(Info::any(), State::#state{}) -> {no_reply, State::#state{}}

Responds to non-OTP messages. This function should never be invoked.

terminate(Reason::any(), State::#state{}) -> ok

Handles the shutdown of the server.

code_change(OldVersion::any(), State::#state{}, Extra::any()) -> {ok, State::#state{}}

Responds to code changes.
