elysium
An example application is provided to start the root supervisor.

elysium_bs_serial
Elysium_bs_serial is a buffering strategy which enforces serial initiation of Cassandra session queries, although the duration of a query will vary so they are not guaranteed to execute in serial order.

elysium_buffer_sup
The elysium buffer supervisor is an empty supervisor.

elysium_compile_utils
Functions that Makefile can invoke from erl -s which returns things like the OTP Release.

elysium_config
Because elysium is used in high-concurrency scenarios to enhance performance, it needs to avoid process contention.

elysium_connection
An elysium_connection is an seestar_session gen_server.

elysium_connection_sup
Elysium_connection_sup supervises all Cassandra connections.

elysium_default_config
This module is an example of standard elysium configuration.

elysium_queue
Elysium_fsm is a gen_fsm that handles client queries about the state of the session, pending_requests and load balancing ets_buffer queues.

elysium_random
Random utility functions used for decay and restart delays.

elysium_sup
Elysium contains a gen_fsm which manages an ets_buffer FIFO queue and a set of Cassandra connection workers that are ordered in the queue for allocation.
