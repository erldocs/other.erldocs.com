rtmp_socket

RTMP socket module.

RTMP socket module. Designed to look like rtmp mode of usual TCP socket. If you have used {packet, http}, you will find many common behaviours.

When working on server side, you should accept Socket::port() with: 
   {ok, RTMP} = rtmp_socket:accept(Socket).
   receive
     {rtmp, RTMP, connected} ->
       rtmp_socket:setopts(RTMP, [{active, once}]),
       loop(RTMP)
   end
   loop(RTMP) ->
     receive
       {rtmp, RTMP, disconnect, Statistics} ->
         ok;
       {rtmp, RTMP, #rtmp_message{} = Message} ->
         io:format("Message: ~p~n", [Message]),
         loop(RTMP)
     end.
You are strongly advised to use {active, once} mode, because in any other case it is very easy to crash whole your server with OOM killer.

setopts(RTMP::pid(), Options::[{Key, Value}] | {Key, Value}) -> ok

Just the same as inet:setopts/2 this function changes state of rtmp socket.
Available options:

chunk_size - change outgoing chunk size

window_size - ask remote client to send read acknowlegement after WindowSize bytes

amf_version - change AMF0 to AMF3, but only before first call

consumer - change messages consumer

getopts(RTMP::pid(), Options::[{Key, Value}] | {Key, Value}) -> ok

Just the same as inet:getopts/2 this function gets state of rtmp socket.
Available options:

chunk_size - get outgoing chunk size

window_size - get remote client read acknowlegement window size bytes

amf_version - get AMF0 or AMF3

consumer - get messages consumer

address - get remote client IP and port

getstat(RTMP::pid(), Stats::[Key]) -> Values::[{Key, Value}]

Just the same as inet:getstats/2 this function gets statistics of rtmp socket.
Available options:

recv_oct - number of bytes received to the socket.

send_oct - number of bytes sent from the socket
