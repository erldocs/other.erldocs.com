erlam_sched

Functions

run(SchedOpts, Expression) -> term()

Send the initial expression to the primary processor and then waits for the result message. This function will "run" an expression using the scheduling options and nothing more.

spawn(Fun, Env) -> term()

Spawn an Erlam function as a new process using the loaded scheduling behaviour. This implementation is kinda tricky: the process which calls this will be a scheduler (through safe_step/1 -> step/2 -> safe_spawn/1), so we send an internal spawn message to self(), which it will read after the step has completed. This also has the effect of queuing the spawns.

broadcast(Message) -> term()

Broadcast a message to all schedulers. Useful for synchronizing?

send(ProcID, Message) -> term()

Message a particular scheduler on a process ID. This could be useful for passing a process between servers.

check_mq() -> term()

Check the local message queue (other side of send/2). Note that the only time this will valid is during a schedmodule:step/2 call.

check_mq(ForceHang) -> term()

get_id() -> term()

Get the calling scheduler's Logical Processor's ID or ProcID.

get_ids() -> term()

Get a list of LPU IDs where scheduler's are bound to.

is_primary() -> term()

Check if the current ID is equal to the primary id. This will tell us if the current LPU has been marked as primary by startup.

return(Process) -> term()

Return the value of the process as the result of the computation.

error(Result) -> term()

Return error status code on bad op.
