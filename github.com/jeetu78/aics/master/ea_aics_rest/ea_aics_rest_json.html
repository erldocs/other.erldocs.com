ea_aics_rest_json

AICS rest JSON validation and parsing functions.

AICS rest JSON validation and parsing functions. For the resources that use JSON as an input, it generates the internal records with the needed fields.

Types

error_reason() = missing_parameter
                       | unknown_parameter
                       | badtype_parameter
                       | duplicated_parameters
                       | unexpected_json

external_parameters() = [{binary(), any()}, ...]

field_type() = integer
                     | boolean
                     | binary
                     | natural
                     | string
                     | float
                     | json_type()

internal_parameters() =
            [{record_field_name(), record_field_value()}, ...]

json_type() = {json, validation_spec()}

record_field_name() = binary()

record_field_value() = integer()
                             | boolean()
                             | binary()
                             | string()
                             | float()
                             | internal_parameters()

validation_error() =
            {invalid_json, {error_reason(), any()}, external_parameters()}

validation_spec() =
            [{binary(), mandatory | optional, field_type()}, ...]

Functions

parse(ValidationSpec::validation_spec(), JsonFields::external_parameters()) -> internal_parameters() | validation_error()

It receives the resource tye and the fields of the JSON (output of jsx:decode()) and returns the internal record representation of the resource. In the process, it validates the input values and returns an error tuple in case of invalid input. @ TODO: At the moment it just validates and returns a list of fields. It should actually generate the record (we might use exprecs for that).
