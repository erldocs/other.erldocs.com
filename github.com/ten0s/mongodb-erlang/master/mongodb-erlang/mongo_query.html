Functions

write(DbConn::dbconnection() (see module mongo_connect), Write::write(), GetlasterrorParams::getlasterror_request()) -> getlasterror_reply()

Send write and getlasterror request to mongodb over connection and wait for and return getlasterror reply. Bad getlasterror params are ignored. Caller is responsible for checking for error in reply; if 'err' field is undefined then success, otherwise it holds error message string.

write(DbConn::dbconnection() (see module mongo_connect), Write::write()) -> ok

Send write to mongodb over connection asynchronously. Does not wait for reply hence may silently fail. Doesn't even throw connection failure if connection is closed.

command(DbConn::dbconnection() (see module mongo_connect), Command::command(), SlaveOk::boolean()) -> document() (see module bson)

Send command to mongodb over connection and wait for reply and return it. Boolean arg indicates slave-ok or not. 'bad_command' error if bad command.

find_one(DbConn::dbconnection() (see module mongo_connect), Query::query()) -> maybe(document() (see module bson))

Send read request to mongodb over connection and wait for reply. Return first result or nothing if empty.

find(DbConn::dbconnection() (see module mongo_connect), Query::query()) -> cursor() (see module mongo_cursor)

Send read request to mongodb over connection and wait for reply of first batch. Return a cursor holding this batch and able to fetch next batch on demand.
