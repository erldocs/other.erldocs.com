eipmi_encoder

A module providing encoding functionality for RMCP packets.

A module providing encoding functionality for RMCP packets.

Functions

ack(Rmcp_header::#rmcp_header{version=0..255, seq_nr=0..255, class=0..255}) -> binary()

Encodes a RMCP ACK packet. An ACK is a requested response to a formerly received message. To be able to associate the ACK with the message it is generated for specific values must/will be copied from the header of the received message.

ping(Rmcp_header::#rmcp_header{version=0..255, seq_nr=0..255, class=0..255}, Asf_ping::#asf_ping{iana=non_neg_integer(), tag=0..255}) -> binary()

Encodes a RMCP ASF Ping packet. A Ping is usually sent to retrieve the IPMI capabilities of the far end.

ipmi(Rmcp_header::#rmcp_header{version=0..255, seq_nr=0..255, class=0..255}, Properties::proplist() (see module proplists), Req::request() (see module eipmi), Data::binary()) -> binary()

Encodes a RMCP IPMI packet. This encapsulates the logic of binary packet construction with authentication and checksum calculation. Please note that currently only IPMI requests are supported.

request(Properties::proplist() (see module proplists), X2::{req_net_fn() (see module eipmi), 0..255}, Data::binary()) -> binary()

Encodes a raw IPMB request according the standard format: rsSA, netFn/rsLUN, chk1, rqSA, rqSeq/rqLUN, cmd, <data>, chk2 Calling this directly is useful for e.g. bridged requests. Refer to chapter 6.13, BMC Message Bridging in the IPMI specification.
