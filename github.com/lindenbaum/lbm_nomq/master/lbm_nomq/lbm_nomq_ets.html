lbm_nomq_ets

An implementation of the lbm_nomq_dist behaviour based on a local ETS table using global locks to distribute subscriptions.

An implementation of the lbm_nomq_dist behaviour based on a local ETS table using global locks to distribute subscriptions. This is quite similar to how pg2 distributes its internal state.

The table ?MODULE contains the following terms: {{topic, Topic}, Counter}: a topic and its current subscriber counter {{subscriber, Topic, #lbm_nomq_subscr{}}}: a subscriber {{waiting, Topic, {pid(), reference()}}}: a waiting pusher
