swc_vv

An Erlang implementation of a Version Vector.

An Erlang implementation of a Version Vector.

Types

counter() = integer() >= 0

id() = term()

vv() = [{Key :: id(), Entry :: counter()}]

Functions

new() -> vv()

Initializes a new empty version vector.

ids(V::vv()) -> [id()]

Returns all the keys (ids) from a VV.

get(K::id(), V::vv()) -> {ok, counter()} | error

Returns the counter associated with an id K. If the key is not present in the VV, it returns 0.

join(A::vv(), B::vv()) -> vv()

Merges or joins two VVs, taking the maximum counter if an entry is present in both VVs.

filter(F::(id(), counter()) -> boolean(), V::vv()) -> vv()

It applies some boolean function F to all entries in the VV, removing those that return False when F is used.

add(VV::vv(), X2::{id(), counter()}) -> vv()

Adds an entry {Id, Counter} to the VV, performing the maximum between both counters, if the entry already exists.

min(VV::vv()) -> counter()

Returns the minimum counters from all the entries in the VV.

min_key(VV::vv()) -> id()

Returns the key with the minimum counter associated.

reset_with_same_ids(VV::vv()) -> vv()

Returns the VV with the same entries, but with counters at zero.

delete_key(VV::vv(), Key::id()) -> vv()

Returns the VV without the entry with a given key.
