war_server_callback

Server callback.

Server callback

Functionalities: 1. Depending on the mode (active), execute or queue the decisions 2. Allow subcribers, who will receive the decisions

Working: All "CLUSTER" commands are handled by the gen_server

Active state: Commands are passed on to the war_db module and results returned

In-active state: Commands are queued in this state. When "trig_active" is called, create a marker in the queue and start processing it. The decisions are sent to the subscriber till we reach the mark and then we remove the subscriber and process the rest of the queue. When queue empty, switch back to active state.

Note: Reponses for "CLUSTER" commands are lost in inactive state
