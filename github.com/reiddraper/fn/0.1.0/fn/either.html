either

An implementation of the Either monad, but with an emphasis on being compatible with existing Erlang code.

An implementation of the Either monad, but with an emphasis on being compatible with existing Erlang code. Hence using {ok, ...} and {error, ...} instead of Right and Left (Right is success, Left is failure, traditionally). Otherwise, nomenclature should be identical to Haskell.

Types

either(A, B) = {ok, A} | {error, B}

Functions

return(A) -> either(A, term())

Wrap A in {ok, A}. Put another way, returns puts A into the Either monad.

bind(E::either(A, B), F::(A) -> either(A, B)) -> either(A, B)

Apply F to the value inside the monad if it's {ok, ...}, otherwise just return the error tuple.

lift(M::either(A, Z), F::(A) -> C) -> either(C, Z)

Lets you use a function to act on an Either monad, but your function can return a value. It will automatically be wrapped back into an {ok, ...} tuple.

pipe(M::either(A, Z), Funs::[(A) -> C]) -> either(C, Z)

Lets you chain monadic function to apply to M. You might use it like pipe({ok, 5}, [Increment, Increment, Increment]).
