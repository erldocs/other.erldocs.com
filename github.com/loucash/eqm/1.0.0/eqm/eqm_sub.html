eqm_sub

A Subscriber is a component that accepts a sequenced stream of elements provided by a Publisher and pass them to an owner.

A Subscriber is a component that accepts a sequenced stream of elements provided by a Publisher and pass them to an owner. At any given time a Subscriber might be subscribed to at most one Publisher.

Types

subscriber()

Functions

start_link(Publisher, Owner, Size, StateName) -> term()

Starts new subscriber process which subscribes at start to publisher. The initial state is wait, process is waiting for subscription from publisher.

cancel(Sub::subscriber()) -> ok

Cancel subscription, returns all messages from buffer and stops subscriber.

notify(Sub::subscriber()) -> ok

Forces the buffer into its notify state, where it will send a single message alerting the Owner of new messages before going back to the passive state.

active(Sub::subscriber()) -> ok

Forces the buffer into an active state where it will send the data it has accumulated.

resize(Sub::subscriber(), Size::pos_integer()) -> ok

Allows to take a given buffer, and make it larger or smaller.

info(Sub::subscriber()) -> {ok, {non_neg_integer(), non_neg_integer()}}

Returns info about buffer, size and number of messages

stop(Sub::subscriber()) -> ok

Stops subscriber and drops messages from buffer

ref(Subscriber::subscriber()) -> reference()

pid(Subscriber::subscriber()) -> pid()

subscription(Sub::pid(), Subscription::subscription() (see module eqm_pub)) -> ok

Publisher calls this function after successful subscription.

post(Sub::pid(), Msg::any()) -> ok

Publisher calls this function to send new message.

cancelled(Sub::pid()) -> ok

Publisher calls this function to confirm that subscription is cancelled.
