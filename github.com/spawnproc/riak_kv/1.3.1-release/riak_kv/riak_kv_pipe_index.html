riak_kv_pipe_index

Query indexes stored on the local Riak KV vnode sharing the same index as the pipe vnode owning this worker.

Query indexes stored on the local Riak KV vnode sharing the same index as the pipe vnode owning this worker. This is the same idea as riak_kv_pipe_get, but with the listkeys operation instead of get.

Inputs to this worker may be {Bucket :: binary, Query :: query()}. Query may take the form of either {eq, Index, Key} or {range Index, StartKey, EndKey}'.

This fitting also understands cover input, as generated by riak_pipe_qcover_fsm. When processing cover input, the FilterVNodes parameter is passed along to the KV vnode for filtering there. (When not processing cover input, the KV vnode is passed an empty FilterVNodes list.)

The convenience function queue_existing_pipe/3 can be used to list the index matches directly into an existing pipe.

Query and index, and send the results as inputs to the given pipe. This starts a new pipe with one fitting (riak_kv_pipe_index), with its sink pointed at the destination pipe. The riak_pipe_qcover_fsm module is used to trigger querying on the appropriate vnodes. The eoi message is sent to the pipe as soon as it is confirmed that all querying processes have started.

Note that log/trace messages are sent to the sink of the original pipe. It is expected that that sink is an fsm type.
