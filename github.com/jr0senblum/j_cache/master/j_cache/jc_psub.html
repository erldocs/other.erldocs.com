jc_psub

jc_psub 1) notifies clients of changes to j_cache items and provides varying degrees of information about the change depending on the subscription type; and 2) allows clients to subscribe to and broadcast topic events.

jc_psub 1) notifies clients of changes to j_cache items and provides varying degrees of information about the change depending on the subscription type; and 2) allows clients to subscribe to and broadcast topic events.

Subscriptions are a local afair. If a subscriber's node goes down, clients will need to re-subscribe.

Mnesa LOCAL tables, ps_sub and ps_client, are used:

ps_sub maps a subscription to a list of listening Pids ps_client lists the unique pids, a reference used to demonitor them/check to see if they are still up.

load() -> non_neg_integer() | {error, any()}

Return an estimate of how much load the server is under. It counts the number of subscriptions + the number of Pids that are currenlty subscribed to anything. Under extreme load this can time-out resulting in an error.

client_count() -> non_neg_integer() | {error, any()}

Return the number of distinct, subscribing Pids being served by the server. Under extreme load this function can time-out.
