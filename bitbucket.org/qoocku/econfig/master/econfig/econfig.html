subscribe(App::atom()) -> ok

Subscribes calling process onto this application subscription list. The application processes publish message where App is id of an application for which Value has been set to the parameter pointed by KeyPath.

subscribe() -> ok

Subscribes the calling process ont its application subscription list.

Retrieves current application configuration value pointed by given key path.

get(X1::atom() | key_path(), X2::key_path() | any()) -> any()

Retrieves a configuration value pointed by given key path. If X1 is an atom it denotes an application. X1 may be a list of atoms (key path) so then X1 is the default value if the entry is not found. Otherwise X2 is a key path. In the latter case an error is raised if the entry is not found.

get(App::atom(), KeyPath::key_path(), Default::any()) -> any()

Retrieves a configuration value pointed by given key path. If value does not exists it returns Default.

set(Keys::key_path(), Value::any()) -> ok

Sets a confiuration value for current application.

copy_from(Node::node(), App::atom()) -> ok

Copies App configuration from Node node.

handle_info(X1::change_info(), Mod::module(), State::any()) -> NewState::any()

Utility for process handling configuration change. It may be used in gen_servers like this: 

  handle_info (...) ->
    ...;
  handle_info (Info, State) ->
    {noreply, econfig:handle_info(Info, ?MODULE, State)}.

or in gen_event handlers: 

  handle_info (...) ->
    ...;
  handle_info (Info, State) ->
    {ok, econfig:handle_info(Info, ?MODULE, State)}.

See also: subscribe/0, subscribe/1.
