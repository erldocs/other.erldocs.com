Parse transformer that simplifies the partial application and the composistion of functions in Erlang.

Performs parse transformion that modifies the semantic of Erlang to simplify the writing of partial application and composistion of functions expressions. To deal with this goal and make the parsing easier, This module also implements the gen_trans behaviour.


EXPORTS 

 parse_transform(Forms, Options) -> NewForms | {error, Errors, []}

Types: 
      Forms, NewForms = [erlang_form()]
      Options = [compiler_options()]
      Errors = [{Filename, [ErrorInfo]}]
         FileName = atom() | string()
         ErrorInfo = see separate description below.

Implements the actual transformation at compile time. This function is called by the compiler to do the source code transformation if and when the option {parse_transform, lambda_expr} is passed to the compiler. This function starts a new gen_trans by calling gen_trans:start/3 with lambda_expr as callback module.

If the abstract code format is successfully parsed, the function returns NewForms. if it fails the function returns {error, Errors, []}.

See compile(3).

 format_error(ErrCode) -> ErrMessage

Types: 
      Errcode = term()
      ErrMessage = string()

Takes an error code returned by one of the other functions in the module and creates a textual description of the error. The ErrCode can be either of the following terms. The format_error/1 function returns a descriptive string which describes the error.

lambda_expr_badarg. Argument of lambda/1 is not a lambda expression. {lambda_expr_badarity, N}. lambda/1 called with N arguments. (N /= 1) {reserved_placeholder_usage, Var}. Variable Var: syntax reserved for placeholders. invalid_placeholder_number. Placeholder _0 is illegal. Other ErrCode. Unknown error: ErrCode.


Error Information

The ErrorInfo mentioned above is the standard ErrorInfo structure which is returned from lambda_expr module. It has the following format:

    {ErrorLine, lambda_expr, ErrorDescriptor}

A string which describes the error is obtained with the following call:

    apply(lambda_expr, format_error, ErrorDescriptor)
