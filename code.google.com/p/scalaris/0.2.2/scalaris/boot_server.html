boot_server

The boot server maintains a list of chord# nodes and checks the availability using a failure_detector.

The boot server maintains a list of chord# nodes and checks the availability using a failure_detector. It also exports a webpage on port 8000 containing some statistics. Its main purpose is to give new chord# nodes a list of nodes already in the system.

Functions

returns the number of nodes known to the boot server

connect() -> term()

node_list() -> [pid()]

returns all nodes known to the boot server

ping() -> [int]

pings all known nodes

ping(PID) -> term()

starts the server; called by the boot supervisor
