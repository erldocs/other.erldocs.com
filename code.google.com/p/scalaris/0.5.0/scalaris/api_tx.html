api_tx

API for transactional access to replicated DHT items.

API for transactional access to replicated DHT items. For single request/single item operations, we provide read/1, write/2, and test_and_set/3 functions that directly commit.

For compound transactions a transaction log has to be passed through all operations and finally has to be committed. This is supported by the functions new_tlog/0, read/2, write/3, req_list/2, and commit/1.

Types

client_key() = string()

client_value() = any()

commit_result() = {ok}
                        | {fail, timeout}
                        | {fail, abort, [client_key()]}

listop_result() = write_result() | {fail, not_a_list}

numberop_result() = write_result() | {fail, not_a_number}

read_result() = {ok, client_value()}
                      | {fail, timeout | not_found}

request() = request_on_key() | {commit}

request_on_key() = {read, client_key()}
                         | write_request()
                         | {add_del_on_list,
                            client_key(),
                            client_value(),
                            client_value()}
                         | {add_on_nr, client_key(), client_value()}
                         | {test_and_set,
                            client_key(),
                            Old :: client_value(),
                            New :: client_value()}

result() = read_result()
                 | write_result()
                 | listop_result()
                 | numberop_result()
                 | testandset_result()
                 | commit_result()

testandset_result() = write_result()
                            | {fail,
                               not_found |
                               {key_changed,
                                RealOldValue :: client_value()}}

write_request() = {write, client_key(), client_value()}

write_result() = {ok} | {fail, timeout}
