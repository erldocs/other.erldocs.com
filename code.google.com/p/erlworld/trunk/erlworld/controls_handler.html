controls_handler

This is a data collection for storing the current state of the controls.

This is a data collection for storing the current state of the controls. You can give it mouse locations and the press and release of mouse buttons and keys. You can also grab the current state of the controls using the get_controls function.

Functions

new() -> ControlsHandler::controls_handler()

Creates a new process that stores controls changes. This can then be pinged for a snapshot of the controls.

set_mouse_location(CHandler::controls_handler(), Location::{X::interger(), Y::integer}) -> NewCHandler::controls_handler()

Sets the mouse location stored in this ControlsHandler to the location given.

set_mouse_down(CHandler::controls_handler(), Mouse::left | middle | right) -> NewCHandler::controls_handler()

Sets the button to be down for the value given. The value given should be one of the mouse button constants.

set_mouse_up(CHandler::controls_handler(), Mouse::left | middle | right) -> NewCHandler::controls_handler()

Sets the button to be up for the mouse button given.

set_key_char(CHandler::controls_handler(), Key::integer()) -> NewCHandler::controls_handler()

Sets the current key character we are currently looking at. Any keypresses that come in before or after this character which are not mapped will be presumed to map against this char.

set_key_down(CHandler::controls_handler(), Key::integer()) -> NewCHandler::controls_handler()

Sets the key to be down for the given key code.

set_key_up(CHandler::controls_handler(), Key::integer()) -> NewCHandler::controls_handler()

Sets the key to be up for the given key code. Any characters mapped against the given key will no longer be mapped as being down.

get_controls(CHandler::controls_handler()) -> Controls::controls()

Requests a snapshot current state of the controls from this Controls Handler.
