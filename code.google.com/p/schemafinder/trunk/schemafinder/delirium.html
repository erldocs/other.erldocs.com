delirium

Functions

admit() -> {atomic, ok} | {aborted, Reason}

Query for the current node to be admitted to the global schema. If this function fails then the current node must not join the global schema. mnesia should not be started until after this routine says ok.

condemn() -> {atomic, ok} | {aborted, Reason}

Equivalent to condemn(node()).

condemn(Node::atom()) -> {atomic, ok} | {aborted, Reason}

Mark a node as dead. It will be removed on the next delirium pass.

force_prune() -> {ok, pid()} | {error, {prune_underway, pid()}}

Force an immediate removal. The delirium server will periodically remove condemned or timed-out nodes. This function will force an immediate removal.

heartbeat() -> {atomic, ok} | {aborted, Reason}

Equivalent to heartbeat(node()).

heartbeat(Node::atom()) -> {atomic, ok} | {aborted, Reason}

Update the timestamp associated with a node, in order to avoid automatic removal.

immortalize() -> {atomic, ok} | {aborted, Reason}

Equivalent to immortalize(node()).

immortalize(Node::atom()) -> {atomic, ok} | {aborted, Reason}

Mark a node as immortal, so that it will not be removed until heartbeat () or condemn () are called on it. This is useful for when a node is being taken down for planned but extensive maintainenance.

immortalize_remote() -> {atomic, ok} | {aborted, Reason}

Equivalent to immortalize_remote(node()).

immortalize_remote(Node::atom()) -> {atomic, ok} | {aborted, Reason}

Mark a node as immortal via another node. Does not require mnesia to be started.

See also: immortalize/1.

sanctify() -> {atomic, ok} | {aborted, Reason}

Equivalent to sanctify(node()).

sanctify(Node::atom()) -> {atomic, ok} | {aborted, Reason}

Mark a node as sanctified. It will be allowed to join the schema again. This can only be called on buried nodes.
