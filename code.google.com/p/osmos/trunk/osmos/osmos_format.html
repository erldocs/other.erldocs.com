osmos_format

A method of encoding and decoding some set of Erlang terms to and from binaries.

A method of encoding and decoding some set of Erlang terms to and from binaries.

An #osmos_format{} record has two fields:

to_binary ((any()) -> binary()): convert a term to binary. This is not required to accept all Erlang terms.

from_binary ((binary()) -> any()): convert a binary returned by the to_binary function back to the original term.

The convenience functions in this module return #osmos_format{} records for several common term encodings.

Functions

binary() -> #osmos_format{}

A binary. The binary format is the identical binary.

term() -> #osmos_format{}

An erlang term. The binary format is the erlang external term format.

uint64_delete() -> #osmos_format{}

An integer in the range [0,2^64), or the atom 'delete'. The binary format is an unsigned 64-bit big-endian integer, or an empty binary to indicate deletion.

uint64_vector_delete() -> #osmos_format{}

A tuple of one or more integers, each in the range [0,2^64), or the atom 'delete'. The binary format is a concatenated sequence of big-endian, unsigned 64-bit integers, or an empty binary to indicate deletion.

string_vector() -> #osmos_format{}

A tuple of strings as binaries, which may not contain ASCII NUL. The binary format is the strings NUL-terminated and concatenated.
