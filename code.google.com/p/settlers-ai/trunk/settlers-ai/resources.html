resources

This module keeps track of the resources and is used for calculating what can be built with the resources.

This module keeps track of the resources and is used for calculating what can be built with the resources. It communicates with the trader module to obtain possible trades.

Functions

init() -> Pid

The init function spawn the resources process.

resources(Res, Pieces, IDs) -> ok

The process that keeps track of the resources and recieves messages from the actuator and devCards processes with instructions. It send messages to the trader to get the possible trades.

discardCards(Amount, Res) -> {NewRes, DiscardRes}

Discard cards when a seven is roll and we have more than seven cards. It first try to discard wood and clay and if that's not enough it discard other resources too. It uses a round robin rule so it discard evenly over the resources it tries to discard.

getBestHand(Res, City) -> {BestHand, Builds, Value}

Takes a list of hands (generated by the trader) and calls getBestBuild to evaluate them and then compare them to each other. City is used when callling getBestBuild

getBestBuild(Res, City) -> {Builds, Value}

Calculates all the possible builds that can be done with a hand and then weights them and returns the best builds. City is the number of cities we have built and is used to update the weights according to the strategy used.
